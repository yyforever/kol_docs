# 多邮箱绑定与发送（从 xxl0119 拆出）

> 来源：xxl0119 版本拆分，功能复杂度较高，单独立项
> 创建：2026-03-11
> 状态：需求梳理中

---

## 原始用户反馈（原样记录）

> 邮件邀约—发件人设置，支持绑定多个gmail邮箱，进行发送。客户反馈，聚星邮箱回复率没有gmail好，gmail有大量发件要求，需要绑定多个gmail。其他竞品可以绑定多个gmail。而且这点是竞品作为与我们对比的主要优势和卖点。+企业邮箱也需要同时绑定多个（1人负责多个品牌的场景）。场景上需要支持多品牌指定邮箱，也需要支持单品牌邮箱池防封。

## 现状分析（基于线上截图 2026-03-11）

### 发件设置（账户级）

- 3 个 Tab：**Nox聚星邮箱** / **谷歌邮箱** / **企业邮箱**
- 每种类型**只能绑定 1 个**邮箱
- Gmail 通过 OAuth 授权绑定，提示"新注册邮箱限制外发频率 20封/小时"

### 邮件管理 — 创建邮件项目 / 编辑邮件

- 发件人选择：**radio 单选**，从已绑定的 3 个邮箱中选 1 个
- 不支持多选、不支持轮发、不支持按品牌分组
- 未设置的类型灰显不可选

### 消息中心 — 回复网红

- 回复框只显示"回复至：{网红名}"，**没有发件人选择器**
- 以前只有 1 个邮箱所以不需要选；多邮箱后必须支持切换发件人

### 现状核心限制

```
当前：每种邮箱类型 → 1个账号 → 项目/回复均单发件人，无需选择
目标：每种邮箱类型 → N个账号 → 项目可选邮箱池，回复可切换发件人
```

## 改造范围

> Nox聚星邮箱不在本次范围内（背后已有独立邮箱池，逻辑不同）。
> 本次只针对 **Gmail** 和 **企业邮箱**。

### 改造点 1：发件设置 — 支持多邮箱绑定

- 谷歌邮箱 tab：从"绑定 1 个"改为**邮箱列表 + 添加按钮**
- 企业邮箱 tab：同上
- Nox聚星邮箱 tab：不改动

### 改造点 2：邮件管理 — 发件人选择改为二级多选

现行：3 个 radio 平铺单选

改为**二级选择**：
- 第一级：邮箱类型单选（Nox聚星 / 谷歌 / 企业）
- 第二级：该类型下的具体邮箱**多选**（checkbox）
- 选中谷歌或企业时，**默认勾选第一个**邮箱
- 选中多个 = 邮箱池模式，系统自动轮发（后端简单轮询）

### 改造点 3：消息中心 — 回复时支持切换发件人

- 回复框增加**发件人选择器**
- 默认使用该会话原始发件邮箱（保持对网红的一致性）
- 可手动切换为其他已绑定邮箱（少数场景，如原邮箱被封）

## 用户场景

### 场景 A：多品牌指定邮箱

1 人运营多个品牌，需要用不同邮箱发邀约（身份隔离）。

- 绑定多个 Gmail（brand-a@gmail.com, brand-b@gmail.com, ...）
- 绑定多个企业邮箱（同理）
- 创建项目时选择对应品牌的邮箱（选 1 个）

### 场景 B：单品牌邮箱池防封

同一品牌大量外发，需分散到多个 Gmail 避免触发限流/封号。

- 同品牌绑定 N 个 Gmail
- 创建项目时多选邮箱（选 N 个），系统自动轮发
- 需要限流监控（Gmail 500/2000 封/天限制）

### 竞争压力

- 竞品已支持多 Gmail 绑定，作为与聚星对比的**主要卖点**
- 属于竞争防御性需求，不做会持续丢单
