## 当前定位

`kol-creator / 聚星-网红经纪人` 是网红端产品。当前阶段聚焦第一版目标人群、核心痛点和“AI 商单经纪人 / 商单处理台”假设的验证与打磨。

核心研究问题仍然是：

```text
哪一类网红，在哪一种赚钱场景里，遇到什么问题，
已经愿意付出时间、沟通、机会成本或佣金来解决？
```

当前最值得验证的痛点是：

```text
有一定内容能力、愿意接品牌合作的中小网红，
缺少可持续获得、可信、匹配、报酬合理、
能成交并能顺利完成的品牌合作来源。

即使看到合作机会，他们也常常难判断：
这单是否真实、是否适合自己、钱和样品是否值得、
品牌是否可能选中、交付和付款是否可靠、做完后是否还有后续。
```

当前要验证的不是孤立表单、资料页或合作证明，而是这条赚钱链路里哪个环节最痛：触达靠谱机会、判断是否值得、进入品牌筛选、谈条件、完成交付、到账和获得后续。

当前更准确的第一版产品假设是：

```text
面向有内容基础、愿意接品牌合作、但没有稳定商单来源的中小达人，
提供一个 AI 商单经纪人：
帮他发现、收拢、判断和回复品牌合作机会，
让他少浪费时间在低质机会和无效沟通上，
更快把值得做的机会推进到报价、确认和成交。
```

这仍然是假设，不是已被证明的结论。下一步调研要验证的是：这个 AI 商单经纪人是否真的比“自有 Offer 承接”“任务列表”“资料页”更命中目标达人痛点。

## 研究边界

- 第一版用户必须是一类具体网红；品牌、seller、聚星运营和业务团队是交易关系方与证据来源。
- 内部项目用于理解真实交易如何发生、品牌如何筛人、达人为什么拒绝或拖延。
- `资料提交`、`商业档案`、`合作证明` 属于可能机制，只有在帮助网红拿到更靠谱、更合适、能成交并能完成的合作时才有价值。
- 当前阶段允许把“AI 商单经纪人 / 商单处理台”作为候选方案验证，但不能把它当作已成立结论。
- 商业合作机会管道是后续可能演化方向；第一版先验证“真实商单处理”是否成立。
- 不预设开放任务市场、完整交易平台、SaaS 订阅工具或纯数据分析产品。
- `Agent / AI` 只能作为降低报价、沟通、机会判断成本的机制变量；不能直接替代痛点证明。
- 可以复用 kol-agent / Openclaw 类底层能力，但这不是当前阶段的产品定位前提。
- 品牌上倾向子品牌定位；NoxInfluencer 可作为母品牌背书，不要求完全避开。

## 下一步

下一步用真实商单处理校验痛点。按“解剖麻雀”的方式做，不追求材料多，追求链路完整：

- [ ] 选定 3-5 个内部真实品牌合作项目，作为高可信样本，每个项目必须能追到 `邀约 -> 回复/沉默 -> 报价/拒绝 -> 品牌筛选 -> 确认 -> 上线/掉队 -> 付款/复购`。
- [ ] 补充验证其他商单来源：连接 Gmail 导入达人已有商单邀约，并做外部公开 Offer 聚合样本；聚合边界继续调研，第一阶段先看质量和价值。主动外联第一版先不优先做。
- [ ] 每个项目至少覆盖一种不同矛盾：价格低、样品/产品价值、垂类不匹配、确认后不上线、专业垂类高门槛、顺利完成。
- [ ] 对每个项目写出达人最关心的信息：钱、样品、品牌、产品、交付要求、付款方式、是否可长期合作、报价机制。
- [ ] 主动邀约合适网红，也允许达人浏览精选机会，记录他们是否愿意继续了解、最先问什么、为什么拒绝、为什么沉默。
- [ ] 记录品牌最终为什么选中或拒绝达人，重点看均播、受众、互动率、内容质感、报价、品类适配和交付风险。
- [ ] 记录确认后不上线的真实原因，重点看脚本/初稿修改、达人拖延、样品没到、产品问题、付款和授权。
- [ ] 主动找反例：有好项目但不行动、价格不错但拒绝、样品高但不适合、资料完整但品牌不选、确认后仍掉队。
- [ ] 再判断是否需要产品化为商单收件箱、商单判断页、报价/回复辅助、账号数据授权、机会分发、商业档案、合作证明或其他形态；第一版核心指标看品牌方接受报价或返回报价的比例。
