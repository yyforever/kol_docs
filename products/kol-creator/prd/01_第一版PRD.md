# 聚星-网红经纪人第一版 PRD

> 状态：第一版产品假设，待真实验证
> 更新时间：2026-05-17
> 调研依据：`research/01`、`02`、`03`、`05`、`10`、`11`、`13`

---

## 1. 关键推理链

目标用户先聚焦：

```text
有内容基础、愿意接品牌合作，
但没有稳定商单来源的中小品牌内容创作者 / 商单内容创作者。
```

调研和内部项目共同指向一个判断：

```text
这类达人真正缺的不是“更多列表”或“资料页”，
而是更稳定地拿到值得处理的品牌合作机会，
并把其中合适的机会更省力地推进到报价、确认和成交。
```

推理链：

1. 目标达人想要的是收入，收入来自真实品牌合作，不来自“看到机会”本身。
2. 品牌合作能否变成收入，取决于机会是否真实、匹配、报酬合理、交付可控、付款可靠。
3. 现有机会来源碎片化，达人需要先花时间判断值不值得继续，再报价、追问、议价和跟进。
4. 内部项目显示，达人拒绝和掉队常见原因是价格低、产品/样品不匹配、脚本反复改、达人拖延、样品没到；品牌筛人主要看均播、受众、互动率、内容质感、报价和交付风险。
5. 因此第一版不应先做泛 Offer 平台、资料页或完整交易平台，而应先做“真实商单处理”。
6. AI 的首要价值不是替达人完成项目执行，而是降低商单判断、报价、回复、追问和跟进成本。
7. 如果产品有效，应该能让更多值得继续的商单推进到明确价格互动：品牌接受报价，或品牌返回新的报价 / counteroffer。

第一版产品定位：

```text
AI 商单经纪人 / 商单处理台。
```

核心承诺：

```text
帮达人发现、收拢、判断和回复品牌合作机会，
让达人少浪费时间在低质机会和无效沟通上，
更快把值得做的机会推进到报价、确认和成交。
```

第一版不承诺稳定赚钱，也不承诺一定接到单；只验证平台能否更好地处理真实商单机会。

## 2. 第一版产品闭环

第一版只做一个闭环：

```text
商单进入
-> AI 判断
-> AI 起草报价 / 回复 / 追问
-> 达人确认关键动作
-> 记录品牌反馈和后续结果
```

### 2.1 商单来源

| 来源 | 第一版做法 | 验证重点 |
|---|---|---|
| 聚星自有商单 | 作为高可信样本推给匹配达人 | 能否驱动真实行动和闭环结果 |
| 达人已有 Gmail 商单邀约 | 连接 Gmail 并导入相关邀约 | 达人是否愿意交给平台判断和回复 |
| 外部公开 Offer 聚合 | 收集公开机会并结构化解释；第一阶段先看质量和价值 | 能否筛掉低质机会，而不是堆噪音 |
| 主动品牌外联 | 第一版不优先做，只保留后续方向 | 后续再验证是否能带来有效回复 |

外部公开 Offer 聚合的第一阶段不把合规边界当作主验证问题，主验证问题是：

```text
这些公开机会是否足够真实、匹配、报酬合理，并且值得达人继续处理。
```

但仍应避免明显高风险来源，例如需要绕过登录限制、违反平台访问限制或无法保留原始来源的机会。

### 2.2 商单收件箱

商单收件箱是“待处理商单”的统一工作台，不是普通任务列表。

每条商单至少记录：

- 来源：自有商单 / Gmail / 外部公开 Offer。
- 品牌、产品、平台、合作类型。
- 报酬、样品、佣金或预算信息。
- 交付物、时间、修改次数、使用权、付款条件。
- AI 判断：建议继续 / 谨慎继续 / 不建议继续。
- 下一步动作：报价 / 追问 / 回复 / 拒绝 / 暂存。
- 当前状态和结果。

### 2.3 商单判断页

每个商单必须给出明确判断：

```text
建议继续 / 谨慎继续 / 不建议继续
```

判断必须回答：

- 这单为什么值得或不值得继续。
- 钱、样品、佣金或报价机制是否清楚。
- 工作量、交付要求和修改成本是否合理。
- 使用权、广告投放、whitelisting、raw files、exclusivity 是否有风险。
- 付款方式和结算周期是否明确。
- 产品、品牌和达人内容是否匹配。
- 下一步必须追问什么。

判断页的标准不是字段完整，而是达人看完能决定：

```text
继续谈 / 报价 / 追问 / 拒绝 / 暂存
```

### 2.4 报价建议

报价建议必须给谈判依据，不只是给一个数字。

需要输出：

- 建议报价区间。
- 依据：账号数据、内容类型、交付物、使用权、修改次数、付款周期。
- 可谈条件：缩短授权、减少 rights、增加样品、加佣金、降低修改次数。
- 风险提醒：低价买断、长期授权、无限修改、付款后置。

如果数据不足，直接提示：

```text
当前无法给出可靠报价，需要补充账号数据 / 交付要求 / rights / 预算范围。
```

### 2.5 回复与跟进助手

第一版支持：

- 接受邀约回复。
- 报价回复。
- 条款追问。
- 礼貌拒绝。
- 议价回复。
- follow-up。
- 长期合作维护。

自动化边界：

| 类型 | 规则 |
|---|---|
| 必须达人确认 | 首次对品牌发送、报价、接受关键条款、拒绝合作、涉及价格/付款/rights 的回复 |
| 可自动处理 | 解析商单、提取字段、识别风险、生成草稿、提醒 follow-up、归档状态、更新记录 |
| 后续可半自动 | 在达人授权规则明确后，对低风险 follow-up 自动发送 |

原则：关键节点让达人确认，其他低风险步骤尽量自动。

### 2.6 结果记录

结果记录用于验证产品是否有效，不做完整 CRM。

必须记录：

- 商单来源。
- 达人是否查看。
- 是否采用 AI 判断、报价或回复。
- 是否发送回复。
- 品牌是否回复。
- 品牌是否接受报价或返回报价。
- 后续是否确认、交付、上线、付款。
- 掉队原因。
- 达人是否愿意下次继续使用。

## 3. 验证口径

### 3.1 核心指标

第一版核心指标不是商单数量、打开率或资料填写率，而是：

```text
品牌方接受报价或返回报价的比例。
```

建议统计口径：

```text
分母：达人已通过平台向品牌发送报价或价格相关回复的商单。
分子：品牌接受报价，或品牌返回新的报价 / 明确 counteroffer 的商单。
```

只计入核心指标：

- 品牌接受达人报价。
- 品牌返回新的报价或明确 counteroffer。

辅助指标：

- AI 判断/报价/回复采用率。
- Gmail 邀约导入后的有效商单识别率。
- 外部公开 Offer 聚合后的有效商单占比。
- 品牌有效回复率。
- 达人发送报价率。
- 达人下次继续使用意愿。

### 3.2 成立条件

第一版成立需要看到：

- 目标达人愿意连接 Gmail、导入真实商单，或愿意处理平台提供的商单。
- 达人认为 AI 判断、报价和回复至少更省时间，且愿意采用。
- 非自有来源能产生有效商单，不只是低质噪音。
- 平台能推动一部分商单进入核心指标：品牌接受报价或返回报价。
- 达人愿意下次继续使用。

### 3.3 反例

出现以下情况，要调整方向：

- 达人不愿意把真实商单交给平台处理。
- AI 判断看起来正确，但达人不采用。
- AI 回复让品牌体验变差。
- 非自有来源只能产生低质机会，无法推进到报价互动。
- 自有商单真实、报酬合理，但目标达人仍不行动。
- 达人行动了，但品牌大多不选，且原因不可被平台改善。

## 4. 第一版不做

第一版不做：

- 完全开放任务市场。
- 品牌自助发布后台。
- 完整 CRM。
- 合同、付款、结算闭环。
- 无确认的关键节点自动发送。
- 明显高风险的数据抓取。
- 把商业档案、合作证明、履约管理当作默认主功能。

这些功能只有在结果记录显示它们是关键断点时，才进入下一轮。

## 5. 第一轮实施清单

先完成：

- 选定第一批品牌内容创作者样本。
- 选定 `3-5` 个聚星自有真实品牌项目，作为高可信闭环样本。
- 接入 Gmail，导入并识别达人已有商单邀约。
- 做外部公开 Offer 聚合样本，先验证质量和价值；边界问题继续调研。
- 定义商单收件箱字段。
- 定义商单判断页输出格式。
- 定义报价建议字段和不可报价时的补充信息要求。
- 定义 AI 回复 / 追问 / follow-up 的关键节点确认规则。
- 定义结果记录字段。
- 定义品牌方接受报价或返回报价的统计口径。
- 跑第一轮真实商单处理验证。

第一轮结束后，再判断是否扩展到自动匹配、更开放的机会分发、半自动外联、主动品牌外联、被选中原因反馈、商业档案、履约管理、合作证明或品牌自助发布。
