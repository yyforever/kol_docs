## 1. 判断口径

第一版要验证的主痛点是：

```text
有一定内容能力、愿意接品牌合作的中小网红，
缺少可持续获得、可信、匹配、报酬合理、
能成交并能顺利完成的品牌合作来源。

即使看到合作机会，
也难判断这单是否真实、是否适合自己、钱和样品是否值得、
品牌是否可能选中、交付和付款是否可靠、做完是否有后续。
```

### 1.1 痛点标准

痛点必须写成：

```text
谁在什么场景遇到什么问题，愿意付出多少成本来解决。
```

- 什么样的网红。
- 在什么合作场景。
- 卡在机会、价格、匹配、品牌选择、样品、内容成本、付款、复购中的哪一段。
- 已经付出了什么成本。
- 是否愿意为解决问题付费、抽佣、提交资料、接受邀约、接低价试单或绑定平台。

### 1.2 证据使用纪律

- 内部项目证据很重要，但只能间接解释网红处境，不能把产品主语改成品牌、运营或业务团队。
- 品牌筛选字段说明“网红如何被选中”，不直接等于网红最痛的是“填资料”。
- 合作证明说明“下一次合作可能更容易”，不直接等于第一入口就是“证明沉淀”。
- 真实合作供给说明“我们有实践验证条件”，不直接等于应该做任务市场。
- `AI 商单经纪人 / 商单处理台` 是当前第一版产品假设，不是证据本身。它需要被真实行动、授权意愿、报价/沟通成本下降和核心结果验证。
- 公开社区材料能发现变量，不能单独证明发生率。

## 2. 当前主要矛盾排序

### 2.1 总表

| 人群 | 赚钱/目标 | 当前第一矛盾 | 第二矛盾 | 当前不应当作主因 | 证据状态 | 聚星/Nox 可介入性 |
|---|---|---|---|---|---|---|
| 有内容基础、愿意接品牌合作的中小网红 | 品牌广告、测评、样品 + 现金、佣金、授权素材 | 缺少可持续获得、可信、匹配、报酬合理、能成交并能完成的真实品牌合作来源 | 即使看到合作，也难判断是否适合自己、钱和样品是否值得、品牌是否会选、交付/付款是否可靠、做完是否有后续 | 不是单纯不会整理资料，也不是缺一个任务列表 | L3-L4 | 高：聚星有真实项目供给和达人数据，第一版应验证 AI 商单处理能否把商单推进到品牌接受报价或返回报价 |
| 品牌内容新手 / 商单内容创作者 | 第一批付费单、样片、评价、品牌背书 | 缺少第一批靠谱且钱/样品/工作量合理的合作 | 容易接低价、高工作量、授权过宽、付款不稳的单 | 不是低价平台天然坏，也不是 portfolio 页面能解决 | L4 | 高：可用真实品牌项目测试价格、样品、交付、权益对行动的影响 |
| 有作品但无稳定客户的品牌内容创作者 | 更多付费项目、复购、retainer | 缺少可持续获得、可信、匹配、报酬合理、能成交并能完成的合作来源 | 品牌难快速判断其是否适合具体 brief，且 case study / 数据回传不足 | 不是单纯不会拍，也不是单纯缺商业资料 | L4 | 中高：可验证“真实项目 + 明确筛选理由 + 报价机制”是否比泛任务有效 |
| TikTok Shop Affiliate 中愿意接品牌合作的人 | 样品、佣金、品牌补贴、付费合作 | 难判断商品/样品/佣金/品牌合作是否值得投入 | 收入受 listing、内容、退款、归因、settled commission 和平台规则影响 | 不是只缺商品列表、高佣金或 invite | L4 | 中等：可做机会质量解释和风险提示，但平台资格、流量、结算不可控 |
| 成熟中的品牌内容创作者 | 月入几千后继续增长 | 稳定客户、复购、报价、权益、付款和业务系统 | case study、pipeline、CRM、invoice、payment tracking | 不是第一版最可能切入的新手问题 | L3-L4 | 中：更像后续业务系统，不应抢第一版主线 |
| 品牌 / buyer | 低风险买到可用内容或结果 | 不知道谁能低风险完成 brief | 管理成本、交付风险、rights、付款沟通 | 不是第一版目标用户 | L3-L4 | 只作为关系方，用来解释网红为什么被选中或拒绝 |
| Agency / marketplace / creator program | 组织供给、降低交易摩擦 | 规模化供给和质量控制 | 抽成、低价、客户关系锁定、proof 不可迁移 | 不是可直接复制的产品形态 | L3-L4 | 作为替代方案和反例，不应把“已有平台很多”误读成需求已解决 |

### 2.2 当前主战场

当前更值得验证的是：

```text
哪些中小网红对“靠谱、钱合适、值得做的真实品牌合作”最敏感？
他们会为此付出什么？
时间、资料、报价、抽佣、低价试单、平台绑定，还是别的？
AI 商单经纪人是否真的比达人自己处理商单更有效？
```

这比“做不做机会列表”更基础。因为机会列表只能解决“看到机会”，不能自动解决：

- 这单是否真实。
- 钱和样品是否值得。
- 产品是否适合内容。
- 品牌是否会选中。
- 交付是否麻烦。
- 是否能付款或结算。
- 是否有后续合作。

## 3. 证据如何重新解释

### 3.1 内部业务证据

内部专家服务项目提供了最重要的实践入口，但要正确解释。

| 已确认事实 | 对网红端的含义 | 不能推出什么 |
|---|---|---|
| 内部项目都是真实品牌项目，原则上都可以给达人展示 | 我们有真实合作机会作为验证工具 | 不能直接推出任务市场成立 |
| 内部自有 Offer 数量有限 | 自有 Offer 可做高可信校准样本，但不足以证明稳定商单来源 | 不能只靠自有 Offer 验证 AI 商单经纪人可持续成立 |
| 达人拒绝合作最常见原因是价格低 | 钱是高权重变量，必须和样品、工作量、达人大小一起看 | 不能只做“合作说明页”就解决 |
| 达人最关心现金报酬和样品价值 | 机会质量必须先讲清楚钱和样品 | 不能把资料字段完整当价值 |
| 品牌最终筛达人常看均播、受众、互动率 | 这些是网红被选中的关键条件 | 不能误写成“达人主痛点是不会填数据” |
| 确认后不上线常见卡点是脚本/初稿反复修改、达人拖延、样品没到 | 合作执行成本会吞掉机会价值 | 不能只看申请率或确认率 |
| 置换和联盟项目容易找，但不是所有网红都能赚到钱 | 供给多不等于收益稳定 | 不能把低门槛机会当成解决方案 |

### 3.2 公开调研证据

公开信息仍然有价值，但主要是帮助拆变量：

| 证据方向 | 当前保留判断 | 对主痛点的贡献 |
|---|---|---|
| TikTok Shop Affiliate 社区案例 | views/clicks/product views 不等于出单；listing、价格、物流、评价、退款、结算和平台报表都会影响收入 | 说明“看到商品/样品/佣金”不等于真实收益 |
| TikTok Shop seller 案例 | seller 也承担样品、库存、退货、广告和 creator ops 成本 | 说明不能做单边“帮达人薅样品”的产品 |
| 品牌内容创作者案例 | 低价第一单可能是台阶，也可能是坑 | 说明机会质量要看钱、工作量、rights、付款和后续 |
| 低价平台与 marketplace | 有些平台能给新手训练和早期收入，也可能锁住关系、压低价格、限制 proof 迁移 | 说明“任务平台”不是天然解，也不是天然错 |
| 品牌 brief / agency 材料 | 品牌买的是具体目标下的低风险结果，不是抽象好网红 | 说明网红要被选中，必须匹配具体交易目标 |
| case study / analytics 讨论 | 关键数据常在品牌后台，creator 不一定能单方沉淀强 proof | 说明证明资产是后续变量，且依赖交易结构 |
| creator program / agency / 外部平台材料 | 市场已经存在机会分发、品牌关系、报价/付款和执行支持的替代方案 | 说明新增平台不能只做任务聚合，必须证明机会质量、匹配、报价/沟通或成交结果更好 |

## 4. 反例验证

### 4.1 反例一：资料完整不等于痛点命中

品牌确实需要判断网红：均播、受众、互动率、内容质感、报价、垂类匹配、过往合作。

但这只能证明：

```text
网红要被选中，需要让品牌看懂自己是否适合这次合作。
```

不能证明：

```text
网红最想要的是填写资料或维护商业档案。
```

如果没有值得做的合作，资料再完整也没有行动动力。

### 4.2 反例二：合作证明重要，但不是第一痛点

样片、评价、数据、联系人、复购信号确实可能让下一次合作更容易。

但当前证据不足以证明：

```text
中小网红现在最愿意付费解决的是“合作证明沉淀”。
```

更稳妥的说法是：

```text
合作证明是一次真实合作完成后的复利变量。
它是否值得产品化，要看网红是否已经有足够合作频次，且品牌是否认可这些证明。
```

### 4.3 反例三：平台机会不是中性入口

Billo、Collabstr、JoinBrands、TikTok Creator Marketplace、ShopMy、Shopify Collabs 等平台都能提供某些真实机会、流程保障或付款机制。

但平台也可能带来：

- 低价竞争。
- 抽成。
- rights 标准化。
- 客户关系锁定。
- proof 只能在平台内使用。
- 付款触发链路断裂。

所以正确问题不是“要不要做平台”，而是：

```text
什么样的机会能让网红实际赚到钱，并且不会把风险和成本全部转嫁给网红？
```

### 4.4 反例四：内部主动邀约可能比达人申请更高效

业务侧提出：作为中间平台，主动邀约达人通常更高效，为什么要让达人申请？

这说明不能把“申请”写成产品前提。第一阶段处理方式：

```text
第一阶段可以继续由运营主动邀约。
关键不是让达人自己逛市场，
而是看被邀约达人是否愿意继续了解、报价、补资料、接受合作条件。
```

所以第一轮验证不是“申请制机会列表”，而是“真实项目邀约 + 精选机会浏览后的网红行动反应”。邀请和浏览不冲突，但都必须服务于判断：什么样的机会能让什么样的达人行动。

### 4.5 反例五：自有 Offer 不足以证明稳定机会来源

聚星自有商单真实、可信、可追踪，是最好的闭环验证样本。

但它不能直接证明：

```text
达人会长期依赖一个合作机会平台。
```

原因是：

- 自有 Offer 数量有限。
- 自有 Offer 的品类和预算不一定覆盖达人长期需求。
- 如果只有自有 Offer，产品容易变成内部项目分发工具。
- 主痛点是“稳定、高质量、可推进的合作来源”，不是“偶尔看到几个真实项目”。

因此第一版必须验证其他商单来源：达人已有 Gmail 商单邀约、外部公开 Offer 平台或社区机会。只有这些来源中至少一部分能被筛选、解释并推进到品牌接受报价或返回报价，AI 商单经纪人才更可能成立。主动品牌外联保留为后续方向，不作为第一版主路径。

## 5. 当前强弱主因

### 5.1 更强的主因

| 主因 | 为什么更强 | 仍需验证 |
|---|---|---|
| 可持续获得、可信、匹配、报酬合理、能成交并能完成的品牌合作来源不足 | 符合网红端赚钱目标；内部和公开证据都显示机会质量、价格、匹配和成交/交付概率是高权重变量 | 哪类网红最痛、愿意付出什么成本 |
| 机会质量难判断 | 钱、样品、产品、品牌、交付、付款、后续都会影响是否值得做 | 哪些字段真正影响网红行动，而不是我们以为重要 |
| 品牌选择机制不透明 | 品牌实际看均播、受众、互动率、内容质感、报价和场景匹配 | 网红是否愿意按这些标准补充信息，是否能提高推进率 |
| 执行成本吞掉机会价值 | 脚本修改、样品没到、产品问题、达人拖延是确认后常见卡点 | 这些卡点是产品可解决，还是只能通过运营管理 |
| 价格与样品价值是高权重变量 | 业务侧明确价格低是拒绝高频原因，现金和样品价值最受关注 | 不同体量网红的价格/样品权重如何变化 |
| 供给存在但不等于收益稳定 | 内部品牌广告、置换和联盟供给都有，但弱播放量网红未必能赚到钱 | 哪些项目适合开放给什么级别网红 |
| 多来源商单来源是否能提高机会稳定性 | 自有 Offer 不足以支撑长期使用；公开 Offer 和 Gmail 邀约可能补充供给 | 哪种来源能产生高质量行动，而不是增加噪音 |
| 报价和沟通成本可能是可介入断点 | 真实合作中议价、权益、付款、追问经常影响推进；AI 可能降低草稿、报价和条款检查成本 | 达人是否愿意使用，是否真的减少沟通成本或提高推进率 |

### 5.2 暂不作为主线的方向

| 方向 | 当前定位 |
|---|---|
| 单纯合作机会承接 | 它是流程，不是痛点。真实合作机会用来做验证，不等于产品主线。 |
| 多来源商单来源 | 当前第一版供给机制，但仍需验证。它只有在能持续提供更靠谱、更匹配、更容易推进的待处理商单时才成立。 |
| 商业资料沉淀 | 它服务品牌判断和沟通降本，网红是否愿意维护取决于是否带来合作。 |
| 合作证明沉淀 | 它是后续复利变量，不能替代前置的靠谱合作来源和成交。 |
| 公开任务市场 | 当前需要先验证真实项目能否驱动合适网红行动。 |
| AI 自动谈判 | 第一版不预设全自动谈判；先验证 AI 判断、报价、回复和追问是否真的省成本，并在关键节点由达人确认。 |
| Portfolio / media kit 工具 | 只是信任入口，不是成交原因本身。 |
| 只做数据分析 | 数据是辅助，网红要的是可到账、值得做的合作，不是看更多指标。 |

## 6. 可介入性初判

### 6.1 聚星/Nox 可能有优势的地方

| 可介入点 | 为什么可能成立 | 需要验证 |
|---|---|---|
| 真实品牌项目供给 | 内部有品牌广告、置换、联盟和自营推广项目，可做高可信小样本验证 | 哪类网红愿意行动；但还要验证它是否足够支撑长期供给 |
| 多来源商单获取 | Gmail 邀约、外部公开 Offer 可能补足自有商单不足；主动外联后续再验证 | 第一阶段先看来源质量和价值，边界问题继续调研 |
| 机会质量解释 | 聚星知道品牌预算、产品、交付、样品、结算和筛选条件 | 网红是否信任解释，哪些信息真正影响行动 |
| 网红与项目匹配 | Nox/聚星有达人数据、受众和内容判断能力 | 能否提高品牌推进率，而不是只提高运营效率 |
| 报价与合作条件辅助 | 真实流程中价格、样品、权益、时长经常需要组合谈判 | 网红是否愿意接受平台建议，品牌是否认可 |
| AI 回复和追问辅助 | AI 可以降低起草回复、追问关键条款、报价解释的成本 | 达人是否愿意采用，是否需要人工确认发送，是否影响品牌体验 |
| 执行卡点记录 | 样品、脚本、修改、发货、付款等卡点可被结构化记录 | 是否是产品能解决，还是只能由运营项目管理 |

### 6.2 不宜承诺的地方

- 保证网红接到单。
- 保证稳定赚钱。
- 保证品牌一定回复。
- 保证 TikTok Shop 出单、结算或不退款。
- 代替品牌决定预算和选人。
- 代替达人内容能力、沟通能力和执行纪律。
- 代替复杂合同、付款和法律审查。

## 7. 下一步验证

下一步不再继续堆宽词搜索。公开信息已经足够说明变量，但不足以证明产品切入点。

要回到实践：

```text
用多来源真实合作机会，测试网红真实行动。
```

### 7.1 第一轮验证问题

| 问题 | 为什么重要 |
|---|---|
| 什么样的网红看到真实合作机会会立刻感兴趣？ | 找目标用户，不再抽象讲“中小网红” |
| 哪种商单来源最能驱动行动？ | 自有商单不足以证明 AI 商单经纪人可持续成立 |
| 他们最先问什么？ | 判断真正影响行动的变量 |
| 他们为什么拒绝或不回复？ | 找真实痛点，不靠想象 |
| 价格、样品、品牌、产品、内容成本、付款、长期合作，哪个最关键？ | 排主次矛盾 |
| 达人愿不愿意授权账号数据、导入 Gmail 或使用 AI 回复草稿？ | 判断平台关键机制是否可接受 |
| 品牌为什么选中或拒绝他们？ | 判断平台能否提高被选中概率 |
| 确认后为什么不上线？ | 判断执行问题是否是主痛点 |
| 做完是否带来复购或再次合作？ | 判断合作证明和长期复利是否值得产品化 |

### 7.2 第一轮最小实验

- 选 3-5 个内部真实品牌项目，作为高可信闭环样本。
- 至少补 1-2 类非自有机会来源样本，第一版优先 Gmail 邀约和外部公开 Offer；主动外联后续单独验证。
- 每个项目整理达人可读的合作说明。
- 由运营主动邀约，也允许被邀约达人浏览精选机会；不把完全开放市场作为前提。
- 第一阶段优先品牌内容创作者 / 商单内容创作者；其他人群单独记录，不混算平均值。
- 每个掉点必须记录原因，不只记录状态。
- 不把表单填写率当成功标准，成功标准是能否识别高痛人群和关键行动变量。

### 7.3 解剖麻雀记录表

每个项目至少记录一张完整链路表。没有原话就写 `未知`，不要脑补原因。

| 字段 | 记录要求 |
|---|---|
| 项目 | 品牌、产品、平台、合作类型 |
| 机会来源 | 聚星自有商单 / Gmail 邀约 / 外部公开 Offer / 主动外联 / 其他 |
| 项目条件 | 现金、样品、佣金、交付物、付款方式、报价机制 |
| 邀约对象 | 网红类型、阶段、平台、垂类、基础数据 |
| 达人第一反应 | 继续了解 / 报价 / 拒绝 / 沉默 / 拖延 |
| 达人追问 | 最先问什么，按原话记录 |
| 达人拒绝/沉默原因 | 价格、样品、产品不适合、内容成本、品牌不可信、档期、未知 |
| 授权/AI 接受度 | 是否愿意社媒登录、账号授权、导入邀约、使用 AI 回复/报价建议；拒绝原因是什么 |
| 品牌筛选结果 | 推进 / 拒绝 / 待定，以及具体原因 |
| 确认后卡点 | 样品、脚本、初稿、修改、达人拖延、产品问题、付款、授权 |
| 最终结果 | 上线 / 未上线 / 付款 / 退款 / 复购 / 再次合作 |
| 对主因排序影响 | 强化、削弱或新增哪一个主因 |

### 7.4 必须主动找的反例

| 反例 | 用来检验什么 |
|---|---|
| 项目真实、报酬也不错，但达人仍沉默或拒绝 | “缺靠谱合作来源”是否过粗，是否还有产品/垂类/档期/品牌吸引力问题 |
| 样品价值高，但达人不接 | 样品价值是否不是关键，或产品不适配/内容成本更关键 |
| 达人愿意接，但品牌不选 | 平台是否能提高被选中概率，还是品牌筛选不可介入 |
| 达人确认后不上线 | 主痛点是否在执行和履约，而不是机会本身 |
| 低价项目顺利完成并复购 | 低价不是天然坏，需看工作量、权益、付款和后续 |
| 现有平台/agency 已经满足需求 | 聚星/Nox 是否缺差异化 |

反例必须写进主因排序。不能把反例解释成特殊情况后继续保持原判断。
