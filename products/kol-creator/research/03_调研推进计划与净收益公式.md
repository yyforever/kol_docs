## 1. 当前研究方法

本轮采用“从现实问题出发”的调研方法：

```text
根本问题
-> 真实现场
-> 人群与利益关系
-> 赚钱链路
-> 链路断点
-> 主因判别
-> 可介入性判断
-> 实践验证动作
```

这意味着：

- 不从竞品功能出发。
- 不从聚星现有能力出发。
- 不从“任务市场 / 机会列表 / Agent”这些产品形态出发。
- 不把材料按表面标签分类，而按赚钱机制和链路断点分类。
- 不把交易流程变量当成主痛点。`资料提交`、`合作证明`、`机会页面`、`表单` 都只能作为机制变量，不能先写成产品主线。
- 不把“商业合作机会管道平台”当作第一版产品定位。当前第一版产品假设是 `AI 商单经纪人 / 商单处理台`；多来源机会只是商单来源机制，需要继续验证供给来源、机会质量、达人授权意愿、报价/沟通辅助和结果改善。
- 不把品牌侧或运营侧证据直接翻译成网红端需求。所有结论必须回到“什么样的网红，在什么场景，遇到什么问题，已经愿意付出什么成本”。

术语边界：

- 本组文档里的 `UGC creator` 是英文社区里的职业/交易标签，通常指为品牌制作可发布、可投放、可授权使用内容的创作者。
- 它不按泛义“用户生成内容”理解，也不等同于所有达人。
- 中文正文优先写“品牌内容创作者 / 商单内容创作者”，只在引用社区术语、案例编号或市场分类时保留 `UGC`。

## 2. 长期收入公式

当前总公式是“痛点判别地图”：

```text
达人长期收入 =
    可持续获得且值得投入的合作机会数量与质量
  × 机会与达人匹配度
  × 被选择 / 成交 / 转化概率
  × 单次真实收益(现金 + 样品 + 佣金 + 授权收益)
  × 到账概率
  × 后续合作概率
  - 创作成本
  - 沟通成本
  - 风险成本
  - 收入波动成本

后续放大项 =
    可复用合作证据
  + 品牌联系人 / 复购关系
  + 报价依据 / 交易经验
```

这个公式不是数学模型，而是变量地图。它的用途是帮助我们判断：

- 哪个变量是某类达人当前阶段的门槛。
- 哪个变量只是表层症状。
- 哪个变量可以被产品、运营、服务或供给网络改善。
- 哪个变量只能被平台、品牌或达人自身能力解决。

注意：`可复用合作证据` 不是当前主痛点。它是合作完成后的放大项。只有当网红已经拿到值得做、能完成、能到账的合作后，它才可能变成重要产品机制。

### 2.1 公式使用限制

要特别注意四类变量：

| 类型 | 含义 | 例子 |
|---|---|---|
| 门槛变量 | 低于阈值时，其他变量再高也没用 | 没样品资格、佣金被冻结、付款机制不可信、rights 过宽 |
| 分段变量 | 同一变量在不同阶段权重不同 | 新手先卡第一单，成熟 creator 更卡复购/retainer |
| 耦合变量 | 多个变量互相影响，不能拆开看 | 商品匹配、内容转化、listing 可信度、物流和价格 |
| 成本转移变量 | 方案不是消灭成本，而是把成本转移 | marketplace 降低追款风险，但引入抽成、申请成本和低价竞争 |

## 3. 两条主公式

### 3.1 TikTok Shop Affiliate

```text
长期收入 =
    可申请机会数量(PPS/样品资格/历史表现/product eligibility)
  × 商品-受众-内容-listing 匹配度
  × 内容转化概率(曝光 -> 点击 -> 购买)
  × 显示佣金/补贴/样品价值
  × 到账概率(订单确认 × 非退款 × 非违规 × 正确归因 × 数据正常)
  - 样品申请/垫付与选品成本
  - 内容试错/直播/真实使用成本
  - 规则理解/申诉/客服沟通成本
  - 平台规则/类目退款/seller行为/内容真实性/样品现金流风险
```

当前判断：

- H1“缺机会入口”在 TikTok Shop Affiliate 场景被削弱。
- H2“机会选择和匹配不确定”增强。
- H4“到账和风险保护不足”增强。
- H6“内容执行和反馈回路不足”增强，但不能简单等同于“给脚本模板”。

### 3.2 品牌内容创作者 / 商单内容创作者

```text
长期收入 =
    可接触且值得投入的付费品牌项目数量与质量
  × 品牌/brief 与内容能力匹配度
  × 被品牌选择 / 成交概率
  × 真实单次收益(base fee + rights fee + exclusivity fee + whitelisting/ad usage fee + 样品价值)
  × 复购/retainer/长期合作概率
  × 付款到账概率
  - 内容制作与 scope creep 成本
  - 报价/谈判/合同/追款沟通成本
  - 低价买断/scam/usage rights 滥用/付款失败风险
  - 项目断档导致的收入波动成本

后续放大项 =
    样片 / case study / 评价 / 品牌背书
  + 联系人 / 复购线索
  + 报价依据 / 交易边界经验
```

当前判断：

- H1“缺项目来源”保留，但必须改写为“缺可持续获得、可信、匹配、报酬合理、能成交并能完成的合作来源”，不能单独写成“缺项目”。
- H3“商业谈判和价格捕获能力弱”增强。
- H4“付款和风险保护不足”增强。
- H5“持续关系和复利不足”保留为后续放大项，需要通过真实合作完成后的复购/数据回传补证。
- H6“内容执行不足”暂不作为主判断，品牌侧筛选标准需要先补证。

## 4. 候选假设与反证路径

| 假设 | 解释 | 支持证据会长什么样 | 反证会长什么样 |
|---|---|---|---|
| H1 值得做的品牌合作来源不足 | 目标网红可持续获得的真实合作不够稳定、可信、匹配或报酬合理，且未必能成交和完成 | 反复表达没靠谱项目、品牌不回复、机会价格低或不值得做 | 有大量好机会但不接，问题集中在执行、付款或谈判 |
| H2 机会选择和匹配不确定 | 机会存在，但不知道哪些适合自己、能转化、值得做 | views/clicks no sales、商品/listing/受众/内容不匹配 | 达人清楚知道适合什么，但主要卡在执行、付款或谈判 |
| H3 商业谈判和价格捕获能力弱 | 有合作但不会报价、不会谈 rights、不会保护价值 | usage rights、whitelisting、exclusivity、raw files、retainer 困惑 | 达人报价成熟，主要卡在没有项目 |
| H4 履约、付款和风险保护不足 | 做了可能拿不到钱、被滥用内容、佣金冻结或归因失败 | 不付款、佣金冻结、scope creep、scam、退款、样品返款争议 | 风险低频，达人更关心机会、成交或转化 |
| H5 持续关系和复利不足 | 合作完成后不能带来复购、联系人、数据或可展示证明 | repeat client、retainer、评价、样片、品牌联系人反复出现 | 主要卡在首次靠谱合作，暂时不关心长期关系 |
| H6 内容执行和反馈回路不足 | 有机会但不知道如何产出能成交/转化/让品牌满意的内容 | 高频 no sales、内容试错、脚本/复盘需求 | 转化失败主要由商品、价格、物流、平台或品牌筛选导致 |

## 5. 证据分级

| 成熟度 | 定义 | 能做什么 | 不能做什么 |
|---|---|---|---|
| L1 现象信号 | 单个用户抱怨、帖子、评论 | 发现变量，生成问题 | 不能证明高频，不能决定方向 |
| L2 机制确认 | 官方政策、平台规则、竞品流程 | 证明变量真实存在 | 不能证明用户最痛，也不能证明外部产品可解 |
| L3 多源重复 | 多个社区、时间、角色出现类似问题 | 支持阶段性判断 | 不能直接输出产品方案 |
| L4 主因判别 | 能区分主要原因和替代原因 | 指导下一轮验证重点 | 还不能直接定功能 |
| L5 决策级证据 | 高痛、高频、可介入、替代不足 | 支持第一阶段切入点 | 仍需产品验证 |

## 6. 证据解释纪律

每条证据必须写清楚：

- 支持什么。
- 不能证明什么。
- 还有哪些替代解释。

例子：

- `views no sales` 支持 H2/H6，但不能直接证明达人需要内容教学；也可能是商品页不可信、物流慢、价格高、商品饱和、受众无购买意图或平台数据延迟。
- `usage rights 困惑` 支持 H3/H4，但不能直接证明达人需要 marketplace；也可能需要合同模板、报价教育、经纪服务或托管付款。
- `marketplace 有项目` 支持项目入口可平台化，但不能证明 creator 净收益更高；可能只是把没项目换成低价竞争、申请成本、抽成和评价冷启动。
- `低价任务多` 不能直接证明低价任务没价值；要看工作量、授权、付款、评价和后续机会。

## 7. 决策前必须回答的 5 个问题

任一候选切入点进入产品讨论前，必须回答：

| 问题 | 判断标准 |
|---|---|
| 高痛吗 | 是否直接影响长期收入，而不是只是操作烦。 |
| 高频吗 | 是否目标阶段经常遇到，而不是少数极端案例。 |
| 可介入吗 | 聚星/Nox 能否通过供给、数据、流程、运营、信任背书或工具真实改善。 |
| 替代不足吗 | TikTok Shop、Billo、Insense、Collabstr、ShopMy、LTK、Passionfroot、agency/manager 是否已经解决得足够好。 |
| 有差异化吗 | 聚星已有资产是否能形成优势，而不是复制已有工具。 |

如果一个问题只满足“高痛”，但不满足“高频 / 可介入 / 替代不足”，只能作为风险提示或辅助功能。

## 8. 当前证据偏差

- Reddit / 评论容易放大高情绪、高损失和极端案例，适合发现问题，不适合判断发生率。
- 官方文档容易让人高估平台机制的可理解性；机制存在不等于 creator 能掌握。
- 竞品官网容易让人高估某类产品形态有效；竞品存在不等于需求被正确满足。
- NoxInfluencer 样本只能证明人群和内容语言可观察，不能证明痛点。
- NoxInfluencer CLI 2026-05-13 复查出现 HTTP 403 / Cloudflare 网络路径问题；2026-05-14 配置代理后 creator search/profile/content 已可用，但仍只作为样本观察，不作为痛点证据。
- 行业报告容易把 TikTok Shop Affiliate、品牌内容创作者、micro influencer、agency 混成一个 creator economy。

反例和边界条件仍然重要，但下一步不再靠泛搜堆材料，而是通过结构化样本表补证。

#### A. 解剖完整过程案例

| 项目条件 | 钱、样品、产品、平台、交付物、付款方式、报价机制 |
| 邀约方式 | 主动邀约 / 平台申请 / inbound / 其他 |
| 达人第一反应 | 继续了解 / 报价 / 拒绝 / 沉默 / 拖延 |
| 达人追问 | 钱、样品、产品、品牌、交付、付款、长期合作、其他 |
| 达人拒绝或沉默原因 | 原话优先；没有原话时只能标“未知”，不能脑补 |
| 品牌筛选原因 | 选中或拒绝的具体原因 |
| 确认后卡点 | 样品、脚本、初稿、修改、达人拖延、产品问题、付款、授权 |
| 结果 | 上线 / 未上线 / 付款 / 退款 / 复购 / 再次合作 |
| 对主因的影响 | 支持、削弱或改变哪一个主因判断 |

#### B. 做内部真实合作验证

内部真实合作机会不直接等于产品方案。它现在的用途是提供高可信样本，验证：

- 什么样的网红愿意继续了解。
- 达人行动前最想问什么。
- 达人为什么拒绝。
- 品牌/seller 实际按什么字段筛人。
- 哪些信息能影响网红行动，而不是只是字段看起来完整。
- 价格、样品、内容成本、付款、产品适配、长期合作可能性是否会影响行动。
- rights、数据回传、可展示 proof 这些条件是否会影响后续价值，但不把它们预设为第一主因。

建议第一轮内部样本只做小规模：

- `3-5` 个真实合作项目。
- 第一阶段优先品牌内容创作者 / 商单内容创作者。
- 由运营主动邀约合适网红，也允许被邀约达人浏览精选机会；不把开放市场作为前提。
- 每个项目只准备达人判断是否继续需要的信息：钱、样品、品牌、产品、交付、付款、报价机制和后续可能。
- 记录打开/回复/继续了解/报价/补资料/品牌推进/确认/上线/付款的完整漏斗。
- 记录追问字段、拒绝原因、品牌筛选字段、确认后不上线原因。

第一轮验证的最低样本要求：

- 至少 `3` 个项目，最好 `5` 个。
- 每个项目至少邀约 `10` 个匹配达人，除非该垂类本身样本极少。
- 每个项目至少记录 `3` 个有明确行动或明确拒绝原因的达人样本。
- 至少保留 `3` 个反例，不论它们是否支持当前假设。
- 不允许只用汇总率做结论，必须保留具体案例。

#### C. 验证多来源商单来源

只靠内部自有 Offer 不足以证明“稳定合作机会来源”成立。它能证明聚星有高可信供给，但不能证明达人可持续使用。因此第一版必须单独验证另外两类商单来源：

| 机会来源 | 要验证什么 | 不能直接推出什么 |
|---|---|---|
| 达人已有 Gmail 商单邀约 | 平台能否帮达人判断真假、价值、报价、条款和回复策略 | 不能证明所有达人愿意授权 Gmail，也不能自动扩展到私信 |
| 外部公开 Offer 平台/社区 | 平台能否筛掉低质机会，把公开机会翻译成达人可判断的信息 | 第一阶段先看质量和价值；不能复制或抓取明显高风险来源，不能把低质任务堆成市场 |
| 聚星自有商单 | 能否作为高可信样本校准机会质量和成交链路 | 不能单独支撑规模和持续性 |
| 主动品牌外联 | 后续方向，验证平台能否替达人找到有购买意图的品牌，并把沟通推进到报价互动 | 不作为第一版主路径，不能证明 AI 群发外联有效，也不能忽略邮件声誉和品牌体验 |

这一阶段要验证的问题：

- 哪种来源最能让品牌内容创作者产生行动。
- 达人是否愿意授权 Gmail、粘贴邀约、社媒登录或授权账号后台数据。
- 达人是否愿意让 AI 帮忙判断 Offer、起草回复、生成报价或追问条款。
- 多来源机会是否提高“值得继续”的机会比例，而不是只增加噪音。
- 哪些来源能走到品牌接受报价或返回报价；确认、上线和付款作为辅助观察。

#### D. 拆替代方案为什么仍不够

市场里已有平台、agency、creator program、TikTok Shop network、affiliate tool 和 creator marketplace。下一步不是证明“有人做”，而是拆：

- 它解决了什么。
- 它把什么成本转移给 creator、brand、seller 或平台。
- 它是否让 creator 拿到站外可迁移 proof。
- 它是否形成长期关系、复购或业务系统。
- 它是否只是平台内评分、平台内任务和平台内流量。

只有明确替代方案失效点，才可能判断聚星/Nox 是否有差异化。

### 10.3 当前停止条件

公开搜索继续推进的停止条件：

- 连续 3 轮只能找到同类抱怨、平台营销稿、工具榜单或招募帖。
- 新材料不能改变 [11](11_主要矛盾排序与反例验证.md) 的主因排序。
- 新材料不能补全 [09](09_典型案例档案.md) 中缺失的完整过程字段。
- 新材料不能在 [12](12_结构化样本表.md) 中新增变量、反例或替代方案失效机制。

满足以上条件时，不再写搜索备忘，只把公开调研结论带入 AI 商单经纪人假设验证。

AI 商单经纪人假设验证的阶段停止条件：

- 已有至少 `3` 个完整链路案例，且能覆盖不同卡点。
- 已记录至少 `10` 个达人行动样本，包括继续了解、拒绝、沉默、报价、确认后掉队中的至少三类。
- 已出现至少 `3` 个反例，并明确它们如何影响主因排序。
- 已至少验证 `2` 类机会来源，其中一类必须不是聚星自有商单。
- 已记录达人对社媒登录、账号数据授权、Gmail 导入、AI 回复/报价辅助中的至少两类机制的真实接受或拒绝原因。
- 仍无法判断主因时，不输出产品方案，只输出下一轮要补的具体样本。
- 只有当某一类网红在某类真实合作上表现出明确高痛、高频、可介入、替代不足，才进入产品形态讨论。

### 10.4 结论更新规则

后续每次新增真实合作样本后，只允许三种更新：

| 更新类型 | 触发条件 |
|---|---|
| 强化现有主因 | 新案例重复显示同一类网红因同一主因行动、拒绝或掉队。 |
| 调整现有主因 | 反例显示该主因低频、低痛、不可介入，或只是后置变量。 |
| 新增主因 | 新案例出现旧框架解释不了的高痛、高频、可介入问题。 |

禁止的更新方式：

- 只因为一个工具形态容易做，就把它写成产品方向。
- 只因为品牌或运营需要某字段，就把它写成网红痛点。
- 只因为一个达人说“感兴趣”，就判断需求成立。
- 只因为一类反例不好解释，就把它归为特殊情况。

### 10.5 当前最值得验证的问题

| 问题 | 为什么重要 | 验证方式 |
|---|---|---|
| 哪类中小网红最缺能成交并值得做的靠谱合作 | 决定第一版目标用户 | 真实合作邀约后的行动和拒绝记录 |
| 哪种商单来源最可能形成持续高质量待处理商单 | 决定产品是否能超越内部 Offer 承接 | 自有商单、Gmail 邀约、外部公开 Offer 的对比记录；主动外联后续单独验证 |
| 品牌为什么选或不选 creator | 决定是否能帮达人“被选中” | 品牌筛选字段 + 真实合作推进/拒绝记录 |
| 达人是否愿意授权数据或导入邀约 | 决定账号连接、Gmail 导入是否可行 | 社媒登录、后台授权、Gmail 导入的接受和拒绝原因 |
| AI 是否能降低报价和沟通成本 | 决定 Agent 是否只是炫技，还是命中真实成本 | Offer 判断、报价建议、回复草稿、追问条款的使用记录 |
| 平台/agency/creator program 是否让 proof 可迁移 | 决定现有替代方案是否已经足够 | 平台后验复盘和条款拆解 |
| 聚星/Nox 能否提供比列表更有价值的真实商单解释 | 决定可介入性 | 用多来源商单观察达人和品牌是否认可解释字段 |

## 11. 当前暂停项

暂不推进：

- 任务市场方案。
- 商业化方案。

允许推进但必须标注为假设验证材料：

- 多来源机会供给方案。
- 账号登录 / 数据授权方案。
- AI 报价 / 回复辅助方案。
- 机会标准化说明和结果记录方案。

这些材料只能服务验证，不能反过来当作证据。
