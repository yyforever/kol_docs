# 调研推进计划与长期收入公式

> 创建时间：2026-05-11
> 重构时间：2026-05-13
> 状态：执行中
> 项目：`kol-creator / 聚星-网红经纪人`
> 目的：用可迭代公式、证据分级和主因判别组织后续调研，避免过早收敛成产品方案。

---

## 1. 当前研究方法

本轮采用“从现实问题出发”的调研方法：

```text
根本问题
-> 真实现场
-> 人群与利益关系
-> 赚钱链路
-> 链路断点
-> 主因判别
-> 可介入性判断
-> 实践验证动作
```

这意味着：

- 不从竞品功能出发。
- 不从聚星现有能力出发。
- 不从“任务市场 / Offer Board / Agent”这些产品形态出发。
- 不把材料按表面标签分类，而按赚钱机制和链路断点分类。

## 2. 长期收入公式

当前总公式：

```text
达人长期收入 =
    稳定可获得的靠谱机会数量
  × 机会与达人匹配度
  × 被选择 / 成交 / 转化概率
  × 单次真实收益
  × 可复用合作证据
  × 复购 / 长期合作概率
  × 到账概率
  - 创作成本
  - 沟通成本
  - 风险成本
  - 收入波动成本
```

这个公式不是数学模型，而是变量地图。它的用途是帮助我们判断：

- 哪个变量是某类达人当前阶段的门槛。
- 哪个变量只是表层症状。
- 哪个变量可以被产品、运营、服务或供给网络改善。
- 哪个变量只能被平台、品牌或达人自身能力解决。

### 2.1 公式使用限制

要特别注意四类变量：

| 类型 | 含义 | 例子 |
|---|---|---|
| 门槛变量 | 低于阈值时，其他变量再高也没用 | 没样品资格、佣金被冻结、付款机制不可信、rights 过宽 |
| 分段变量 | 同一变量在不同阶段权重不同 | 新手先卡第一单，成熟 creator 更卡复购/retainer |
| 耦合变量 | 多个变量互相影响，不能拆开看 | 商品匹配、内容转化、listing 可信度、物流和价格 |
| 成本转移变量 | 方案不是消灭成本，而是把成本转移 | marketplace 降低追款风险，但引入抽成、申请成本和低价竞争 |

## 3. 两条主公式

### 3.1 TikTok Shop Affiliate

```text
长期收入 =
    可申请机会数量(PPS/样品资格/历史表现/product eligibility)
  × 商品-受众-内容-listing 匹配度
  × 内容转化概率(曝光 -> 点击 -> 购买)
  × 显示佣金/补贴/样品价值
  × 到账概率(订单确认 × 非退款 × 非违规 × 正确归因 × 数据正常)
  - 样品申请/垫付与选品成本
  - 内容试错/直播/真实使用成本
  - 规则理解/申诉/客服沟通成本
  - 平台规则/类目退款/seller行为/内容真实性/样品现金流风险
```

当前判断：

- H1“缺机会入口”在 TikTok Shop Affiliate 场景被削弱。
- H2“机会选择和匹配不确定”增强。
- H4“到账和风险保护不足”增强。
- H6“内容执行和反馈回路不足”增强，但不能简单等同于“给脚本模板”。

### 3.2 品牌内容创作者 / 商单内容创作者

```text
长期收入 =
    可接触的付费品牌项目数量与质量
  × 品牌/brief 与内容能力匹配度
  × 被品牌选择 / 成交概率
  × 真实单次收益(base fee + rights fee + exclusivity fee + whitelisting/ad usage fee + 样品价值)
  × 可复用合作证据(样片/评价/品牌背书/联系人/报价依据)
  × 复购/retainer/长期合作概率
  × 付款到账概率
  - 内容制作与 scope creep 成本
  - 报价/谈判/合同/追款沟通成本
  - 低价买断/scam/usage rights 滥用/付款失败风险
  - 项目断档导致的收入波动成本
```

当前判断：

- H1“缺项目来源”保留，但不能单独成立。
- H3“商业谈判和价格捕获能力弱”增强。
- H4“付款和风险保护不足”增强。
- H5“持续关系和复利不足”保留，需要补证。
- H6“内容执行不足”暂不作为主判断，品牌侧筛选标准需要先补证。

## 4. 候选假设与反证路径

| 假设 | 解释 | 支持证据会长什么样 | 反证会长什么样 |
|---|---|---|---|
| H1 商业需求接入不足 | 达人接触不到足够多真实、合适、可付费需求 | 反复表达没项目、没品牌回复、找不到真实付费机会 | 有大量机会但不接，问题集中在价格、条款、转化、付款或工作量 |
| H2 机会选择和匹配不确定 | 机会存在，但不知道哪些适合自己、能转化、值得做 | views/clicks no sales、商品/listing/受众/内容不匹配 | 达人清楚知道适合什么，但主要卡在执行、付款或谈判 |
| H3 商业谈判和价格捕获能力弱 | 有合作但不会报价、不会谈 rights、不会保护价值 | usage rights、whitelisting、exclusivity、raw files、retainer 困惑 | 达人报价成熟，主要卡在没有项目 |
| H4 履约、付款和风险保护不足 | 做了可能拿不到钱、被滥用内容、佣金冻结或归因失败 | 不付款、佣金冻结、scope creep、scam、退款、样品返款争议 | 风险低频，达人更关心机会、成交或转化 |
| H5 持续关系和复利不足 | 每次合作都从零开始，不能留下复用资产 | repeat client、retainer、评价、样片、品牌联系人反复出现 | 主要卡在首次成交，不关心长期关系 |
| H6 内容执行和反馈回路不足 | 有机会但不知道如何产出能成交/转化/让品牌满意的内容 | 高频 no sales、内容试错、脚本/复盘需求 | 转化失败主要由商品、价格、物流、平台或品牌筛选导致 |

## 5. 证据分级

| 成熟度 | 定义 | 能做什么 | 不能做什么 |
|---|---|---|---|
| L1 现象信号 | 单个用户抱怨、帖子、评论 | 发现变量，生成问题 | 不能证明高频，不能决定方向 |
| L2 机制确认 | 官方政策、平台规则、竞品流程 | 证明变量真实存在 | 不能证明用户最痛，也不能证明外部产品可解 |
| L3 多源重复 | 多个社区、时间、角色出现类似问题 | 支持阶段性判断 | 不能直接输出产品方案 |
| L4 主因判别 | 能区分主要原因和替代原因 | 指导下一轮验证重点 | 还不能直接定功能 |
| L5 决策级证据 | 高痛、高频、可介入、替代不足 | 支持第一阶段切入点 | 仍需产品验证 |

### 5.1 当前证据成熟度

| 方向 | 当前成熟度 | 说明 |
|---|---|---|
| TikTok Shop Affiliate: views/clicks no sales | L3-L4 | 已有多源重复，开始拆主因，但缺对照样本。 |
| TikTok Shop Affiliate: 佣金/退款/归因/样品风险 | L2-L3 | 机制和案例都有，但发生率与持续投入影响仍不清楚。 |
| TikTok Shop Affiliate: invite 噪音 | L3 | creator 和 seller 两侧都有信号，需要更多 creator 拒绝理由。 |
| 品牌内容创作者: rights/pricing/payment | L3-L4 | 多源重复且机制清楚，是当前最强证据区。 |
| 品牌内容创作者: 第一单/低价平台 | L3 | 已有大量线索，但需要按“留下什么证据”重采样。 |
| 品牌内容创作者: 品牌筛选标准 | L2-L3 | 有品牌侧信号，但还不够系统。 |
| 复购/retainer/长期关系 | L2-L3 | 方向重要，但证据不足。 |

## 6. 证据解释纪律

每条证据必须写清楚：

- 支持什么。
- 不能证明什么。
- 还有哪些替代解释。

例子：

- `views no sales` 支持 H2/H6，但不能直接证明达人需要内容教学；也可能是商品页不可信、物流慢、价格高、商品饱和、受众无购买意图或平台数据延迟。
- `usage rights 困惑` 支持 H3/H4，但不能直接证明达人需要 marketplace；也可能需要合同模板、报价教育、经纪服务或托管付款。
- `marketplace 有项目` 支持项目入口可平台化，但不能证明 creator 净收益更高；可能只是把没项目换成低价竞争、申请成本、抽成和评价冷启动。
- `低价任务多` 不能直接证明低价任务没价值；要看工作量、授权、付款、评价和后续机会。

## 7. 决策前必须回答的 5 个问题

任一候选切入点进入产品讨论前，必须回答：

| 问题 | 判断标准 |
|---|---|
| 高痛吗 | 是否直接影响长期收入，而不是只是操作烦。 |
| 高频吗 | 是否目标阶段经常遇到，而不是少数极端案例。 |
| 可介入吗 | 聚星/Nox 能否通过供给、数据、流程、运营、信任背书或工具真实改善。 |
| 替代不足吗 | TikTok Shop、Billo、Insense、Collabstr、ShopMy、LTK、Passionfroot、agency/manager 是否已经解决得足够好。 |
| 有差异化吗 | 聚星已有资产是否能形成优势，而不是复制已有工具。 |

如果一个问题只满足“高痛”，但不满足“高频 / 可介入 / 替代不足”，只能作为风险提示或辅助功能。

## 8. 当前证据偏差

- Reddit / 评论容易放大高情绪、高损失和极端案例，适合发现问题，不适合判断发生率。
- 官方文档容易让人高估平台机制的可理解性；机制存在不等于 creator 能掌握。
- 竞品官网容易让人高估某类产品形态有效；竞品存在不等于需求被正确满足。
- NoxInfluencer 样本只能证明人群和内容语言可观察，不能证明痛点。
- 行业报告容易把 TikTok Shop Affiliate、品牌内容创作者、micro influencer、agency 混成一个 creator economy。

下一轮要主动找反例和边界条件。

## 10. 下一轮调研计划

### 10.1 优先方向 A：品牌到底怎么筛达人

目标：补品牌侧证据，防止只听达人侧抱怨。

问题：

- 品牌为什么选某个 creator。
- 品牌为什么不回复 pitch。
- 品牌看样片时判断什么。
- 交付可靠性、沟通速度、授权清楚度和价格分别多重要。

采样标准：

- 10-15 条品牌/agency 侧高上下文样本。
- 15-25 条真实任务 brief / casting call。
- 8-12 条 creator pitch 成功/失败复盘。

停止条件：

- 连续 3 轮只重复“品牌要可靠/要作品集”，但没有更具体筛选信号时停止。

### 10.2 优先方向 B：第一单到底留下什么

目标：判断第一单是训练场还是消耗。

问题：

- 第一单来源是什么。
- 价格、交付物、授权、付款如何。
- 是否留下样片、评价、品牌背书、联系人、复购或报价依据。
- 低价任务什么条件下值得做。

采样标准：

- 15-25 条新手 first client 案例。
- 10-15 条低价任务正反例。
- 8-12 条从一次合作转复购/retainer 的案例。

停止条件：

- 新增案例只说“拿到第一单”，但不说明后续证据时停止同类搜索。

### 10.3 优先方向 C：TikTok Shop Affiliate 主因判别

目标：拆解 no sales / no payout / no acceptance。

问题：

- 高播放不出单的主因是什么。
- creator 为什么拒绝 invite/sample。
- 商品机会在哪个阶段失效。
- payout / refund / attribution 风险是否影响继续投入。

采样标准：

- 有商品页、价格、评价、物流、视频角度、CTA 的 no-sales 案例。
- creator 主动拒绝 invite/sample 的一手理由。
- 中小 creator 出单后遇到冻结、退款、归因异常的案例。

停止条件：

- 连续 3 次搜索只重复“views no sales”但不能拆新主因时停止同类搜索。

### 10.4 优先方向 D：四象限验证

| 象限 | 需要补的证据 |
|---|---|
| 新手 TikTok Shop Affiliate | 首单、样品、选品、内容转化 |
| 成熟 TikTok Shop Affiliate | 归因、退款、冻结、稳定商品供给 |
| 新手品牌内容创作者 | 被品牌选中、样片可信度、低价训练场 |
| 成熟品牌内容创作者 | 复购、retainer、授权定价、轻量工作流 |
