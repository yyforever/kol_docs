# 企业微信机器人

本文档说明如何把企业微信机器人接入 NoxClaw。接入完成后，企业微信成员可以通过私聊机器人或在群聊中 @ 机器人发送消息，NoxClaw 会把消息同步到当前席位的渠道会话中，并按后端确认的会话目标进行回复。

当前 NoxClaw 企业微信渠道使用 **WebSocket 长连接** 模式。你只需要在 NoxClaw 中填写企业微信机器人的 **Bot ID** 和 **Secret**；`URL`、`Token` 和 `Encoding-AESKey` 属于 URL 回调模式信息，当前 NoxClaw 配置弹窗不需要填写。

## 配置前准备

开始前，请确认你已经具备以下条件：

- 可以登录 NoxClaw，并且当前账号有使用渠道配置的权限。
- 可以登录企业微信，并且有创建或管理智能机器人的权限。
- 已准备一个用于接入 NoxClaw 的企业微信智能机器人。建议一个企业微信机器人只绑定一个 NoxClaw 席位，避免多个席位共享同一个机器人导致消息归属混乱。
- 可以在企业微信管理后台或企业微信客户端中创建 API 模式机器人，并选择「使用长连接」。

Secret 属于敏感凭据，请只在 NoxClaw 配置弹窗中使用，不要通过聊天、截图或公开文档泄露。

## 一、创建企业微信机器人

企业微信支持管理员和普通成员创建智能机器人。两种方式最终都需要进入 API 模式并选择长连接。

### 管理员创建

1. 打开企业微信管理后台。
2. 使用管理员账号登录。
3. 进入「安全与管理」或「管理工具」下的「智能机器人」入口。
4. 点击「创建机器人」。
5. 如果页面先进入 AI 自动生成流程，请选择「手动创建」。
6. 在创建方式中选择「API 模式创建」。

> 图片占位：企业微信管理后台创建智能机器人页面。

### 普通成员创建

1. 打开企业微信客户端。
2. 进入「工作台」。
3. 找到「智能机器人应用」并点击「创建机器人」。
4. 如果页面先进入 AI 自动生成流程，请选择「手动创建」。
5. 在创建方式中选择「API 模式创建」。

> 图片占位：企业微信客户端创建智能机器人页面。

## 二、填写机器人基础信息

进入 API 模式创建页面后，先完成基础信息：

| 配置项 | 说明 |
| --- | --- |
| 机器人名称 | 建议填写容易识别的名称，例如 `NoxClaw Assistant`。 |
| 机器人头像 | 按企业内部规范上传头像，便于成员在会话中识别。 |
| 可见范围 / 可使用成员 | 选择允许使用该机器人的成员、部门或标签。 |

配置可见范围时，请至少包含需要通过企业微信使用 NoxClaw 的成员。保存基础信息后，再进入 API 配置区域。

> 图片占位：企业微信机器人基础信息和可见范围页面。

## 三、选择长连接并获取凭证

在「API 配置」区域中选择「使用长连接」。

1. 确认接入方式为「使用长连接」。
2. 复制 **Bot ID**。
3. 点击「点击获取」或页面中的获取入口，复制 **Secret**。
4. 妥善保存 Bot ID 和 Secret，后续需要填写到 NoxClaw。

长连接模式不需要填写 URL、Token 或 Encoding-AESKey。如果页面中同时提供 URL 回调模式，请不要切换到 URL 回调模式。

> 图片占位：企业微信 API 配置中选择使用长连接并获取 Bot ID / Secret 页面。

## 四、在 NoxClaw 中配置企业微信

1. 打开 NoxClaw，进入「渠道配置」。
2. 找到「企业微信机器人」卡片，点击「配置」。
3. 确认接入方式显示为「WebSocket 长连接」。
4. 填写从企业微信复制的 **Bot ID** 和 **Secret**。
5. 点击「注册」。

提交后，NoxClaw 会把企业微信机器人配置同步到当前席位的运行态。配置同步过程中，渠道状态会显示为「绑定中」；同步成功后，渠道会进入「已连接」。

> 图片占位：NoxClaw 企业微信机器人配置弹窗。

## 五、开始使用

1. 在企业微信中找到刚创建的机器人。
2. 点击「发消息」进入机器人私聊，或把机器人加入需要协作的群聊。
3. 直接给机器人发送一条测试消息，或在群聊中 @ 机器人发送消息。
4. 回到 NoxClaw 的渠道会话页面，等待消息同步后查看企业微信渠道会话。
5. 在 NoxClaw 中回复该会话。对于企业微信群聊消息，如果系统能识别发送人，NoxClaw 会把站内回复投递到该发送人的私聊中，而不是直接回到群聊。

企业微信机器人绑定成功只代表渠道账号已经可用。首次外部消息到达之前，NoxClaw 可能显示「已连接但暂无会话」，这是正常状态。

> 图片占位：企业微信中找到机器人并发送测试消息。

## 六、重新绑定和解绑

- 如果企业微信渠道进入异常状态，可以点击「重新绑定」，重新填写 Bot ID 和 Secret 后提交。NoxClaw 会在同一次重新绑定流程中清理失败生命周期并重新同步配置。
- 如果企业微信渠道已经连接或正在绑定中，需要先点击「解绑」后再重新配置。
- 解绑后，NoxClaw 会移除当前席位与该企业微信机器人的绑定关系。后续消息不再进入当前席位的企业微信渠道会话。

## 七、关于 URL 回调模式

企业微信 API 模式还支持 URL 回调接入。该模式通常需要配置 `URL`、`Token` 和 `Encoding-AESKey`，并把生成的 Webhook 地址回填到企业微信。

当前 NoxClaw 企业微信配置弹窗只支持 WebSocket 长连接模式，不展示 URL 回调模式，也不要求填写 Token 或 Encoding-AESKey。请优先按本文档的长连接方式接入。

## 八、常见问题

### 1. NoxClaw 需要填写 Token 或 Encoding-AESKey 吗？

当前不需要。NoxClaw 企业微信配置弹窗只要求填写 Bot ID 和 Secret，并固定使用 WebSocket 长连接模式。Token 和 Encoding-AESKey 是 URL 回调模式下常见的事件校验信息，当前不在 NoxClaw 弹窗中配置。

### 2. 注册提示 Bot ID 或 Secret 无效怎么办？

请回到企业微信智能机器人的 API 配置区域，确认已选择「使用长连接」，然后重新复制 Bot ID 和 Secret。确认没有多余空格，并确认 Secret 没有被重新生成或失效。

### 3. 渠道已连接，但 NoxClaw 里没有企业微信会话怎么办？

这是正常的初始状态。企业微信渠道配置同步成功后会先显示已连接；只有当机器人收到私聊消息或群聊 @ 消息后，NoxClaw 才会生成对应的渠道会话。

### 4. 企业微信机器人没有响应怎么办？

请依次检查：

1. 企业微信机器人是否已经保存并启用。
2. 机器人 API 配置是否选择「使用长连接」。
3. NoxClaw 中填写的 Bot ID 和 Secret 是否与企业微信中的凭据一致。
4. 机器人可见范围是否包含当前测试成员。
5. NoxClaw 渠道卡片是否已经显示已连接。
6. 群聊中是否已经 @ 机器人。当前群聊消息通常需要 @ 机器人后才会进入处理链路。

### 5. Secret 泄露了怎么办？

请立即在企业微信机器人 API 配置中重新获取或重置 Secret，然后回到 NoxClaw 重新绑定企业微信机器人。不要继续使用已经泄露的旧密钥。
