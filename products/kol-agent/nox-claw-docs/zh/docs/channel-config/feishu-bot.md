# 飞书机器人

本文档说明如何把飞书机器人接入 NoxClaw。接入完成后，飞书用户可以通过私聊机器人或在群聊中 @ 机器人发送消息，NoxClaw 会把消息同步到当前席位的渠道会话中，并按后端确认的会话目标进行回复。

当前 NoxClaw 飞书渠道使用 **WebSocket 长连接** 模式。你只需要在 NoxClaw 中填写飞书应用的 **App ID** 和 **App Secret**；`Encrypt Key`、`Verification Token` 和 URL 回调地址属于 Webhook 模式信息，当前 NoxClaw 配置弹窗不需要填写。

## 配置前准备

开始前，请确认你已经具备以下条件：

- 可以登录 NoxClaw，并且当前账号有使用渠道配置的权限。
- 可以访问飞书开放平台，并且有创建或管理企业自建应用的权限。
- 已准备一个用于接入 NoxClaw 的飞书企业自建应用。建议一个飞书机器人只绑定一个 NoxClaw 席位，避免多个席位共享同一个机器人导致消息归属混乱。
- 可以在飞书开放平台为应用添加机器人能力、开通权限、配置事件订阅并发布应用。

App Secret 属于敏感凭据，请只在 NoxClaw 配置弹窗中使用，不要通过聊天、截图或公开文档泄露。

## 一、创建飞书应用

1. 打开飞书开放平台，进入「开发者后台」。
2. 点击「创建企业自建应用」。
3. 填写应用名称、应用描述和图标。应用名称建议使用能被业务成员识别的名称，例如 `NoxClaw Assistant`。
4. 创建成功后进入应用详情页。

> 图片占位：飞书开放平台创建企业自建应用页面。

## 二、添加机器人能力

1. 在应用详情页左侧菜单中进入「应用能力」。
2. 找到「机器人」能力并启用。
3. 根据企业内部规范设置机器人名称、头像和描述。
4. 保存后确认机器人能力已经启用。

> 图片占位：飞书应用能力中启用机器人页面。

## 三、配置权限

NoxClaw 需要飞书机器人具备接收消息、发送消息和识别发送人的能力。请在飞书开放平台的「权限管理」中按页面提示添加以下权限类型：

| 权限类型 | 用途 |
| --- | --- |
| 读取用户基础信息 | 用于识别消息发送人，并把私聊和群聊消息归并到正确的渠道会话。 |
| 接收私聊消息 | 用于接收用户直接发给机器人的消息。 |
| 接收群聊消息 | 用于接收群聊中 @ 机器人的消息。 |
| 发送消息 | 用于让 NoxClaw 把回复投递回飞书。 |
| 读取群聊基础信息 | 用于识别群聊来源和群消息上下文。 |

如果你的飞书开放平台支持通过 JSON 批量导入权限，可以按企业内部审核口径导入与消息收发、用户基础信息、群聊基础信息相关的权限。不要为了接入 NoxClaw 基础消息能力而额外开通日历、任务、文档写入等无关权限，除非你的企业明确需要这些飞书扩展能力。

> 图片占位：飞书开放平台权限管理页面。

## 四、获取应用凭证

1. 在应用详情页左侧菜单中进入「凭证与基础信息」。
2. 复制 **App ID**。
3. 点击查看并复制 **App Secret**。
4. 不要复制 App Key、Encrypt Key 或 Verification Token 到 NoxClaw 飞书配置弹窗。

> 图片占位：飞书应用凭证与基础信息页面。

## 五、在 NoxClaw 中配置飞书

1. 打开 NoxClaw，进入「渠道配置」。
2. 找到「飞书机器人」卡片，点击「配置」。
3. 确认接入方式显示为「WebSocket 长连接」。
4. 填写从飞书开放平台复制的 **App ID** 和 **App Secret**。
5. 点击「注册」。

提交后，NoxClaw 会校验 App ID / App Secret，并把飞书机器人配置同步到当前席位的运行态。配置同步过程中，渠道状态会显示为「绑定中」；同步成功后，渠道会进入「已连接」。

> 图片占位：NoxClaw 飞书机器人配置弹窗。

## 六、配置飞书事件订阅

返回飞书开放平台，进入应用详情页的「事件与回调」。

1. 在「订阅方式」中选择「使用长连接接收事件」。
2. 点击「验证」。验证成功后，页面会显示长连接已连接或验证成功。
3. 在「事件配置」中添加消息接收事件，确保机器人可以收到私聊和群聊消息。
4. 如果飞书开放平台页面提示需要添加机器人相关回调能力，请按页面提示添加；基础消息接入的核心是消息接收事件和长连接订阅方式。

NoxClaw 当前不使用「将事件发送至开发者服务器」这种 URL 回调方式，因此不需要复制或粘贴 Webhook 地址。

> 图片占位：飞书事件与回调中选择长连接接收事件页面。

## 七、发布应用

飞书应用必须发布后，企业成员才能正常使用机器人。

1. 点击应用详情页上方的「创建版本」。
2. 填写版本号和版本说明，例如 `1.0.0`、`接入 NoxClaw 飞书机器人`。
3. 提交发布。如果需要管理员审批，请联系企业管理员完成审批。
4. 发布成功后，在飞书客户端中搜索机器人名称，确认机器人可以被找到。

> 图片占位：飞书应用创建版本和发布页面。

## 八、开始使用

1. 在飞书客户端中搜索并打开刚创建的机器人。
2. 直接给机器人发送一条测试消息，或把机器人添加到群聊并 @ 机器人发送消息。
3. 回到 NoxClaw 的渠道会话页面，等待消息同步后查看飞书渠道会话。
4. 在 NoxClaw 中回复该会话。对于飞书群聊消息，如果系统能识别发送人，NoxClaw 会把站内回复投递到该发送人的私聊中，而不是直接回到群聊。

飞书机器人绑定成功只代表渠道账号已经可用。首次外部消息到达之前，NoxClaw 可能显示「已连接但暂无会话」，这是正常状态。

> 图片占位：飞书客户端搜索机器人并发送测试消息。

## 九、重新绑定和解绑

- 如果飞书渠道进入异常状态，可以点击「重新绑定」，重新填写 App ID 和 App Secret 后提交。NoxClaw 会在同一次重新绑定流程中清理失败生命周期并重新同步配置。
- 如果飞书渠道已经连接或正在绑定中，需要先点击「解绑」后再重新配置。
- 解绑后，NoxClaw 会移除当前席位与该飞书机器人的绑定关系。后续消息不再进入当前席位的飞书渠道会话。

## 十、常见问题

### 1. NoxClaw 需要填写 Encrypt Key 或 Verification Token 吗？

当前不需要。NoxClaw 飞书配置弹窗只要求填写 App ID 和 App Secret，并固定使用 WebSocket 长连接模式。Encrypt Key 和 Verification Token 是 Webhook 模式下常见的事件校验信息，当前不在 NoxClaw 弹窗中配置。

### 2. 注册提示 App ID 或 App Secret 无效怎么办？

请回到飞书开放平台的「凭证与基础信息」重新复制 App ID 和 App Secret。确认没有复制到 App Key、没有多余空格，并确认 App Secret 没有被重置。

### 3. 渠道已连接，但 NoxClaw 里没有飞书会话怎么办？

这是正常的初始状态。飞书渠道配置同步成功后会先显示已连接；只有当机器人收到私聊消息或群聊 @ 消息后，NoxClaw 才会生成对应的渠道会话。

### 4. 飞书机器人没有响应怎么办？

请依次检查：

1. 飞书应用是否已经发布并通过审批。
2. 机器人能力是否已经启用。
3. 权限管理中是否已经开通消息接收、消息发送和用户基础信息相关权限。
4. 事件订阅是否选择「使用长连接接收事件」，并且消息接收事件已经添加。
5. NoxClaw 渠道卡片是否已经显示已连接。
6. 群聊中是否已经 @ 机器人。当前群聊消息通常需要 @ 机器人后才会进入处理链路。

### 5. App Secret 泄露了怎么办？

请立即在飞书开放平台重置 App Secret，然后回到 NoxClaw 重新绑定飞书机器人。不要继续使用已经泄露的旧密钥。
