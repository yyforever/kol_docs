## 一、定位

### 一句话

**NoxInfluencer Agent 是网红营销全流程 AI 员工——品牌输入 Campaign brief，Agent 自主完成从找人→邀约→谈判→履约→追踪的完整流程。**

### 项目原则（2026-02-25 确认）

以下原则是项目基石，指导所有设计决策，不可违背：

| # | 原则 | 含义 |
|---|------|------|
| **P1** | **自主 Agent，无编排** | LLM 自主决定流程和工具调用，不使用编排框架/状态机/DAG |
| **P2** | **基于外部 Agent 运行时** | 依赖 OpenClaw（或同类 Agent 运行时）提供 agentic loop、Context 管理、记忆系统、工具调用、会话管理、定时调度。不重建这些机制，在其上构建业务层 |
| **P3** | **持久记忆，越用越好** | **核心飞轮**——品牌级记忆不断积累，Agent 对品牌的理解持续加深。平台级数据提升需专门研发投入，不依赖 Agent 自动产生 |
| **P4** | **每品牌持久会话** | 每个品牌拥有独立的持久 Agent 会话，记忆隔离，经验独立积累 |
| **P5** | **Agent 主动工作** | Agent 是员工不是工具——主动通知进展、跟进达人回复、提醒截止日期、建议下一步行动 |
| **P6** | **Tool 是原子操作** | Tool 不暗示调用顺序，不含 workflow 建议，Agent 自主判断下一步 |
| **P7** | **HITL 是交互层的事** | 审批确认的方式（按钮/对话）不影响 Agent 架构，是 UX 设计范畴 |

### 品牌关系

聚星（NoxInfluencer）品牌下两条产品线：

| 产品线 | 产品名（暂定） | 集成方 | 终端用户 | 形态 |
|--------|-------------|-------|---------|------|
| kol-api | NoxInfluencer API | Agent 平台 / 开发者 | 品牌（通过第三方 Agent 间接使用） | MCP / Skill / CLI |
| **kol-agent** | **NoxInfluencer Agent** | **聚星平台** | **品牌（直接使用）** | **嵌入聚星平台的 AI 员工** |

两条线共享 Core 层，并行开发，独立计费。

### 品类

| 品类 | 代表 | 人的角色 | NoxInfluencer Agent |
|------|------|---------|:------------------:|
| 传统 SaaS | GRIN, Modash, Aspire | 人操作工具 | ❌ |
| AI 辅助 SaaS | GRIN Gia, SARAL SIA | 人操作，AI 建议 | ❌ |
| **AI 员工** | Stormy, Aha, AMT | **人审批，AI 执行** | ✅ |

### 与 kol-api 的关系

```
┌─────────────────────────────────────────────────┐
│  kol-agent（本项目）                              │
│  嵌入聚星平台，品牌直接使用                        │
│  自主 Agent + Dashboard/Chat 混合 + 审批确认      │
├─────────────────────────────────────────────────┤
│  kol-api Core 层（并行开发，共享）                  │
│  discover · analyze · outreach · negotiate · manage│
├─────────────────────────────────────────────────┤
│  聚星 Service 层（数据/邮件/基础设施）              │
└─────────────────────────────────────────────────┘
```

- **kol-api** = 达人营销能力层，通过 MCP/Skill/CLI 分发到各 Agent 平台，品牌通过第三方 Agent 间接使用
- **kol-agent** = 面向品牌的完整产品，**嵌入聚星平台**，自主 Agent + Dashboard/Chat + 审批确认
- 两者共享同一套 Core 层，**并行开发**
- kol-agent 订阅**包含 API 使用**，不按 credit 额外计费

## 二、市场判断

### 判断 1：网红营销 AI Agent 赛道已从概念进入产品期

置信度：**高**

事实依据（2026-02 更新）：
- 5 个真 Agent 竞品已上线或 Beta：Stormy（YC S25）、Aha（朱啸虎投资未确认）、AMT（NFX $3.5M）、图灵、Scrumball（CrowdCore 已转型 Content Intelligence，不再计入）
- Stormy 定价已稳定：Free / $80 / $170 三档，Agent 层含 5 个 AI Agent 24/7 工作
- Aha 定位"24/7 AI Employee"：声称 5M+（PR 合作公告膨胀至 50M+）达人库、全链路自动化、escrow 模式、客群收窄至"Built for AI companies"
- GRIN 推出 Gia（AI assistant），传统 SaaS 开始 AI 化
- 行业预测：2026 全球网红营销市场 **$32.55B**，agentic workflow 是主要增长驱动

### 判断 2：现有竞品都处于早期，窗口仍然开放

置信度：**高**

- 5 个 Agent 竞品最早的（Stormy）也仅 YC S25，产品成熟度 60-70%
- 功能空白巨大：**履约管理、关系维护、跨平台归因**无人覆盖
- 客户隔离部署：5 个竞品**零覆盖**
- 窗口期估算：6-12 个月

### 判断 3：中国出海品牌是被低估的高价值细分

置信度：**高**

- 聚星 40 万存量客户，绝大多数是中国出海品牌
- Aha/图灵定位中国出海但各有缺陷（Aha 无 PLG + 客群收窄至"Built for AI companies"，图灵验证不足）
- 聚星天然具备中文界面 + 出海场景理解 + 存量客户零获客成本

## 六、产品形态

### 6.1 嵌入聚星平台

**kol-agent 嵌入聚星现有平台，作为新的 Agent 栏目。** 聚星平台作为入口，Agent 作为升级功能提供。

优势：
- 存量用户无需注册新账号
- 复用聚星已有的用户系统、权限、客户数据
- 降低开发成本（不用另建用户体系和支付系统）
- 品牌可以在"手动操作"和"Agent 自动"之间无缝切换
- 海外本土品牌通过聚星正常注册流程进入，Agent 是平台功能之一

### 6.2 界面：Dashboard + Chat 混合

| 界面 | 用途 | 交互 |
|------|------|------|
| **Chat** | 发起 Campaign、搜索达人、指令式操作 | 自然语言，"帮我找 10 个美妆 TikTok 达人" |
| **Dashboard** | Campaign 看板、达人 CRM、数据报表 | 可视化，状态和效果一目了然 |
| **审批流** | 邀约确认、谈判确认 | 结构化 UI，关键节点人工拍板 |

### 6.3 Human-in-the-loop 设计

> Agent 自主决定执行什么操作，下表定义的是**权限边界**（哪些操作需要人确认），不是预定义流程。确认方式是交互层的事（按钮/对话均可），对 Agent 架构没有影响。

| 操作 | Agent 自主度 | 人的角色 | MVP 阶段 |
|------|:----------:|---------|:--------:|
| 达人搜索/筛选/评估 | 🟢 全自主 | 看结果 | 全自动 |
| 邀约邮件内容生成 | 🟢 全自主 | 可选预览 | 全自动 |
| 邀约邮件发送 | 🟡 需确认 | 审批发送 | **AI 起草→人工审核→发送** |
| 谈判策略 | 🟡 需确认 | 审批预算范围 | AI 起草→人工审核 |
| 谈判执行 | 🟡→🟢 | 授权后自主 | **AI 起草→人工审核** → 后续放开全自动 |
| 合同签署 | 🔴 人工 | 品牌签 | 人工 |
| 履约/效果追踪 | 🟢 全自主 | 看报告/告警 | 全自动 |

**渐进式放权**：MVP 先 AI 起草+人工审核，积累少量经验后快速放开全自动发送。

### 6.3a 渐进放权（D12）

> 三级放权模型（L1 保守→L2 标准→L3 自主），品牌手动升级，维度独立控制（邀约/谈判/Follow-up 可独立设置），合同/超预算始终需人工。默认 L1，可随时降级。
>
> **详细方案设计（三级权限矩阵、UI 差异、升级流程）见 `03_产品设计.md` §六。**

## 七、商业模式概要

> 商业模式详细设计见 `04_定价与商业模式.md`。本节仅概述核心决策。

**即开即用的订阅制**：客户付订阅费获得即开即用的 AI 员工，技术细节全部对用户隐藏。所有层级**功能完全一致**，区别只是配额。

## 八、技术架构概览

> 技术架构详细设计（部署模式、多租户、聚星后端集成、邮件系统）见 `05_PRD.md` §三。本节仅概述核心架构决策。

**架构决策（2026-02-25 确认）**：
- kol-agent 作为**独立微服务**部署，与聚星 Java 后端通过 API 互相调用
- **自主 Agent 架构**（类 OpenClaw/Claude Code）：LLM + Tools + Context，无工作流引擎/状态机/编排框架
- 邮件发送走聚星现有基础设施（月发百万级，行业头部水平）

```
┌─────────────────────────────────────────────────┐
│  交互层：Dashboard + Chat 混合界面（嵌入聚星）     │
│  Campaign 看板 / 达人 CRM / 审批确认 / Chat 对话   │
├─────────────────────────────────────────────────┤
│  自主 Agent（独立微服务）                          │
│  LLM（推理+决策）→ 自主调用 Tools → 观察结果 → 循环│
│  ├── Tools: kol-api Core 层                      │
│  │   discover·analyze·outreach·negotiate·manage  │
│  ├── Context: 品牌画像 / Campaign 记忆 / Skills   │
│  └── 记忆: 对话历史 / 合作数据 / 学到的偏好        │
├──────────── API 互相调用 ────────────────────────┤
│  聚星平台（Java 后端）                             │
│  用户系统 / 权限 / 支付 / 邮件系统（百万级/月）     │
│  亿级达人数据 / 合作历史数据                        │
└─────────────────────────────────────────────────┘
```

**关键技术选型**：

| 组件 | 决策 |
|------|:----:|
| Agent 架构 | **自主 Agent（agentic loop）**，不使用编排框架 ✅ |
| 后端 | **独立微服务**，与聚星 Java 后端 API 互调 ✅ |
| 邮件发送 | 聚星现有邮件基础设施 ✅ |
| 前端/Chat UI/LLM | 实施时研发自定 ⚪ |

**部署模式**（对用户透明）：Starter/Pro 共享实例 + Enterprise AgentPod 独立容器。详见 `05_PRD.md` §三。
