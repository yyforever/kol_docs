## 一、定位

### 一句话

**NoxInfluencer Agent 是网红营销全流程 AI 员工——品牌输入 Campaign brief，Agent 自主完成从找人→邀约→谈判→履约→追踪的完整流程。**

### 品牌关系

聚星（NoxInfluencer）品牌下两条产品线：

| 产品线 | 产品名（暂定） | 集成方 | 终端用户 | 形态 |
|--------|-------------|-------|---------|------|
| kol-api | NoxInfluencer API | Agent 平台 / 开发者 | 品牌（通过第三方 Agent 间接使用） | MCP / Skill / CLI |
| **kol-agent** | **NoxInfluencer Agent** | **聚星平台** | **品牌（直接使用）** | **嵌入聚星平台的 AI 员工** |

两条线共享 Core 层，并行开发，独立计费。

### 品类

| 品类 | 代表 | 人的角色 | NoxInfluencer Agent |
|------|------|---------|:------------------:|
| 传统 SaaS | GRIN, Modash, Aspire | 人操作工具 | ❌ |
| AI 辅助 SaaS | GRIN Gia, SARAL SIA | 人操作，AI 建议 | ❌ |
| **AI 员工** | Stormy, Aha, AMT | **人审批，AI 执行** | ✅ |

### 与 kol-api 的关系

```
┌─────────────────────────────────────────────────┐
│  kol-agent（本项目）                              │
│  嵌入聚星平台，品牌直接使用                        │
│  Agent 编排 + Dashboard/Chat 混合 + 审批流        │
├─────────────────────────────────────────────────┤
│  kol-api Core 层（并行开发，共享）                  │
│  discover · analyze · outreach · negotiate · manage│
├─────────────────────────────────────────────────┤
│  聚星 Service 层（数据/邮件/基础设施）              │
└─────────────────────────────────────────────────┘
```

- **kol-api** = 达人营销能力层，通过 MCP/Skill/CLI 分发到各 Agent 平台，品牌通过第三方 Agent 间接使用
- **kol-agent** = 面向品牌的完整产品，**嵌入聚星平台**，Agent 编排 + 审批流
- 两者共享同一套 Core 层，**并行开发**
- kol-agent 订阅**包含 API 使用**，不按 credit 额外计费

## 二、市场判断

### 判断 1：网红营销 AI Agent 赛道已从概念进入产品期

置信度：**高**

事实依据（2026-02 更新）：
- 6 个真 Agent 竞品已上线或 Beta：Stormy（YC S25）、Aha（朱啸虎）、AMT（NFX $3.5M）、图灵、Scrumball、CrowdCore
- Stormy 定价已稳定：Free / $80 / $170 三档，Agent 层含 5 个 AI Agent 24/7 工作
- Aha 定位"24/7 AI Employee"：5M+ 达人库、全链路自动化、escrow 模式
- GRIN 推出 Gia（AI assistant），传统 SaaS 开始 AI 化
- 行业预测：2026 全球网红营销市场 **$32.55B**，agentic workflow 是主要增长驱动

### 判断 2：现有竞品都处于早期，窗口仍然开放

置信度：**高**

- 6 个 Agent 竞品最早的（Stormy）也仅 YC S25，产品成熟度 60-70%
- 功能空白巨大：**履约管理、关系维护、跨平台归因**无人覆盖
- 客户隔离部署：6 个竞品**零覆盖**
- 窗口期估算：6-12 个月

### 判断 3：中国出海品牌是被低估的高价值细分

置信度：**高**

- 聚星 40 万存量客户，绝大多数是中国出海品牌
- Aha/图灵定位中国出海但各有缺陷（Aha 无 PLG + 客群窄，图灵验证不足）
- 聚星天然具备中文界面 + 出海场景理解 + 存量客户零获客成本

## 六、产品形态

### 6.1 嵌入聚星平台

**kol-agent 嵌入聚星现有平台，作为新的 Agent 栏目。** 聚星平台作为入口，Agent 作为升级功能提供。

优势：
- 存量用户无需注册新账号
- 复用聚星已有的用户系统、权限、客户数据
- 降低开发成本（不用另建用户体系和支付系统）
- 品牌可以在"手动操作"和"Agent 自动"之间无缝切换
- 海外本土品牌通过聚星正常注册流程进入，Agent 是平台功能之一

### 6.2 界面：Dashboard + Chat 混合

| 界面 | 用途 | 交互 |
|------|------|------|
| **Chat** | 发起 Campaign、搜索达人、指令式操作 | 自然语言，"帮我找 10 个美妆 TikTok 达人" |
| **Dashboard** | Campaign 看板、达人 CRM、数据报表 | 可视化，状态和效果一目了然 |
| **审批流** | 邀约确认、谈判确认 | 结构化 UI，关键节点人工拍板 |

### 6.3 Human-in-the-loop 设计

| 操作 | Agent 自主度 | 人的角色 | MVP 阶段 |
|------|:----------:|---------|:--------:|
| 达人搜索/筛选/评估 | 🟢 全自主 | 看结果 | 全自动 |
| 邀约邮件内容生成 | 🟢 全自主 | 可选预览 | 全自动 |
| 邀约邮件发送 | 🟡 需确认 | 审批发送 | **AI 起草→人工审核→发送** |
| 谈判策略 | 🟡 需确认 | 审批预算范围 | AI 起草→人工审核 |
| 谈判执行 | 🟡→🟢 | 授权后自主 | **AI 起草→人工审核** → 后续放开全自动 |
| 合同签署 | 🔴 人工 | 品牌签 | 人工 |
| 履约/效果追踪 | 🟢 全自主 | 看报告/告警 | 全自动 |

**渐进式放权**：MVP 先 AI 起草+人工审核，积累少量经验后快速放开全自动发送。

## 七、商业模式

### 7.1 订阅制 + 配额上限

订阅后**核心功能全部可用**，各类操作设**配额上限**（正常使用足够），订阅后配额放大。与聚星现有模式一致。

| 层级 | 月费 | 策略 | 目标用户 |
|------|:----:|------|---------|
| **Free** | $0 | 给够跑完 1 个完整 Campaign 的配额（搜索+邀约+AI 谈判） | 体验全流程→转化 |
| **Starter** | $29 | 配额放大，满足 SMB 日常使用 | SMB / DTC |
| **Pro** | $99 | 配额继续放大 + CRM + 报表 | 中型品牌 |
| **Growth** | $199 | 高配额 + 全功能 | 重度用户 / 小 Agency |
| **Enterprise** | 定制 | 独立 Agent 空间（AgentPod）+ 私有化 + 定制 | MCN / 大品牌 |

**配额细节暂不定**——需要实际使用数据来校准。当前策略：Free 层给够 1 个完整 Campaign 的量，付费后按层级放大。

### 7.2 关键定价决策

| 决策 | 选择 | 理由 |
|------|------|------|
| **计费方式** | **订阅 + 配额上限** | 与聚星现有模式一致；配额是上限，正常使用够用 |
| **Free 层** | **1 个完整 Campaign 的配额** | 让品牌走完全流程是最强转化 |
| **付费墙** | 配额用完后需订阅 | 自然过渡 |
| **API 使用** | **订阅包含** | 品牌不需要额外算 credit |
| **独立空间** | Enterprise 起提供 | Free-Growth 共享资源池（应用层隔离）；Enterprise 独立 Agent 容器（AgentPod） |

### 7.3 vs 竞品定价

| | Free | 低价 | 中价 | 高价 |
|-|:----:|:----:|:----:|:----:|
| **聚星 Agent** | $0（1 Campaign） | **$29** | **$99** | **$199** |
| Stormy | $0（110 credits） | $80 | — | $170 |
| CrowdCore | — | $19 | $49 | $79 |
| Aha | — | — | 按交付付费 | — |

低端比 Stormy **便宜 64%**（$29 vs $80），高端**功能碾压**（全平台+履约+归因+客户隔离）。

### 7.4 与 kol-api 的关系

- kol-agent 用户内部调用 Core 层 → **不消耗 kol-api credit**
- kol-api 是独立产品线（给其他 Agent 平台的开发者/品牌用户）
- 两条线独立计费，**不互相挤压**

## 八、技术架构概览

### 8.1 四层架构

```
┌─────────────────────────────────────────────────┐
│  Layer 0: 聚星平台（入口 + 用户系统）              │
│  现有用户体系、权限、支付                          │
├─────────────────────────────────────────────────┤
│  Layer 1: kol-agent 前端（Agent 栏目）            │
│  Dashboard + Chat 混合界面（嵌入聚星）             │
│  Campaign 看板 / 达人 CRM / 审批流 / Chat 对话     │
├─────────────────────────────────────────────────┤
│  Layer 2: Agent 编排层                            │
│  LLM 编排 / 工作流引擎 / 记忆管理 / 审批状态机     │
├─────────────────────────────────────────────────┤
│  Layer 3: kol-api Core 层（共享）                  │
│  discover · analyze · outreach · negotiate · manage│
├─────────────────────────────────────────────────┤
│  Layer 4: 聚星 Service 层                         │
│  亿级达人数据 / 邮件系统 / 合作数据 / AI 引擎      │
└─────────────────────────────────────────────────┘
```

### 8.2 部署模式分层

| 用户层级 | 部署模式 | 隔离方式 | AgentPod |
|---------|---------|---------|:--------:|
| **Free-Growth** | 共享资源池 | 应用层逻辑隔离（数据库 tenant_id） | 不需要 |
| **Enterprise** | **每客户独立 Agent 容器** | 容器级物理隔离 | ✅ 核心基础设施 |
| **私有化** | 客户自有服务器 | 完全独立 | ✅ 容器导出+部署 |

**为什么 Enterprise 需要独立容器？**

| 需求 | 共享多租户 | 独立容器（AgentPod） |
|------|:---------:|:-----------------:|
| 数据隔离 | 应用层（可接受） | **物理隔离**（Enterprise 要求） |
| Agent 记忆 | 共享池 | **独立上下文** |
| 崩溃隔离 | 互相影响 | **互不影响** |
| 私有化迁移 | 需重构 | 容器直接导出 |

**AgentPod 已有基础**：
- ✅ Control Plane + 调和引擎 + Docker API
- ✅ Traefik 自动子域名 + HTTPS
- ✅ PostgreSQL 状态存储
- 🟡 需新增：kol-agent Adapter

**成本估算**（Enterprise 容器）：
- 单容器：~256MB RAM + 0.5 vCPU
- 单台 8GB/4 核服务器可承载 ~20-30 个 Enterprise Agent
