## 定位

品牌输入 Campaign brief → Agent 自主完成找人→邀约→谈判→履约→追踪。嵌入聚星平台，每个 Enterprise 客户独立 Agent 空间（AgentPod）。

## 核心决策（2026-02-23）

| 决策 | 结论 |
|------|------|
| 品牌 | NoxInfluencer / 聚星（不换） |
| 产品形态 | **嵌入聚星平台**（非独立站） |
| 界面 | Dashboard + Chat 混合 |
| 计费 | **纯订阅制**（包 API），$0/29/99/199/定制 |
| Free 层 | 1 个完整 Campaign |
| P0 用户 | 双线并列：中国出海（存量维稳）+ 海外本土（增量优先） |
| MVP 谈判 | AI 起草 + 人工审核 → 快速放开全自动 |
| 部署 | 云端优先，Enterprise 用 AgentPod 独立容器 |
| 与 kol-api | 并行开发，共享 Core 层 |

## 核心决策补充（02 新增）

| 决策 | 结论 |
|------|------|
| D11 CRM | **必须进 MVP** — 数据积累基础 + 留存引擎 |
| D11a CRM 导入 | **支持导入已有网红关系** — 冷启动加速 |
| D12 审批细节 | **后续调整** — 渐进放权方向确定，细节需实际使用校准 |

## 业务确认（2026-02-25）

| 项 | 确认 |
|-----|------|
| 邮件规模 | ~100 万/月（建联+谈判+合作管理），行业头部 |
| AI 建联 | 做到与人工一致可行，响应率更多取决于"找谁"而非"怎么写" |
| Agent 架构 | **自主 Agent**（类 OpenClaw/Claude Code），无编排框架，LLM + Tools + Context |
| 后端架构 | **独立微服务**，与聚星 Java 后端 API 互调 |
| 邮件基础设施 | 走聚星现有 |
| 嵌入可行性 | ✅ 已确认（不是大问题） |
| 谈判 PoC 数据 | ✅ 历史数据充足 |
