## 调研TODO

按顺序推进，每步调研完毕确认结论后再进下一步。

### Phase 1：定位验证
- [ ] **竞品API深度体验**：实际注册Modash/CreatorDB等试用，拿到真实数据样本评估质量
- [ ] **目标客户访谈/需求验证**：和3-5个潜在买家（Agent开发者/SaaS构建者）聊，验证需求和付费意愿
- [ ] **聚星现有数据质量评估**：实际调用聚星内部API，评估数据覆盖、新鲜度、准确度
- [ ] **定位确认**：基于以上验证，确认或调整01中的定位假设

### Phase 2：场景与能力
- [ ] **开发者旅程验证**：基于Phase 1访谈，验证02中的场景优先级
- [ ] **API能力清单确认**：哪些endpoint必须有，哪些可以砍
- [ ] **MCP vs REST优先级**：开发者实际偏好
- [ ] **假粉检测方案**：评估Phyllo等供应商，确定v1方案
