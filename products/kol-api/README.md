## 文档结构

从调研到PRD的完整管线。每个阶段产出一个核心文档，后一个阶段依赖前一个。

```
kol-api/
├── README.md                  ← 本文件（导航）
│
├── 01_定位与假设.md            ← 产品定位、核心假设、成功标准
│                                谁买？为什么买？凭什么是我们？
│
├── 02_用户场景.md              ← 具体调用场景、开发者旅程
│                                开发者从注册到付费的完整路径
│
├── 03_API能力设计.md           ← endpoint清单、数据模型、能力分层
│                                卖什么能力？怎么分tier？
│
├── 04_定价与商业模式.md         ← 定价策略、限量机制、防竞品
│                                怎么收钱？怎么防数据被搬？
│
├── 05_PRD.md                  ← 最终PRD（MVP scope、优先级、验收标准）
│                                工程团队拿到就能开工的文档
│
└── _research/                 ← 补充调研素材（不进PRD）
    └── ...
```

**原则**：每个文档只回答一个核心问题，文档之间有明确依赖关系。01→02→03→04→05线性推进，每步产出经确认后才进下一步。
