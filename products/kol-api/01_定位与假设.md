## 一、定位

### 一句话

**kol-api 是品牌广告主在 AI Agent 中使用的达人数据能力——让任意 Agent 都能帮品牌找到对的达人。**

### 品类

不是"达人数据开发者 API"，是 **AI Agent 达人数据能力**。

### 类比

Brave Search 之于 AI Agent 的搜索能力 = kol-api 之于 AI Agent 的达人营销能力。

- Brave Search 不做自己的 Agent 平台，在各平台上可选启用
- 品牌用户能感知到"用了 vs 没用"的结果质量差异
- 多平台分发，不绑定任何一家

### 定位翻转

| 维度 | 旧定位（占位版） | 新定位 | 翻转原因 |
|------|----------------|--------|---------|
| **客户** | 开发者 | 品牌广告主 | 品牌有营销预算，付费意愿和能力远高于开发者 |
| **交付方式** | REST API + SDK | Agent 平台 Skill/MCP | 品牌通过 Agent 使用能力，不需要写代码 |
| **体验标准** | DX（文档/SDK/错误码） | 结果质量 + 交互自然度 | 品牌在意"一句话给我靠谱清单"，不在意 API 设计 |
| **竞品** | Modash API / CreatorDB | 人工调研 / MCN 推荐 / SaaS 仪表盘 | 品牌的替代方案不是别的 API，是别的工作方式 |
| **分发** | RapidAPI / MCP 目录 | Agent 商店（OpenClaw/Claude/ChatGPT...） | 品牌在 Agent 平台里找工具，不在开发者目录 |

## 二、市场判断

定位基于三个判断。前两个有数据支撑，第三个是趋势推断——也是最大的不确定性。

### 判断 1：AI Agent 正在成为新的办公入口

置信度：**高**

事实依据：
- OpenClaw 2025-11 启动，2026-01 48 小时内 9K→106K Stars，目前 157K+
- ClawHub 10 天内从 0 增至 5,700+ Skill，日增 500+
- Manus $100M ARR，Meta $2B 收购进行中
- Claude Code、ChatGPT、Copilot 均在构建 Agent 商店/插件体系
- 非技术岗位（营销、运营、HR）开始使用 Agent 处理日常工作

### 判断 2：达人营销品类在 Agent 生态中是空白

置信度：**高**

事实依据：
- 6 个超级 Agent 商店（OpenClaw/Claude/ChatGPT/Copilot/Manus/Google），达人营销工具数均为 **零**
- MCP 目录 17,000+ server 中，Marketing 仅 9 个，达人数据专业工具 0 个生产级可用
- ClawHub Marketing 品类 700+ Skill，达人数据方向几乎空白
- 综合缺口率 > 99%

> 详见 `竞品分类图谱.md` 第六节

### 判断 3：品牌广告主会通过 Agent 做达人营销

置信度：**中**——这是本文最关键也最不确定的判断。

支撑逻辑：
- 达人营销核心动作（搜索、筛选、比对、评估）是信息密集型工作，天然适合 AI 处理
- 品牌当前痛点明确：人工筛选耗时数天，MCN 推荐有利益偏差，SaaS 工具学习成本高且价格高
- Agent 是解决这些痛点的自然界面：一句话描述需求 → 即时获得结构化结果

反面论据：
- 目前没有公开数据显示品牌广告主正在用 Agent 做达人筛选
- Agent 办公习惯尚在早期，非技术岗位的渗透率未知
- 达人营销决策往往涉及主观判断（品牌调性、内容风格），AI 能否胜任存疑

**结论：这是一个趋势预判，不是已验证的需求。kol-api 是先发卡位，不是追随需求。**

## 三、客户

### 核心客户

使用 AI Agent 办公的**品牌广告主 / 营销团队**。

画像：
- 有达人营销预算（年 10 万-数百万）
- 已在使用 OpenClaw / Claude / ChatGPT 等 Agent 平台
- 日常需要搜索、筛选、评估达人
- 对 MCN 推荐的中立性有疑虑，或不愿为 SaaS 仪表盘付年费

### 核心场景

| 场景 | 用户对 Agent 说 | kol-api 提供 |
|------|---------------|-------------|
| 达人发现 | "帮我找 10 个美国美妆类 TikTok 达人，粉丝 5-20 万" | 秒级结构化达人清单 |
| 达人评估 | "这个达人数据真实吗？受众主要在哪？" | 假粉检测 + 受众画像 |
| 竞品对标 | "竞品最近合作了哪些达人？效果如何？" | 品牌合作历史 + 效果数据 |
| 预算估算 | "跟这个达人合作大概要多少钱？" | CPM/CPE 估算 |

### v1 不服务

| 需求 | 原因 | 替代方案 |
|------|------|---------|
| 达人 CRM / 合同 / 支付 | 不是数据能力，是工作流产品 | GRIN / Impact |
| 纯中国市场（抖音/小红书/B 站） | v1 优先国际市场验证 | 后续版本 |
| 创作者授权数据（收入/私信） | 需要 Creator OAuth，技术路径不同 | Phyllo |

## 四、价值主张

### 核心价值

品牌当前做达人筛选的方式：
- 人工搜索：3-5 天 + 1 个人力
- SaaS 仪表盘：$5K-30K/年，需 Sales call，需学习操作
- MCN 推荐：免费但有利益偏差

用了 kol-api 的 Agent：**一句话，30 秒，数据驱动，无偏差。**

这不是"更便宜的 Modash"——是完全不同的交互方式。

### 凭什么是我们

| 优势 | 说明 | 置信度 |
|------|------|:------:|
| **数据资产** | 聚星 10 年达人数据，6 平台覆盖（YT/TikTok/IG/FB/X/NaverBlog），含合作效果数据和频道价值估算 | 中 |
| **成本结构** | 中国团队，运营成本是 Modash/HypeAuditor 的 1/3-1/5 | 高 |
| **零竞争** | 达人营销 Agent 能力市场当前生产级竞品为零 | 高 |
| **先发时机** | 99%+ 供给缺口 + 窗口期 6-12 个月 | 高 |
| **已有 API** | 聚星已有 6 平台 API endpoint（搜索/频道/视频/评论/受众/品牌分析），非从零开始 | 高 |

### 凭什么不是我们（诚实面对）

| 劣势 | 严重度 | 说明 |
|------|:------:|------|
| **全球数据质量未验证** | 🔴 | 聚星在亚洲可能强，但 Agent 用户以英语市场为主，欧美数据质量未知 |
| **无假粉检测** | 🔴 | 品牌最核心需求之一，v1 缺失意味着价值打折 |
| **数据规模不透明** | 🟠 | 聚星宣称"millions"，Modash 350M+，HypeAuditor 207.8M+，差距可能大 |
| **AI 搜索不成熟** | 🟠 | 当前聚星 AI 搜索是标签匹配，非真正自然语言理解（`nox-api-research.md` 第 3 节） |
| **海外品牌零认知** | 🟡 | NoxInfluencer 在海外开发者/品牌中无认知度 |
| **Agent 平台经验为零** | 🟡 | 从未在 MCP/Skill 生态发布过产品 |

> **数据质量（🔴）是最大前置风险。** 如果聚星数据在欧美市场不达标，整个定位的价值基础不成立。见第八节 H4。

## 五、竞争格局

### 品牌视角的替代方案

品牌广告主不会拿 kol-api 跟 Modash API 比——他们会拿 Agent 体验跟自己当前的工作方式比：

| 替代方案 | 成本 | 缺点 | kol-api 的优势 |
|---------|------|------|--------------|
| 手动搜索（平台+搜索引擎） | 3-5 天人力 | 无结构化输出，效率低 | 秒级 + 结构化 |
| MCN 推荐 | 免费 | 有利益偏差，不中立 | 数据驱动，无偏差 |
| SaaS 仪表盘 | $5K-30K/年 | 需 Sales call，学习成本高 | 对话式交互，零学习成本 |
| 不用数据，靠直觉 | 零 | 效果不可控 | 数据辅助决策 |

### Agent 生态竞品

当前无生产级竞品。详见 `竞品分类图谱.md` 品类 2：

- CreatorDB MCP：31 tools，但 Glama F 级不可安装，零使用量
- AnyTag MCP：仅 PR 阶段，无公开代码
- Composio：已关停
- SociaVault：三 A 评分但周下载仅 2，功能浅层

### 需持续监控

| 信号 | 触发条件 | 影响 |
|------|---------|------|
| CreatorDB 修复 LICENSE | GitHub 仓库变更 | 31 tools，威胁立升中高 |
| Modash 推出 MCP/Skill | 官方公告 | DX 标杆 + 350M 数据，高威胁 |
| Agent 平台内置达人搜索 | 平台更新 | 被平台替代的风险 |

## 六、分发策略

### 核心原则

**多平台分发，不绑定任何一家。**

Agent 平台格局变化极快（OpenClaw 两个月内三次改名），绑定单一平台风险极高。

### 分发矩阵

| 平台 | 格式 | 优先级 | 理由 |
|------|------|:------:|------|
| **OpenClaw / ClawHub** | SKILL.md | P0 | 当前最火，157K Stars，非技术用户增长快 |
| **Claude Code / Anthropic** | MCP Server | P0 | 企业 Agent 生态最成熟 |
| **ChatGPT App Store** | GPT Action | P1 | 3 亿+ MAU，最大消费级入口 |
| **Copilot Agent Store** | Agent | P1 | M365 企业入口 |
| **Glama / Smithery / mcp.so** | MCP Server | P1 | MCP 信任三角，影响专业用户决策 |
| **SkillsMP / Skills.sh** | SKILL.md | P2 | 跨平台开放标准目录 |

### 技术架构

一套核心 API，两个接入层：

```
品牌广告主 → Agent 平台（OpenClaw / Claude / ChatGPT ...）
                ├── SKILL.md 接入层 → OpenClaw / Claude Code / Codex CLI
                └── MCP Server 接入层 → Claude / Cursor / Glama
                         ↓
                    kol-api 核心服务（REST API）
                         ↓
                    聚星数据后端
```

MCP 不替代 REST，是在 REST 之上加 Agent 接入层。核心 API 只建一次。

## 七、商业模式

### 收入模型假设

| 模式 | 说明 | 适用阶段 |
|------|------|---------|
| **Freemium + 按量付费** | 免费额度（50 次/天）→ 超出按次计费 | v1 核心 |
| **订阅制** | 月付 $29-199，按查询量分层 | 用户习惯验证后 |
| **平台分成** | 通过 Agent 平台付费机制（如 ClawHub 第三方市场 70% 分成） | 视平台政策 |

### 定价锚点

| 对比对象 | 品牌当前付出 | kol-api 目标价 |
|---------|-----------|--------------|
| SaaS 仪表盘（Modash 平台） | $135-1,350/月 | $29-199/月（便宜 3-7x） |
| 人工调研（1 个实习生） | ~$2,000/月工资 | 替代，非价格竞争 |
| MCN 推荐 | "免费"但有隐性成本 | 提供中立数据的溢价 |
| Agent Skill 已有定价 | $10-200/月（ClawHub 已有付费案例） | 符合用户预期 |

> 注：具体定价策略待 `04_定价与商业模式.md` 展开。

### 商业模式

| # | 假设 | 判断 | 验证方式 | 时间 |
|---|------|:----:|---------|------|
| H7 | 多平台分发运维成本可控 | 🟡 | 实际上架 2-3 个平台后评估 | 上线后 |
| H8 | 平台不会抽取过高分成 | ✅ | ClawHub 当前不抽成，持续跟踪 | 持续 |

### 前置验证项（开工前必须做）

**H4（数据质量）是唯一的前置红线。**

建议方案：
1. 抽样 100 个欧美头部达人（Beauty/Fashion/Tech 各类别）
2. 对比聚星数据 vs 竞品公开数据（Modash/HypeAuditor 平台可试用）
3. 评估三个维度：覆盖率（有没有）、新鲜度（多久更新）、准确度（粉丝数/互动率偏差）
4. 如果覆盖率 < 70% 或准确度偏差 > 20%：需补数据源或调整定位至亚洲市场

## 十、风险

按严重度排序：

| # | 风险 | 严重度 | 发生概率 | 应对 |
|---|------|:------:|:------:|------|
| 1 | **需求不存在** — 品牌还没准备好用 Agent 做达人营销 | 致命 | 中 | 最小版本先测需求，不投重资源 |
| 2 | **数据不达标** — 聚星欧美数据 vs 竞品差距过大 | 致命 | 中 | 开工前抽样验证 H4 |
| 3 | **平台变局** — Agent 平台格局剧变，今天的热门明天衰落 | 高 | 高 | 多平台分发，一套 API 多个接入层 |
| 4 | **巨头入场** — Modash/GRIN 快速推出 MCP/Skill | 高 | 低 | 抢先发 + PLG 定价 + 多平台网络效应 |
| 5 | **AI 搜索质量差** — 自然语言→达人结果不准确、不可靠 | 高 | 中 | 技术 PoC 先行，降低对 AI 搜索的依赖（支持结构化筛选） |

### 退路

如果 H1（品牌用 Agent 做达人营销）12 个月内未验证，kol-api 可回退至开发者 API 定位，将已建成的 MCP/Skill 层作为获客渠道。回退成本低：MCP Server + SKILL.md + REST API 是同一基础设施的不同接入层。
