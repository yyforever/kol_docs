## 核心场景（按调用量排序）

### 场景1：Agent集成 — "我的Agent需要查达人"

**用户**：构建营销Agent/工作流的团队（Stormy、Aha类产品，或企业内部Agent）

**旅程**：
```
Agent收到用户指令"帮我找10个美妆类TikTok达人"
  → 调用 MCP tool: search_creators(niche="beauty", platform="tiktok", min_followers=10000)
  → 返回结构化结果（含匹配度评分、预估报价、假粉率）
  → Agent筛选后推荐给用户
```

**核心endpoint**：
- `search_creators` — 自然语言/结构化搜索
- `get_creator_profile` — 单个达人详情
- `check_authenticity` — 假粉检测

**调用模式**：中频批量（每次10-500个结果），按任务触发
**价值**：Agent不用自己建数据层，专注上层逻辑

---

### 场景2：SaaS白标 — "我要在我的产品里嵌入达人搜索"

**用户**：构建达人营销SaaS的创业公司、Agency内部工具

**旅程**：
```
SaaS用户在界面输入筛选条件
  → 后端调用 REST API: GET /creators?niche=fitness&platform=instagram&country=US
  → 返回达人列表（含头像、粉丝数、互动率、联系方式）
  → 前端渲染搜索结果页
```

**核心endpoint**：
- `GET /creators` — 批量搜索（支持20+筛选维度）
- `GET /creators/{id}` — 达人详情
- `GET /creators/{id}/audience` — 受众画像
- `GET /creators/{id}/content` — 近期内容分析

**调用模式**：高频持续（SaaS用户每次操作=1次API调用）
**价值**：不用自建达人数据库，3天接入 vs 3个月自建

---

### 场景3：外联增强 — "我有达人handle，需要补充数据"

**用户**：已有外联工具的团队，需要enrichment

**旅程**：
```
用户有一批达人的Instagram handle
  → 批量调用 POST /creators/lookup (handles=[...])
  → 返回每个达人的完整profile + 联系方式 + 假粉率
  → 导入自己的CRM/外联工具
```

**核心endpoint**：
- `POST /creators/lookup` — 按handle/URL批量查询
- `GET /creators/{id}/contacts` — 联系方式（邮箱/社媒）

**调用模式**：低频大批量（一次100-10000个）
**价值**：不用人工逐个查，批量enrichment

---

### 场景4：效果监控 — "追踪合作达人的表现"

**用户**：正在运行campaign的品牌/Agency

**旅程**：
```
Campaign上线后
  → 调用 GET /creators/{id}/stats?date_from=2026-02-01
  → 返回粉丝增长、互动率变化、近期内容表现
  → 或注册 Webhook: content_published → 自动通知新内容上线
```

**核心endpoint**：
- `GET /creators/{id}/stats` — 实时统计
- `POST /webhooks` — 内容发布/指标变化通知

**调用模式**：中频持续（Webhook驱动）
**价值**：不用人工盯着看，自动追踪

## 开发者旅程（从注册到付费）

```
第0分钟 ─ 发现
  │  渠道：MCP目录 / RapidAPI / HN帖子 / SEO文章 / GitHub SDK
  │
  ▼
第1分钟 ─ 注册（零摩擦）
  │  邮箱注册 → 立即获得API Key → 无需信用卡
  │  免费额度：1,000次调用/月
  │
  ▼
第5分钟 ─ 首次调用成功（Aha Moment）
  │  curl / Python SDK / MCP工具 → 返回真实达人数据
  │  "这数据质量不错，比自己爬强多了"
  │
  ▼
第1-7天 ─ 原型验证
  │  用免费额度跑通自己的场景
  │  发现数据覆盖够、字段够、响应快
  │
  ▼
第2-4周 ─ 触发付费
  │  免费额度用完 / 需要高级字段（受众画像、假粉检测）
  │  升级 Starter $49/月 → 10,000次调用
  │
  ▼
第2-6月 ─ 深度集成
  │  成为产品核心依赖 → 升级 Pro $149/月 → 50,000次调用
  │  或按量付费（超量$0.01/次）
  │
  ▼
持续 ─ 锁定
  │  迁移成本高（数据模型已深度集成）
  │  续约率预期 >90%
```

## 免费层设计

| 维度 | 设计 | 理由 |
|------|------|------|
| **调用量** | 1,000次/月 | 够跑原型，不够生产 |
| **数据字段** | 基础字段（粉丝数、互动率、niche、国家） | 让人看到价值 |
| **付费墙** | 受众画像、假粉检测、联系方式、批量lookup | 生产刚需 |
| **信用卡** | 不要求 | 74%开发者把透明定价列为首要因素 |
| **MCP** | 免费可用（同样1,000次限额） | MCP是获客渠道不是收费点 |

## 与kol-agent的关系

kol-agent（路线2）是kol-api的**最大内部客户**：

| kol-agent功能 | 调用的kol-api endpoint |
|--------------|----------------------|
| 达人发现/匹配 | `search_creators` |
| 达人验证 | `check_authenticity` + `get_creator_profile` |
| 外联数据 | `GET /creators/{id}/contacts` |
| 效果追踪 | `GET /creators/{id}/stats` + Webhooks |
| AI匹配 | `search_creators`（自然语言模式） |

**设计原则**：kol-api的能力边界 = kol-agent所需的全部数据层。kol-api做好了，kol-agent就是上层应用逻辑。
