# 02 用户场景

> 状态：已同步（核心假设上线后验证）
> 更新：2026-03-20
> 依赖：`01_定位与假设.md`、`_research/竞品分类图谱.md`、`_research/海外发布中心市场分析.md`

### ⚠️ 定位演进说明

本文在商业化评审后进一步收敛了 Day 1 边界。品牌的本质痛点仍是**人工成本高**，但 Day 1 不追求一口气做完整自动化，而是先做最可防守、最容易付费的三段：**搜索、深度评估、效果监控**。

NoxInfluencer 的定位从"全链路自动化优先"收敛为 **AI 驱动的达人搜索、深度评估与效果监控能力层**——先把数据产品做深，再决定是否向执行自动化扩展。

## 一、品牌广告主的达人营销现状

### 1.1 通用工作流与人工成本分布

品牌做达人营销的核心链路一致，但**各环节的人工成本差异巨大**：

```
搜索 → 筛选 → 评估 → 邀约 → 谈判 → 执行管理 → 效果追踪
 8%     12%     8%    22%    33%     10%        7%
 └─── 28% ───┘        └── 55% ──┘
  信息密集型              沟通密集型（最烧人力）
```

> 数据来源：Influencer Hero 研究——激活 50 个达人的完整流程，人工需 101-175 小时

**核心发现**：搜索+筛选+评估虽然只占 28% 的人工时间，但它是最适合做 Agent Day 1 切入口的部分，因为可即时展示价值、结构化程度高、且最容易做精细化计费。效果追踪虽然只占 7% 的操作时间，却直接决定 ROI 判断和复投，是高价值能力，应上调到 P0。邀约+谈判仍是高痛点，但在 Day 1 里不是最高优先级。

### 1.2 各环节痛点量化

| 环节 | 占人工时间 | 痛点 | 量化数据 | 来源 |
|------|:---------:|------|---------|------|
| 搜索 | ~8% | 跨平台手动搜索，无结构化对比 | 人工搜索 3-5 天/轮 | 行业调研 |
| 筛选 | ~12% | 假粉泛滥，肉眼无法判断 | 49% IG 达人存在某种程度欺诈 | HypeAuditor 报告 |
| 评估 | ~8% | 受众画像需多源交叉验证 | 79% 营销人员认为 ROI 归因是最大挑战 | eMarketer |
| **邀约** | **~22%** | 响应率低，联系方式难找 | 需发 333+ 封邮件激活 50 人（响应率 8-20%），邮箱覆盖率仅 25-40% | Influencer Hero |
| **谈判** | **~33%** | 无标准定价，87.9% 营销人员说越来越难 | 平均每个达人 6 轮邮件往返，58% 达人要求混合报酬 | Modash 2025 调查 |
| 执行管理 | ~10% | 物流/审稿/合规/付款全靠人盯 | 各环节分散在不同工具和表格 | 聚星客户反馈 |
| 效果追踪 | ~7% | ROI 未知 + 曝光量不足 | 这是两个本质痛点需求 | 聚星客户反馈 |

### 1.3 EU/US 品牌 vs 国内品牌的关键差异

| 维度 | EU/US 品牌 | 国内品牌（聚星客户） |
|------|-----------|-------------------|
| **主要工作方式** | SaaS 仪表盘 + 手动搜索 | SaaS 工具自助（聚星平台） |
| **工具支出** | 自助 $49-299/月，中型 $478-2,000/月 | 平台年费制为主 |
| **最痛环节** | 假粉/数据真实性 + 邀约谈判耗时 | 人工成本高（全链路都费劲）+ ROI 未知 |
| **合规压力** | FTC 单条违规 $53K+，GDPR 上限收入 4% | 相对宽松，但趋严 |
| **AI 采用** | 82% 使用 AI 工具，仅 35% 看到显著效果 | 起步阶段 |
| **人力成本** | 专职经理 $83K-120K/年，高级 $215K/年 | 相对较低但也在增长 |

v1 优先服务 EU/US 品牌：Agent 渗透率高、工具支出意愿已验证、人力成本压力大（自动化 ROI 更明显）。

## 二、用户旅程（从发现到付费）

### 2.1 发现路径：Agent 自动发现是第一优先级

**必须正视的现实**：非技术用户不会主动逛 Skill 商店。ClawHub 安装需要 CLI（`npx clawdhub@latest install`），对品牌广告主是硬门槛。

因此，**Agent 自动发现和调用 NoxInfluencer 是最关键的分发策略**——品牌不需要知道 NoxInfluencer 的存在，只需要在 Agent 对话中提出达人营销需求，Agent 自动发现并调用。

| # | 路径 | 优先级 | 说明 |
|---|------|:------:|------|
| A | **Agent 自动发现并调用** | **最高** | 品牌在 Agent 中提需求 → Agent 自动发现 NoxInfluencer → 直接调用并返回结果。类似 ChatGPT 自动调用 Brave Search，用户无需安装。需要在各平台注册为"达人营销"品类的首选工具 |
| B | **代理商/IT 预配置** | 高 | 有人帮品牌在 Agent 环境中配好 NoxInfluencer，品牌通过聊天界面使用 |
| C | **内容营销引流** | 中 | 品牌从行业文章/视频/案例看到效果 → 让技术人员或 Agency 协助配置 |
| D | **品牌自助安装** | 低 | 仅在技术门槛降低后可行（ChatGPT Apps 免代码安装 / OpenClawd.ai 托管方案） |

**落地要求**：
- 各 Agent 平台的"达人营销"语义触发词注册（influencer / KOL / creator marketing）
- ChatGPT Apps 上架（对非技术用户最友好，3 亿+ MAU）— 品牌获客最重要的入口
- MCP Server 元数据完善（工具描述精准，让 Agent 能自动匹配用户意图）

### 2.2 首次使用体验（转化时刻）

品牌第一次对 Agent 说出达人需求，30 秒内收到结构化清单——这个体验要远超手动搜索 3-5 天的现状。

```
用户："帮我找 10 个美国美妆类 TikTok 达人，粉丝 5-20 万，互动率 > 3%"

Agent 返回（30 秒内）：
┌──────────────────────────────────────────────────┐
│  达人候选清单（10/10 符合条件）                     │
│                                                    │
│  1. crt_7fd91a — TikTok / US / beauty            │
│     粉丝区间：10-20 万, 互动率 4.2%, 可信度：高      │
│     预估合作费: $800-1,200/条                       │
│                                                    │
│  2. crt_20b4c3 — TikTok / US / makeup            │
│     粉丝区间：5-10 万, 互动率 5.1%, 可信度：高       │
│     预估合作费: $500-800/条                         │
│     ...                                            │
│                                                    │
│  需要我分析其中某位达人的详细资料吗？               │
└──────────────────────────────────────────────────┘
```

**对比**：手动方式 = 3-5 天 + 打开 5 个网站 + 导出 Excel + 人工交叉验证。NoxInfluencer Day 1 先返回内部 `creator_id` 和不可外链摘要，把低价搬库的空间压缩掉；只有更高价的 `basic_info` 搜索或进一步调 `analyze_creator` 才返回频道链接和可外部关联信息。

### 2.3 留存驱动力

| 驱动力 | 说明 | 对应场景 |
|--------|------|---------|
| **搜索结果可信** | 搜到的不是一堆外链，而是可继续分析的高相关候选 | 搜索 |
| **深度评估有门槛** | 真实性、受众、链接、趋势都在 analyze 里，品牌觉得"值" | 评估 |
| **持续监控形成复用** | 同一批达人需要反复跟踪频道表现，不是一次性查询 | 监控 |
| **越用越准** | Agent 记住品牌偏好、历史分析对象和 watchlist | 记忆层 |

### 2.4 付费模式：并入聚星现有套餐

2026-03-20 起，NoxInfluencer 不再走独立 `credit` 售卖，而是作为**聚星主产品内的 Skill 技能能力**接入现有套餐。品牌用户不再理解“买多少 credit”，而是理解“当前套餐给了我多少 Skill 技能额度，以及对应服务还能不能用”。

#### 套餐承接结构

| 层级 | 状态 | Skill 技能使用口径 |
|------|------|----------------|
| **未登录** | 不可用 | 不能使用任何 Skill 能力 |
| **免费用户** | 一次性试用 | 搜索 10 次，`analyze_creator` 30 次，`track_performance` 不开放，`manage_campaigns` 可用 |
| **AI创业版** | 月付（仅英文站 / 海外承接） | 开放基础 Skill 能力；Skill 技能额度较低；服务配额较低；无多租户 |
| **企业版** | 季付起 | 开放更多 Skill 能力；Skill 技能额度更高；服务配额更高；支持多租户 |
| **高级企业版** | 年付 | 开放更高 Skill 能力；Skill 技能额度更高；服务配额更多 |
| **定制版** | 定制合同 | 按客户规模和场景定制 |

> 付费档差异同时看两层：一是 `Skill 技能额度` 规模，二是可用能力与底层服务配额规模。中文站不开放新的低价 `AI创业版`，默认走企业销售路径；英文站保留 `AI Startup` 月付承接。

#### 双配额规则

Skill 调用采用**双配额校验**：

1. 先校验 `Skill 技能额度`
2. 再校验该操作实际会消耗的底层服务配额
3. 两者都有余量，调用才成功
4. 任一不足，立即拦截，并明确提示缺的是哪一类额度

> 当前口径：`discover_creators`、`analyze_creator`、`track_performance.start` 暂按**每次调用消耗 1 次 Skill 技能额度**处理；`track_performance.query` 与 `manage_campaigns` 不消耗 Skill 技能额度。底层服务配额的具体映射以实际发生为准，由平台策略控制，不在用户侧写死。

#### 免费用户的试用旅程

```
品牌注册聚星账号
  → 获得一次性免费试用
  → discover_creators 最多 10 次
  → analyze_creator 最多 30 次
  → manage_campaigns 可免费记录上下文
  → 想继续使用 / 想用监控 / 底层服务配额不足
  → Agent 提示升级
  → 跳转对应站点 pricing 页
  → 中文站默认进入企业销售路径；英文站可进入 AI Startup 或更高档位
```

#### 升级承接

| 触发条件 | Agent / 页面提示 |
|---------|----------------|
| `Skill 技能额度` 用尽 | 明确提示“Skill 技能额度不足，请升级套餐” |
| 底层服务配额不足 | 明确提示“对应服务配额不足，请升级套餐” |
| 该能力在当前层级为 0 配额 | 正常可点，但点击后提示“当前套餐未包含该能力，请升级” |

#### 当前体验设计

| 设计点 | 选择 |
|------|------|
| 免费试用周期 | **一次性**，不按月重置 |
| 付费用户 Skill 技能额度 | **按月重置** |
| AI创业版周期 | **月付（仅英文站 / 海外承接）** |
| 企业版周期 | **季付起** |
| 高级企业版周期 | **年付** |
| 定制版承接 | 销售团队 |
| 额外加量包 | 暂不自助售卖；有需要时联系销售 |

> 假设 H9：有限免费试用 + 明确拦截提示，能驱动免费用户向主站套餐升级。上线后通过免费→升级漏斗验证。

## 三、核心使用场景

### 3.1 P0：达人发现（搜索+筛选+评估）

**痛点**：人工搜索 3-5 天，假粉泛滥（49% IG 达人有欺诈），受众画像需多源验证。

**交互示例**：

```
用户：帮我找欧洲健身类 Instagram 达人，粉丝 1-10 万，最好经常发 Reels

Agent：找到 15 位符合条件的达人候选，按互动率排序：

 #  creator_id   平台    粉丝区间   互动率   Reels占比   国家
 1  crt_91af2d   IG      5-10万     6.3%    78%        德国
 2  crt_41be88   IG      5-10万     5.8%    65%        英国
 3  crt_a74d10   IG      1-5万      5.2%    82%        荷兰
 ...

用户：分析一下 crt_91af2d 的数据真实性

Agent：
  频道链接：instagram.com/fitvibes_de
  数据真实性：✅ 可信（87/100）
  - 粉丝增长自然，无突增  - 92% 真实互动  - 评论质量高
  受众：女性 72%, 25-34 岁 48%, 德国 45%
  品牌契合度：与你目标受众重合约 62%，高度推荐
```

**NoxInfluencer 提供**：`search_creators`、`get_creator_profile`、`check_authenticity`、`get_audience`

> 注：以上为场景级能力名称。工程实现中，这些能力被合并为 `discover_creators` + `analyze_creator` 两个 Tool。`discover_creators` 默认只返回内部 `creator_id` 和不可外链摘要；若显式使用更高价的 `basic_info` 档位，也可返回 URL / 外部 ID 等可关联字段。完整深度数据仍在 `analyze_creator` 返回（详见 03 第三节 Tool 定义和 §7.3 能力覆盖映射）。

### 3.2 P2：邀约触达

**痛点**：占 22% 人工时间。需发 333+ 封邮件才能激活 50 个达人。联系方式覆盖率仅 25-40%，手动找邮箱极耗时。**大规模邮件触达能力在市场上稀缺**，且需要数据支撑——哪些达人邮箱已验证、历史响应率如何、是否有合作意向。

少量邀约可走品牌自己邮箱，大规模邀约必须通过 NoxInfluencer。

**交互示例**：

```
用户：帮我邀约刚才找到的 12 位可联系达人，推广我们新出的蛋白粉

Agent：已准备邀约方案：

  可直接邀约：12 位（邮箱已验证）
  预估响应率：15-25%（基于历史数据和达人活跃度）
  预计回复：2-3 位

  邮件内容预览（已根据每位达人定制）：
  ──────────────────────────
  To: @fitvibes_de
  Subject: Collaboration with [Brand] — Protein Powder Launch 🏋️

  Hi [Name],
  Love your Reels about home workouts — especially the 30-day challenge
  series (2.1M views!). We're launching a plant-based protein powder
  for the EU market and think your audience is a perfect fit.

  Would you be interested in a gifted collaboration + paid Reel?
  Budget range: €600-900 per Reel.

  Looking forward to hearing from you!
  ──────────────────────────

  确认发送吗？还是需要调整邮件内容？

用户：发送吧

Agent：已发送 12 封邀约邮件。我会跟踪回复情况：
  - 送达确认后通知你
  - 有回复时立即告知
  - 3 天无回复的自动发 follow-up（你可以关闭）
```

**NoxInfluencer 提供**：`get_contacts`（联系方式+可达性评分）、`send_outreach`（大规模个性化邮件发送）、`track_responses`（响应追踪+自动跟进）

### 3.3 P2：谈判协商

**痛点**：占 33% 人工时间——全链路最大的单项成本。平均每个达人 6 轮邮件往返，87.9% 营销人员说谈判越来越难，58% 达人要求混合报酬（底价+分成）。目前**全市场仅 3 家新公司**（Aha/Janney AI/Stormy AI）在做自动谈判，极度稀缺。

**交互示例**：

```
用户：@fitvibes_de 回复了，要价 €1,200 一条 Reels，帮我谈

Agent：
  分析：
  - 市场基准：同量级欧洲健身达人 Reels 中位数 €750
  - 该达人要价高于中位数 60%，但互动率也高于中位数 1.4x
  - 合理区间：€700-900
  - 建议策略：先提 €700 + 产品赠送，目标成交 €800-850

  我来代你谈判吗？预算上限设多少？

用户：上限 €900，尽量谈到 €800

Agent：收到，开始谈判。我会：
  1. 回复达人，提出 €700 + 产品赠送的初始报价
  2. 根据达人回复在 €700-900 区间内灵活协商
  3. 每轮谈判进展实时同步给你
  4. 达成一致后发合作确认邮件（你审核后发出）

  ───── 谈判进行中 ─────
  Round 1: 我方出价 €700 + 产品 → 达人还价 €1,000
  Round 2: 我方 €800 + 产品 + Story 加推 → 达人还价 €900
  Round 3: 我方 €850 含 1 Reel + 2 Stories → 达人接受 ✅

  谈判完成：€850（含 1 条 Reel + 2 条 Story），比要价降 29%
  需要我发送合作确认邮件吗？
```

**NoxInfluencer 提供**：`get_pricing_benchmark`（市场定价基准）、`negotiate`（AI 自动谈判，在预算范围内代品牌与达人邮件往返）、`get_negotiation_history`（谈判记录）

**竞品对标**：Janney AI 案例——AI 将 $3,500 谈到 $2,000（4 封邮件，降幅 43%）

### 3.4 P1：CRM 与达人关系管理

**痛点**：合作过的达人散落在邮件和表格里，无法系统追踪。

**交互示例**：

```
用户：我之前合作过哪些健身达人？效果怎么样？

Agent：
  你的健身达人合作历史（共 8 位）：

  复投推荐（效果好 + 关系良好）：
  ✅ @fitvibes_de — 合作 3 次, 平均互动率 5.8%, 最近合作 2 周前
  ✅ @strongandsara — 合作 2 次, 平均互动率 4.2%, 回复速度快

  观望：
  ⚠️ @gym_daily — 合作 1 次, 互动率 2.1%（低于预期）

  黑名单：
  ❌ @fake_fit — 合作后发现假粉率 38%，已标记

  需要对推荐达人发起新一轮邀约吗？
```

**NoxInfluencer 提供**：`get_collaboration_history`（合作历史）、`update_creator_status`（达人白名单/黑名单/备注）

> **Day 1 / v1.1 分期**：Day 1 的 `manage_campaigns` 仅支持 Campaign 创建 + 查询（CRM 只读）。合作历史查询、达人黑白名单等写操作在 v1.1 `manage_campaigns` 增强版中加入。

### 3.5 P2：主动提醒与监控

**痛点**：关键节点靠人记，容易漏。

**能力清单**：

| 提醒类型 | 场景 | 触发条件 |
|---------|------|---------|
| 合作到期 | 合同快到期，是否续约 | 合同到期前 7 天 |
| 内容发布 | 达人发布了合作内容 | 检测到赞助内容上线 |
| 数据异动 | 达人粉丝骤降或互动率异常 | 偏离基线 > 30% |
| 邀约跟进 | 邮件无回复 | 发送后 3 天无响应 |
| 竞品动态 | 竞品与特定达人开始合作 | 检测到新赞助内容 |

**NoxInfluencer 提供**：`set_alert`（配置提醒规则）、`get_alerts`（获取触发的提醒列表）

**交互示例**：

  ⬜ 审稿    等待达人提交内容初稿
  ⬜ 发布    约定 2/25 发布
  ⬜ 结算    内容上线后 30 天结算

  下一步：产品送达后提醒达人确认收货

**NoxInfluencer 提供**：`get_campaign_status`（合作状态追踪）、`update_campaign_stage`（推进合作阶段）

### 3.7 P1：竞品对标

**痛点**：不知道竞品在跟谁合作、效果如何。

**交互示例**：

## 四、不同品牌规模的差异化路径

### 4.1 SMB/DTC（`￥290-990 / $29-99 / 月`）

- **现状**：Excel + 手动搜索 + 零散截图记录
- **发现路径**：Agent 自动推荐（路径 A）
- **核心价值**：用低门槛搜索拿到候选名单，再用 analyze/track 做更深决策
- **典型用法**：每月筛 50-200 个候选达人，深度分析 20-50 个，持续跟踪 10-30 个频道

### 4.2 中型品牌（`￥990-1,990 / $99-199 / 月`）

- **现状**：已用 SaaS 工具（Modash/AspireIQ），但搜索质量、历史跟踪和跨平台比较仍靠人
- **发现路径**：内容营销（路径 C）或代理商推荐（路径 B）
- **核心价值**：比传统 SaaS 更轻，且对 Agent 更友好；搜索、分析、监控是一套统一能力层
- **典型用法**：团队共享 watchlist，持续追踪重点频道，按月做复盘和竞品观察

### 4.3 大型品牌/Agency（定制方案）

- **现状**：GRIN/Impact 全套 + 专职团队
- **发现路径**：IT/开发者配置（路径 B）
- **核心价值**：把高质量达人搜索、深度详情和监控能力嵌入现有系统
- **典型用法**：API 集成到内部系统，做内部 candidate pool、watchlist 和效果复盘

## 五、关键假设与风险

### 5.1 定位演进风险

当前定位已进一步收敛为"搜索+详情+监控优先"。这意味着：
- **商业化更可控**：搜索先做低价候选暴露，真正高价值数据在 `basic_info` / analyze / track 中付费解锁
- **防搬数更可执行**：普通搜索不返回可外链字段，只有更高价档位才开放
- **技术复杂度更可控**：Day 1 不必先背邮件发送、自动谈判、复杂流程编排的工程包袱

需要在 01/03/04/05 中同步保持这一口径。

### 5.2 技术门槛假设

品牌大概率不能自助安装，因此 **Agent 自动发现是唯一可规模化的品牌获客路径**。

落地条件：
- ChatGPT Apps 上架后被推荐系统覆盖
- MCP 工具描述精准匹配"influencer/KOL/creator marketing"意图
- 各平台达人营销品类注册为首选工具

### 5.3 搜索分级返回假设

Day 1 的核心假设不是"用户需要拿到一堆外链"，而是：
- 品牌愿意先用内部 `creator_id` 做筛选，再付费看 `basic_info` 或深度详情
- 非外链摘要已经足够支撑搜索质量判断
- 默认情况下 `analyze_creator` 承担主要深度数据出口；只有显式选择 `basic_info` 搜索时，搜索阶段才暴露有限外链字段

### 5.4 监控能力假设

把频道/达人效果监控上调到 P0，需要验证：
- 品牌是否会为持续 watchlist 付费，而不只是一次性查详情
- `track_performance` 的返回是否足够支撑复投和 ROI 讨论
- 哪些监控维度最有价值：粉丝趋势、播放趋势、互动率、发帖频率、异常波动

**验证方式**：上线后跟踪 watchlist 建立率、重复监控调用率和复盘场景占比。

### 5.5 平台演化假设

不押注任何一家 Agent 平台。一套核心服务，多个接入层：

| 平台 | 品牌友好度 | 自动发现支持 | 优先级 |
|------|:---------:|:----------:|:-----:|
| ChatGPT Apps | 最高 | 推荐系统 | 品牌获客 P0 |
| OpenClaw/ClawHub | 低 | Skill 自动匹配 | 技术分发 P0 |
| Claude Code | 低 | MCP 自动发现 | 技术分发 P0 |
