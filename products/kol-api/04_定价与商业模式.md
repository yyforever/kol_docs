## 限量与防竞品机制

这是杨洋特别关注的点。开放API = 竞品可以搬数据。

### 防御策略

| 层级 | 机制 | 说明 |
|------|------|------|
| **L1 调用量限制** | 每tier有硬上限 | Free 1K、Starter 10K、Pro 50K |
| **L2 rate limit** | 10-50 req/s（按tier） | 防止高速批量抓取 |
| **L3 使用模式监控** | 检测异常模式 | 连续遍历ID、覆盖面过广、无搜索直接枚举 |
| **L4 数据混淆** | 免费层不返回精确数据 | 如粉丝数返回"约15K"而非15,234 |
| **L5 合同条款** | ToS禁止数据转售/批量存储 | 法律兜底 |
| **L6 人工审核** | Enterprise客户需审核 | 已知竞品公司拒绝服务 |

### 最敏感的数据

| 数据类型 | 防护级别 | 理由 |
|---------|:-------:|------|
| 联系方式（邮箱） | 🔴 最高 | 直接可被竞品利用 |
| 受众画像 | 🟠 高 | 独家分析数据 |
| 假粉检测 | 🟠 高 | 算法成果 |
| 基础搜索结果 | 🟡 中 | 搜索体验是获客钩子 |
| 公开指标（粉丝数等） | 🟢 低 | 公开信息 |

## kol-agent内部调用

kol-agent（路线2）是最大的API消费者，但**不走公开定价**：

| 方案 | 说明 |
|------|------|
| 内部API Key | 无调用量限制、无付费 |
| 成本核算 | 按实际调用量计入kol-agent成本 |
| 能力同步 | kol-agent需要的endpoint优先开发 |

这样kol-api对外是产品，对内是基础设施。
