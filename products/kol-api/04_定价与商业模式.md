# 04 定价与商业模式

> 状态：已同步
> 更新：2026-03-20
> 依赖：`01_定位与假设.md`、`02_用户场景.md`、`03_API能力设计.md`
> 本文回答：**Skill 技能如何并入聚星现有套餐？怎么做配额治理和防搬数？**

---

## 一、结论

2026-03-20 起，`kol-api` 不再按独立产品线和 `credit` 模式售卖，而是改为**聚星主产品下的 Skill 技能能力专题**。商业化不再追求单独卖 API 调用，而是通过 Skill 能力：

- 增强聚星现有套餐价值
- 驱动免费用户升级
- 驱动企业用户从低配套餐升到更高配套餐
- 在不放开敏感数据批量搬运风险的前提下，让 Agent 使用路径成立

**核心口径只有一句话**：
用户每次调用 Skill 时，必须同时满足：

1. 当前账号有可用的 **Skill 技能额度**
2. 当前账号有可用的 **对应服务配额**

任一不足，就拦截，并明确提示缺的是哪一类额度。

## 二、收入模型

### 2.1 从独立计费改为主产品承接

| 维度 | 旧口径（已废弃） | 新口径（当前） |
|------|----------------|--------------|
| 商业形态 | 独立产品线 | 聚星主产品下的能力专题 |
| 计费单位 | Credit | Skill 技能额度 + 服务配额 |
| 价格承接 | 独立 pricing + Stripe | 聚星主站 pricing 页 |
| 升级动作 | 买更多 credits | 升级主站套餐 |
| 自助加量 | 可想象为超额 credit | 先不做；额外需求联系销售 |
| 免费策略 | 10 credits 一次性 | 一次性试用：10 次搜索 + 30 次查看 |

### 2.2 套餐结构

> 价格金额沿用主站现有口径；本文聚焦结构和配额逻辑。若主站价格变动，以主站 pricing 页为准。

| 层级 | 周期 | 多租户 | Skill 技能使用口径 |
|------|------|------|----------------|
| **未登录** | — | 否 | 不可用 |
| **免费用户** | 一次性试用 | 否 | 搜索 10 次、`analyze_creator` 30 次、`track_performance` 不开放、`manage_campaigns` 可用 |
| **AI创业版** | 月付（仅英文站 / 海外承接） | 否 | 开放基础 Skill 能力；Skill 技能额度较低；服务配额较低 |
| **企业版** | 季付起 | 是 | 开放更多 Skill 能力；Skill 技能额度更高；服务配额更高 |
| **高级企业版** | 年付 | 是 | 开放更高 Skill 能力；Skill 技能额度更高；服务配额更多 |
| **定制版** | 定制合同 | 是 | 按客户规模和场景定制 |

### 2.2.1 付费档差异表达

付费档差异不是只看 `Skill 技能额度`，而是同时看两层：

1. `Skill 技能额度` 规模
2. 可用能力范围与底层服务配额规模

当前对外结构应表达为：

| 层级 | Skill 技能额度 | 可用能力 / 服务配额 | 承接渠道 |
|------|---------------|--------------------|---------|
| **AI创业版** | 较低 | 基础 Skill 能力 + 较低服务配额 | 英文站 / 海外月付入口 |
| **企业版** | 更高 | 更多 Skill 能力 + 更高服务配额 + 多租户 | 主站企业销售路径 |
| **高级企业版** | 更高 | 更高 Skill 能力 + 更多服务配额 + 多租户 | 主站企业销售路径 |
| **定制版** | 合同约定 | 能力与配额按客户规模定制 | 销售团队 |

> 中文站不开放新的低价 `AI创业版`，默认承接企业版及以上销售路径；英文站保留 `AI Startup` 作为月付入门档。

### 2.3 Skill 技能额度规则

| 规则 | 当前口径 |
|------|---------|
| 展示名称 | **Skill 技能额度** |
| 扣减单位 | 每次 Tool 调用 |
| Day 1 口径 | `discover_creators`、`analyze_creator`、`track_performance.start` 暂按每次调用扣 1 次；`track_performance.query` 与 `manage_campaigns` 不扣 |
| 重置周期 | 付费用户按月重置 |
| 免费用户 | 一次性试用，不重置 |
| 归属对象 | 主账号共享 |

> `manage_campaigns` 当前定位为辅助能力，不是卖点，也不受 Skill 技能额度限制。产品上不展示数量上限或保留期，技术侧可设置内部安全上限。

### 2.4 服务配额规则

Skill 技能额度不是唯一门槛。真正执行时，还要校验底层服务配额。

| 规则 | 当前口径 |
|------|---------|
| 校验方式 | 调用发生时按实际操作校验 |
| 公开粒度 | 不在用户侧写死每个 Tool 必然消耗哪一项服务 |
| 平台实现 | 以平台策略定义为准 |
| 典型场景 | 搜索、查看、监控等服务分别由对应服务配额约束 |

> 当前明确口径：**具体服务配额消耗以实际发生为准**。因此，用户文档只强调“双配额”，不把服务映射写死为长期合同。

### 2.5 免费试用策略

| 能力 | 免费用户 |
|------|---------|
| `discover_creators` | 10 次 |
| `analyze_creator` | 30 次 |
| `track_performance` | 0 |
| `manage_campaigns` | 可用，不限额 |

设计原则：

- 免费用户必须能感知到搜索和评估价值
- 免费用户不开放监控，避免形成高成本的长期占用
- 免费用户不支持多租户
- 入口不隐藏，限制在点击后明确提示
- `manage_campaigns` 产品上不展示额度或保留期限制，技术侧只保留内部安全上限

### 2.6 升级与拦截

| 场景 | 系统动作 | 承接页面 |
|------|---------|---------|
| 未登录点击 Skill | 拦截并提示登录 | 登录 / 注册页 |
| Skill 技能额度不足 | 拦截并提示“Skill 技能额度不足” | pricing 页 |
| 服务配额不足 | 拦截并提示“对应服务配额不足” | pricing 页 |
| 当前套餐该能力为 0 配额 | 拦截并提示“当前套餐未包含该能力” | pricing 页 |

当前不做：

- 自助加量包
- Skill 单独付费页
- Skill 内直接支付闭环

如需额外单独购买，统一由销售团队承接。

> 区域承接口径：中文站 pricing 页默认只承接企业版 / 高级企业版 / 定制版；英文站可承接 `AI Startup` 月付档。

## 三、内部价值约束

虽然用户侧不再看到 `credit`，但平台仍需维护一套**内部数据价值下限**，避免通过套餐和配额把真实数据卖穿。

### 3.1 内部数据价值下限

| 数据 / 能力 | 内部下限 |
|------------|---------|
| 搜索基础数据（隐藏关键字段） | `￥0.05 / creator` |
| 搜索含基础信息 | `￥0.20 / creator` |
| 深度数据全维度 | `￥1.40 / creator` |
| 邮箱解锁 | `￥2.0 / creator` |
| 内容监控（15 天） | `￥1.5 / creator / 15 天` |

### 3.2 作用

这套下限不直接展示给用户，但会约束三件事：

1. 套餐的 Skill 能力开放边界
2. 各层级可承载的服务配额规模
3. 销售侧特殊折扣、定制报价和大客户额外授权

结论：**用户侧取消 credit，不等于内部放弃价值约束。**

## 四、Unit Economics

### 4.1 每类能力的边际成本

> H4 已验证：聚星假粉检测和受众画像三平台已有，内部调用成本极低（<$0.001/次），无需第三方达人数据 API。

| 能力 | 主要成本组成 | 估算单次成本 | 商业含义 |
|------|------------|:----------:|---------|
| 搜索 | 聚星查询 + 轻量 AI 解析 | $0.001-0.004 | 低成本入口，适合免费试用 |
| 深度评估 | 聚星查询 + 假粉检测 + 受众画像 + AI 总结 | $0.003-0.010 | 适合作为核心升级驱动力 |
| 监控启动 | 趋势拉取 + benchmark 计算 | $0.005-0.012 | 长期留存能力，必须收紧免费层 |
| `manage_campaigns` | 数据库读写 | $0.001-0.003 | 适合作为低摩擦辅助能力 |
| 联系方式 / 邀约 / 谈判 | 查询 + 邮件 + 多轮 AI 推理 | $0.01+ | 继续保留高门槛能力边界 |

### 4.2 为什么新模型仍然成立

- Skill 能力仍然建立在聚星自有数据上，边际成本低
- 免费层只开放搜索和评估的有限试用，不开放监控
- `manage_campaigns` 免费但成本低，能提升长期留存
- 真正敏感和高价值的数据，仍由服务配额和套餐层级控制

结论：**即使取消 credit 售卖，Skill 作为套餐增强能力仍然具备健康的单位经济学。**

## 五、数据保护（防搬策略）

### 5.1 威胁模型

| 威胁者 | 动机 | 手段 | 严重度 |
|--------|------|------|:------:|
| 直接竞品 | 获取聚星独有数据（假粉、受众画像、关键字段） | 注册账号 + 批量调用 | 🔴 |
| 数据经销商 | 低价获取并转售 | 自动化脚本批量抓取 | 🔴 |
| Agent 自动化滥用 | 链式调用 discover → analyze → 导出 | 自动化高频调用 | 🟠 |
| 白嫖用户 | 反复注册免费账号 | 一次性试用刷号 | 🟡 |

### 5.2 当前防线

#### L1：双配额防线

以前的经济防线是 `credit`，现在改成：

- 套餐边界
- Skill 技能额度
- 服务配额

三者叠加后，依然能把免费和低配套餐的可搬运能力压得很低。

#### L2：数据分级返回

| 数据字段 | 默认搜索 | 深度评估 | 说明 |
|---------|:-------:|:-------:|------|
| `creator_id` | ✅ | ✅ | 内部主键 |
| 平台 / 国家 / 品类 / 粉丝区间 | ✅ | ✅ | 可用于粗筛 |
| 频道链接 / 外部 ID / `handle` | 受控 | ✅ | 敏感字段继续门控 |
| 受众画像 | ❌ | ✅ | 深度能力 |
| 内容分析 | ❌ | ✅ | 深度能力 |
| 联系方式 | ❌ | 独立能力 | 不在 Day 1 免费层开放 |

#### L3：限流与异常检测

继续保留：

- 每分钟 / 每小时请求数限制
- 并发限制
- 单 IP 限额
- 搜索与分析比例异常检测
- 多账号同 IP 关联检测

#### L4：商业和法律兜底

继续保留：

- 已知竞品屏蔽
- KYB / 企业身份校验（企业版及以上）
- ToS 禁止转售和批量存储
- 审计权和违约处理

### 5.3 免费层的特殊策略

| 策略 | 目的 |
|------|------|
| 仅一次性试用 | 防止长期白嫖 |
| 不开放监控 | 控制高成本留存能力 |
| 不支持多租户 | 防止团队级免费搬运 |
| 入口可点但提示升级 | 让用户理解价值，再引导升级 |

## 六、验证重点与风险

### 6.1 关键验证点

| 假设 | 验证方式 | 时间 | 成功标准 |
|------|---------|------|---------|
| 免费试用能触发升级意愿 | 免费用户试用后的升级率 | M1-M3 | 免费→付费升级率达到可接受区间 |
| “Skill 技能额度不足”提示能驱动升级 | 触发拦截后的点击率与升级率 | M1-M3 | 主要限制场景有明显升级动作 |
| 双配额模型用户能理解 | 支持工单、拦截后二跳行为 | M1 | 理解成本可控 |
| Skill 能提升主套餐价值 | 升级率、留存率、活跃度 | M3 | 高于无 Skill 对照组 |

### 6.2 关键风险

| 风险 | 严重度 | 应对 |
|------|:------:|------|
| 用户不理解“双配额” | 高 | 提示语统一为“Skill 技能额度不足”或“服务配额不足” |
| 免费层过于宽松，被滥用 | 高 | 一次性试用 + 不开放监控 + 不支持多租户 |
| 免费层过于严格，感知不到价值 | 中 | 保证 10 次搜索 + 30 次评估足以体验核心价值 |
| 服务映射过于复杂 | 中 | 用户侧不写死映射，平台内部按实际发生计 |
| 企业版口径与主站实际销售口径不一致 | 中 | 对外以主站 pricing 页为准，文档只写结构不写死金额 |

## 七、设计决策记录

| # | 决策 | 选择 | 备注 |
|---|------|------|------|
| D1 | 产品边界 | `kol-api` 不再独立售卖 | 继续留在 `products/kol-api/`，但改为能力专题 |
| D2 | 用户侧名称 | Skill 技能 | 对外统一这样表述 |
| D3 | 套餐结构 | 未登录 → 免费 → AI创业版 → 企业版 → 高级企业版 → 定制版 | `AI创业版` 仅英文站 / 海外承接 |
| D4 | Skill 额度扣减 | `discover/analyze/track.start` 各 1 次 | `track.query` 和 `manage_campaigns` 不计额度 |
| D5 | 免费试用 | 搜索 10 次 + `analyze_creator` 30 次 | 一次性，不重置 |
| D6 | 多租户边界 | 企业版起支持 | 免费和 AI创业版不支持 |
| D7 | 升级路径 | 统一跳 pricing 页 | 额外购买走销售 |
| D8 | 服务配额映射公开粒度 | 不写死 | 以实际发生为准 |
| D9 | 付费档差异 | 同时按 Skill 技能额度与可用能力 / 服务配额区分 | 不只按单一额度解释 |
