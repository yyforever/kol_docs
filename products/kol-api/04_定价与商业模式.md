## 一、收入模型

### 1.1 两层定价逻辑：对外 Credit、对内风控下限

**核心机制**：对外仍以 `Credit` 为主计费单位；对内同时维护一套以"每个 creator / 每个维度 / 每个 15 天周期"为单位的风控下限。
Credit 价格、套餐折扣和赠送额度都要参考这套下限，确保**自助层的实际成交价不低于数据被低价搬走的风险线**。

> 货币口径统一使用购买力换算：`￥1 = $0.1`。这不是实时汇率，而是 2026-03-10 的内部定价换算标准。

| 层 | 作用 | 当前口径 |
|------|------|------|
| **对外计费层** | 品牌实际看到和购买的价格 | `Credit` + 月度套餐 + 超额按量 |
| **对内风控层** | 校验搜索、详情、监控是否被低价出售 | 按 `creator / 维度 / 15 天` 的人民币 / 美元下限 |

### 1.2 对外 Credit 映射（自助版）

| 能力 | Credit 消耗 | 说明 |
|------|:----------:|------|
| `discover_creators`（默认 `protected`） | 1 credit / 页 | 1 次返回最多 20 个候选达人，仅返回内部 `creator_id` 和不可外链摘要 |
| `discover_creators`（`basic_info`） | 4 credits / 页 | 同样最多 20 个候选达人，但返回可外部关联的基础信息；不是默认档位 |
| `analyze_creator` - 数据总览 | 1 credit | 返回频道链接 / 外部频道 ID / `handle` / 基础表现 |
| `analyze_creator` - 粉丝受众 | 2 credits | 返回受众国家、性别、年龄等画像 |
| `analyze_creator` - 内容分析 | 1 credit | 返回近期内容表现和结构化结论 |
| `analyze_creator` - 品牌分析 | 1 credit | 返回品牌契合度、商业合作线索和品牌侧判断 |
| `analyze_creator`（全维度） | 5 credits | 四个维度一起返回 |
| `track_performance` | 3 credits / creator / 15 天 | 监控单个频道 15 天内的效果变化，形成 watchlist 留存 |
| `manage_campaigns`（创建 / 查询） | 0 credit | 免费上下文层，不对写入 / 查询收费 |
| `unlock_email`（后续版本） | 2 credits / creator | 解锁邮箱是独立收费项，不包含在普通 analyze 中 |
| `competitive_intel`（v1.1） | 10 credits / 次 | 高信息密度洞察，聚合多个 creator / brand 信号 |
| `outreach_creators`（v1.2） | 10 credits / creator | 执行自动化能力，包含邮件生成和触达动作 |
| `negotiate`（v1.2） | 5 credits / 轮 | AI 代谈判，每轮单独计费 |

**关键定义**：`基础信息` 指**可关联到外部平台的信息**，如频道 URL、外部频道 ID 等。默认搜索不返回这些字段；只有 `discover_creators.basic_info` 或 `analyze_creator` 才会返回。

> `discover_creators.basic_info` 不是一个新 Tool，而是同一搜索能力下的更高价结果档位。大批量基础信息拉取（特别是 10 万 creator 以上）默认不走自助价，转 Enterprise 风控。

### 1.3 内部风控下限（人民币 / 美元）

> 这部分是内部定价约束，不是面向外部用户展示的官网价格。作用是：确定 Credit 消耗和套餐折扣时，不能把真实数据价值卖穿。

#### 搜索基础数据

| 档位 | 下限单价 | 说明 |
|------|---------|------|
| 搜索基础数据（隐藏关键字段） | `￥0.05 / $0.005 / creator` | 默认搜索结果，不返回 URL / 外部 ID 等可关联字段 |
| 搜索含基础信息 | `￥0.20 / $0.020 / creator` | 返回可外部关联字段；自助版仅开放受控场景 |
| 搜索含基础信息（10 万 creator 以上） | `￥0.40 / $0.040 / creator` | 超大批量场景按更高风控线处理 |

**量级示例**：

| 量级 | 隐藏关键字段 | 含基础信息 |
|------|-------------|-----------|
| 20 个（1 次默认搜索） | `￥1 / $0.1` | `￥4 / $0.4` |
| 10,000 个 | `￥500 / $50` | `￥2,000 / $200` |
| 100,000 个 | `￥5,000 / $500` | `￥20,000 / $2,000` |

#### 深度数据（4 个维度）

| 维度 | 下限单价 |
|------|---------|
| 数据总览 | `￥0.30 / $0.030 / creator` |
| 粉丝受众 | `￥0.50 / $0.050 / creator` |
| 内容分析 | `￥0.30 / $0.030 / creator` |
| 品牌分析 | `￥0.30 / $0.030 / creator` |
| **全维度合计** | **`￥1.40 / $0.140 / creator`** |

**量级示例**：

| 量级 | 全维度深度数据 |
|------|---------------|
| 1 个 | `￥1.4 / $0.14` |
| 10,000 个 | `￥14,000 / $1,400` |
| 100,000 个 | `￥140,000 / $14,000` |

#### 其他高价值数据

| 项目 | 下限单价 | 量级示例 |
|------|---------|---------|
| 邮箱解锁 | `￥2.0 / $0.20 / creator` | 10,000 个 = `￥20,000 / $2,000`；100,000 个 = `￥200,000 / $20,000` |
| 内容监控（15 天） | `￥1.5 / $0.15 / creator / 15 天` | 100 个 = `￥150 / $15`；1,000 个 = `￥1,500 / $150`；10,000 个 = `￥15,000 / $1,500` |

### 1.4 分层定价（对外）

| 层级 | 月费 | Credits/月 | 超额单价 | 目标用户 | 说明 |
|------|------|:---------:|---------|---------|------|
| **Free** | `￥0 / $0` | 10（一次性） | 不可超 | 首次体验 | 够跑 1 次搜索 + 1 次全维度 analyze + 1 个 15 天监控周期 |
| **Starter** | `￥290 / $29` | 200 | `￥1.8 / $0.18` | SMB / DTC 品牌 | 搜索、详情、监控的小团队起步档 |
| **Pro** | `￥990 / $99` | 800 | `￥1.5 / $0.15` | 中型品牌 | 常规研究和批量评估 |
| **Growth** | `￥1,990 / $199` | 1,800 | `￥1.2 / $0.12` | 高频研究团队 / 小型 Agency | 更低边际成本 + 更高并发 |
| **Enterprise** | 定制 | 定制 | 定制 | 大型品牌 / Agency | 大批量基础信息、合同和 SLA |

**年付折扣**：全档约 `17% off`。当前建议口径：

| 层级 | 年付 |
|------|------|
| Starter | `￥2,900 / $290` |
| Pro | `￥9,900 / $990` |
| Growth | `￥19,900 / $1,990` |

**关键设计决策**：

| 决策 | 选择 | 原因 |
|------|------|------|
| 免费额度一次性 vs 每月 | **一次性 10 credits** | 够体验，不够长期低价搬库 |
| 付费墙放在哪 | **基础信息搜索 / 深度分析 / 15 天监控** | 默认搜索只暴露内部 ID，真正可外链或可复用数据在更高价层 |
| 搜索是否允许返回外部信息 | **允许，但不是默认档位** | 同一 Tool 内保留受控 `basic_info` 档，避免低价批量导出 |
| 超额计费 | **按 tier 递减，但不得低于风控下限** | 允许扩容，但不能把下限打穿 |
| 信用卡 | Free 不要求，付费层需要 | 降低注册摩擦，付费时再收卡 |

### 1.6 SaaS 老用户赠送额度

SaaS 订阅和 API / MCP 计费**分开记账**，但现有聚星 SaaS 用户可获得每月赠送额度。赠送额度按标准面值折算后发放，不与 SaaS 套餐费用相互抵扣。

| SaaS 层级 | 赠送口径 | 折算 Credit | 说明 |
|----------|---------|:-----------:|------|
| 企业版 | `￥200 / $20 / 月` | 200 credits / 月 | 单独发到 API / MCP 额度池 |
| 高级企业版 | `￥500 / $50 / 月` | 500 credits / 月 | 单独发到 API / MCP 额度池 |

> 赠送额度只影响可用 credit，不改变 API / MCP 的计费规则、账单和风控阈值。

### 1.8 kol-agent 内部调用（2026-03 补充）

> kol-agent（聚星品牌侧 AI 员工，路线 2）内部调用 Core 层时**不消耗 kol-api credit**。kol-agent 用户付的是 Agent 订阅费（包含 API 使用），内部调用是聚星自有成本，不走 kol-api 的 Credit 体系。
>
> - **区分内外部调用**：Core 层需要在调用来源中标识 `source=kol-agent` vs `source=external`，内部调用不扣 credit
> - **不影响外部定价**：外部 Agent 平台（Claude Desktop / Cursor 等）调用 Core 层仍按 Credit 额度计费
> - **CRM 数据共享**：`manage_campaigns` 承载的 CRM 数据（达人关系、合作历史）kol-agent 和外部 Agent 共用同一套，不因调用来源不同而分裂
>
> 详见 kol-agent `04_定价与商业模式.md` D26。

## 二、付费路径

### 2.1 Agent → 官网 → Stripe（当前唯一验证路径）

**所有成功 MCP 产品用同一套变现模式**：

```
品牌在 Agent 中使用 NoxInfluencer（消耗免费 credit）
        ↓
10 免费 credits 用完 → Tool 返回 upgrade_url
        ↓
品牌去 noxinfluencer.com 注册（Google / GitHub 一键登录，10 秒）
        ↓
获得 API Key → 选择付费套餐 → Stripe 支付
        ↓
API Key 配置到 Agent → 继续使用（月度 credit 自动刷新）
```

**转化漏斗假设**（对标 PLG 基准 3-5%）：

| 阶段 | 数量 | 转化率 | 来源 |
|------|:----:|:-----:|------|
| Agent 安装 | 1,000 | — | 多平台分发 |
| 首次使用 | 600 | 60% | 行业基准 |
| 免费额度用完 | 200 | 33% | 10 credits ≈ 1 次默认搜索 + 1 次全维 analyze + 1 个 15 天监控周期 |
| 注册官网 | 100 | 50% | 注册无摩擦 |
| 付费 | 40 | 40% | 免费已用完，有明确需求才注册 |
| **安装→付费** | — | **4%** | 行业基准 3-5%，取中值 |

> Firecrawl 用一次性免费实现了高于行业基准的转化率（具体数据未公开，但一次性免费层消除了"等下个月重置"的心态）。NoxInfluencer 采用相同策略。

### 2.2 Agent 平台内付费展望（2026 年 2 月实态）

01 和 02 说"目前没有任何成功产品在 Agent 内完成付费"。2026 年 2 月更新：**基础设施已就位，但 SaaS 订阅场景尚无生产级案例。**

| 基础设施 | 进展 | 成熟度 | 对 NoxInfluencer 的影响 |
|---------|------|:------:|----------------|
| **Stripe MCP SDK** | 2025 年 5 月发布，基于 Cloudflare Workers，支持在 MCP Server 内完成支付 | 早期 | 技术可行，但无达人营销品类先例。**可作为 v1.1 实验** |
| **OpenAI Agentic Commerce Protocol** | 已在美国上线，Etsy 卖家可用，Shopify 即将接入，基于 Stripe | 生产级（电商） | 仅限电商交易场景，非 SaaS 订阅。**不适用于 NoxInfluencer** |
| **Visa Intelligent Commerce** | 试点完成，2025 年 11 月美国上线，2026 年初全球扩展 | 早期生产 | 品牌支付可能从信用卡走 Visa Agent Pay，但 NoxInfluencer 仍通过 Stripe 收款。**透明** |
| **Mastercard Agent Pay** | 与 Visa 类似时间线，试点中 | 早期 | 同上 |
| **Google AP2（Agent Payments Protocol）** | 2025 年 9 月发布，60+ 合作伙伴 | 早期 | 大平台入场验证方向，但与 NoxInfluencer 无直接集成 |
| **PayPal + Claude** | 通过 MCP Directory 集成 | 已上线 | PayPal 可作为 Stripe 之外的支付选项 |
| **Masumi Network** | 去中心化 MCP 付费协议（Cardano） | 概念阶段 | **不适用**，过于早期 |

**NoxInfluencer 付费路径决策**：

| 阶段 | 路径 | 理由 |
|------|------|------|
| **Day 1** | 官网 Stripe（唯一路径） | 100% 成功产品验证；品牌广告主习惯网页端付费 |
| **v1.1（上线 3 月后）** | Stripe MCP SDK 实验 | 若 SDK 有生产级案例，试点 Agent 内一键升级 |
| **远期** | OpenAI / Visa Agent Pay 适配 | 跟随大平台节奏，不主动投入 |

### 2.3 Enterprise 销售流程

`￥1,990 / $199 / 月` 以上客户或需要定制的品牌/Agency，走 Enterprise 流程：

```
品牌主动联系（官网 Contact Sales / Agent 内提示）
        ↓
Sales 初步沟通（1-2 call，了解规模和需求）
        ↓
定制方案（credit 量 + 功能 + SLA + 数据范围）
        ↓
合同签署（年付，银行转账或 Stripe Invoice）
        ↓
专属 API Key + 优先支持 + 季度业务回顾
```

**Enterprise 不在 Day 1 范围内**。上线 6 个月后视 Growth 层用户需求启动。

## 三、Unit Economics

### 3.1 每 Tool 服务成本估算

> 基于 H4 验证结论更新（2026-02）：聚星假粉检测和受众画像三平台已有，内部调用成本极低（<$0.001/次），无需第三方 API（Phyllo/HypeAuditor）。这使得 Unit Economics 显著优于初版估算。

| Tool / 能力 | 主要成本组成 | 估算单次成本 | 内部风控下限 | 对外 Credit 映射 | 毛利判断 |
|------|------------|:----------:|:------------:|:---------------:|------|
| `discover_creators.protected` | 聚星查询（<$0.001）+ LLM 解析（~$0.003） | $0.001-0.004 | `￥1 / $0.1`（20 个） | 1 credit / 20 个 | 明显高毛利 |
| `discover_creators.basic_info` | 搜索 + 外链字段暴露 + 更严格限流 | $0.001-0.005 | `￥4 / $0.4`（20 个） | 4 credits / 20 个 | 明显高毛利 |
| `analyze_creator`（全维度） | 聚星查询 + 假粉检测 + 受众画像 + 内容 / 品牌分析 | $0.003-0.010 | `￥1.4 / $0.14` | 5 credits | 高毛利 |
| `track_performance`（15 天） | 趋势拉取 + 聚合 + benchmark 计算 | $0.005-0.012 | `￥1.5 / $0.15` | 3 credits / creator / 15 天 | 高毛利 |
| `manage_campaigns` | 数据库读取 | $0.001-0.003 | — | 0 credit | 记忆层补贴 |
| `unlock_email` | 联系方式查询 + 校验 | $0.002-0.010 | `￥2.0 / $0.20` | 2 credits / creator | 高毛利 |
| `competitive_intel` | 大规模数据聚合 + AI 分析 | $0.03-0.08 | 高于品牌分析单点下限 | 10 credits / 次 | 高毛利 |
| `outreach_creators` | 邮箱解锁 + LLM 邮件生成 + 邮件发送 | $0.01-0.02 | 高于邮箱解锁下限 | 10 credits / creator | 高毛利 |
| `negotiate` | 每轮 LLM 推理 + 邮件往返 | $0.01-0.02/轮 | — | 5 credits / 轮 | 高毛利 |

**成本构成明细**：

| 成本项 | 单次估算 | 说明 |
|--------|:-------:|------|
| 聚星数据查询（搜索/频道/视频） | <$0.001 | 聚星自有数据，内部调用边际成本极低 |
| 聚星假粉检测 | <$0.001 | 聚星已有能力（三平台），内部调用无额外成本 |
| 聚星受众画像 | <$0.001 | YT/TikTok/IG 三平台已有，内部调用无额外成本 |
| AI 推理（单轮） | $0.003-0.01 | GPT-4o-mini ~ $0.003，GPT-4o ~ $0.01，按 output token 计 |
| AI 推理（多轮谈判，3-5 轮） | $0.03-0.10 | 谈判场景是成本最高的操作 |
| 邮件发送（聚星邮件基础设施） | ~$0.001 | 聚星自有邮件通道（Aliyun DirectMail + SendCloud），内部调用边际成本极低 |
| 带宽 / CDN | $0.0005-0.001 | 几乎可忽略 |

**为何 NoxInfluencer 成本结构优于竞品**：

竞品（如基于 Modash API 或 HypeAuditor API 构建的产品）每次调用需支付第三方数据供应商费用（$0.02-0.05/次），而 NoxInfluencer 的数据查询、假粉检测、受众画像全部来自聚星内部能力，边际成本接近零。这意味着：
- **无第三方 API 依赖** = 无外部成本波动风险
- **自有假粉检测** = 核心 IP，竞品需付费采购的能力我们自有
- **三平台受众画像** = 无需等待第三方数据源扩展

### 3.2 各 Tier 毛利率

| 层级 | 月收入 | 估算月成本 | 毛利率 | 说明 |
|------|:------:|:--------:|:------:|------|
| Free | $0 | $0.05-0.20 | — | 获客成本，不计毛利 |
| Starter `￥290 / $29` | $29 | $1-3 | **90-96%** | 默认搜索和全维 analyze 仍显著高于边际成本 |
| Pro `￥990 / $99` | $99 | $4-10 | **90-96%** | 更适合批量 research + 常态监控 |
| Growth `￥1,990 / $199` | $199 | $9-20 | **89-95%** | 规模效应更强，但仍有充足安全边际 |
| Enterprise | $500+ | $30-80 | **84-94%** | 大批量基础信息和定制洞察占比更高 |

**结论：全层级毛利率健康，无负毛利风险。**

**毛利率估算逻辑**：

1. **搜索是低成本入口**：默认 `discover_creators.protected` 维持低门槛获客，但价格仍受每页 20 个 creator 的风控线约束
2. **高价值动作有更高 credit 倍数**：`basic_info` 搜索、全维 analyze、邮箱解锁和 15 天监控都明显高于边际成本
3. **manage_campaigns 免费但成本低**：它是长期记忆层和留存工具，成本很低，对整体毛利影响有限
4. **聚星数据 + 假粉检测 + 受众画像全部自有**：数据查询边际成本接近零，无第三方 API 成本——这是 NoxInfluencer 毛利率显著高于初版估算的根本原因
5. **AI 推理成本持续下降**：GPT-4o-mini 2025 年成本比 GPT-4 2024 年下降 95%

> **SaaS 行业基准**：毛利率 70-80% 为健康，80%+ 为优秀。NoxInfluencer 估算 85-95% 处于优秀区间，主要受益于自有数据（含假粉检测 + 受众画像）+ 低成本操作占比高。

## 四、数据保护（防搬策略）

### 4.1 威胁模型

| 威胁者 | 动机 | 手段 | 严重度 |
|--------|------|------|:------:|
| **直接竞品**（Modash、HypeAuditor 等） | 获取聚星独有数据（假粉评分、受众画像、中国平台数据） | 注册账号 + 批量调用 API | 🔴 致命 |
| **数据经销商** | 低价获取数据转售 | 自动化脚本批量抓取 | 🔴 致命 |
| **Agent 自动化滥用** | Agent 可自主链式调用，无人工审核 | Agent 在循环中调用 discover → analyze → 导出 | 🟠 高 |
| **白嫖用户** | 用完免费额度换账号 | 多 OAuth 账号注册，反复获取 30 免费 credits | 🟡 中 |
| **爬虫/Scraper** | 绕过 API 直接爬官网数据 | 传统爬虫技术 | 🟡 中 |

**最敏感的数据**（优先保护）：

| 数据类型 | 防护级别 | 理由 |
|---------|:-------:|------|
| 频道链接 / 外部 channel id | 🔴 最高 | 一旦低价暴露，竞品可直接回填自己的数据库 |
| 联系方式（邮箱/社媒） | 🔴 最高 | 直接可被竞品利用，是执行自动化能力的核心字段 |
| 真实性评分 + 假粉分析 | 🔴 最高 | 算法成果，聚星核心 IP |
| 受众画像（人口统计） | 🟠 高 | 独家分析数据 |
| 预估合作费用 | 🟠 高 | AI 推算的市场基准 |
| 基础搜索结果（内部 `creator_id` + 摘要） | 🟡 中 | 公开信息经过筛选重组，但不应直接可外链 |
| 公开指标（粉丝数、播放量） | 🟢 低 | 各平台公开 API 可获取 |

### 4.2 技术防御

#### L1：Credit 机制本身是天然防线

Credit 额度制让批量搬取在**经济上不可行**：

| 搬取规模 | 所需 credit | 最低成本 | 经济可行性 |
|---------|:----------:|:-------:|:--------:|
| 1,000 个候选达人摘要 | 100 | ~$4.8（Starter 内含单价） | 可体验，但无外链数据 |
| 1,000 个达人深度分析 | 5,000 | ~$241 | 不划算 |
| 10,000 个达人搜索+分析 | 51,000 | $2,040+（按 Growth 超额） | 明显偏贵 |
| 100,000 个达人全量档案（含深度分析） | 510,000 | $20,400+ | 经济不合理 |
| 1,000,000 个达人 | 5,100,000 | $204,000+ | 完全不合理 |

> 对比：Modash Discovery API $16,200/年 = 3,000 credits/月 × $0.45/credit；ZoomInfo $15,000-50,000/年。NoxInfluencer 的 Credit 机制在经济上与行业对齐。

**Clearbit 经验**：月度 credit 不累积（no rollover），防止攒 credit 后一次性搬取。NoxInfluencer 同样采用月度 credit 不累积策略。

#### L2：Rate Limiting（Agent 场景特别设计）

传统 rate limit（固定窗口/漏桶）对 Agent 场景失效——Agent 调用模式天然高频、突发、难以与攻击区分。

**分层 rate limit 设计**：

| 层级 | 限制 | 说明 |
|------|------|------|
| 每分钟请求数 | Free 5 / Starter 10 / Pro 20 / Growth 40 | 基础限流 |
| 每小时请求数 | Free 50 / Starter 300 / Pro 1,000 / Growth 3,000 | 防止短时间大量调用 |
| 并发请求数 | Free 1 / Starter 2 / Pro 3 / Growth 5 | 防止并行批量抓取 |
| 单 IP 每日请求数 | Free 300 / Starter 1,000 / Pro 3,000 / Growth 10,000 | 防止多 API Key 共享同一 IP |

**Rate Limit 响应头**（每次返回，对标 Clearbit 最佳实践）：

```
X-RateLimit-Limit: 30
X-RateLimit-Remaining: 27
X-RateLimit-Reset: 1707750000
Retry-After: 60
```

#### L3：使用模式检测

| 异常模式 | 检测方法 | 动作 |
|---------|---------|------|
| 连续遍历 ID | 请求 ID 序列的连续性检测 | 标记 + 人工审核 |
| 短时间覆盖大量达人 | 单位时间 unique creator_id 数量 | 触发二次验证 |
| 搜索→分析比例异常 | discover 和 analyze 调用比 > 1:5 | 标记（正常品牌不会分析每个搜索结果） |
| 仅调用高价值 Tool | 只调 analyze/competitive_intel，不调 discover | 标记（正常品牌先搜索再分析） |
| 多账号同 IP | IP-账号关联分析 | 合并限额 |
| Agent 循环调用 | 相同 query 模式短时间重复 | 返回缓存结果 + 标记 |

> **ZoomInfo 经验**：ZoomInfo 用 PerimeterX（HUMAN）做实时 ASN 检测，直接拒绝 datacenter IP（AWS/GCP/Azure）。NoxInfluencer 作为 Agent Tool 不应完全屏蔽 datacenter IP（Agent 本身可能跑在云端），但应对 datacenter IP 施加更严格的模式检测。

#### L4：数据分级返回

| 数据字段 | `discover_creators.protected` | `discover_creators.basic_info` | `analyze_creator` |
|---------|:-----------------------------:|:------------------------------:|:------------------:|
| `creator_id`（内部） | ✅ | ✅ | ✅ |
| 平台 / 国家 / 品类 / 粉丝区间 | ✅ | ✅ | ✅ |
| 互动率 + 粗粒度真实性 | ✅ | ✅ | ✅ |
| 频道链接 / 外部 channel id / `handle` | ❌ | ✅ | ✅（`overview`） |
| 真实性精确分数 + 可疑信号 | ❌ | ❌ | ✅（`brand`） |
| 受众画像 | ❌ | ❌ | ✅（`audience`） |
| 内容分析 | ❌ | ❌ | ✅（`content`） |
| 联系方式（邮箱） | ❌ | ❌ | 仅独立收费项 / outreach 流程 |
| 预估合作费用 | ✅（区间） | ✅（区间） | ✅（`overview`） |
| 竞品合作历史 | ❌ | ❌ | `competitive_intel` 单独返回 |

> **最核心的门控规则**：默认搜索永远不返回可外链字段。品牌可以低价拿到候选人，但不能低价把达人库搬走；若显式购买 `basic_info`，也只开放有限基础字段，不开放完整深度分析。

#### L5：Canary Records（蜜罐数据）

借鉴数据行业最佳实践，在 API 返回中嵌入可追踪的合成数据：

| 机制 | 实现 | 作用 |
|------|------|------|
| **Canary 达人** | 在搜索结果中混入 2-3% 的虚构达人（合理的名字、粉丝数、互动率） | 如果在竞品数据库中发现这些"达人"，证明数据被搬 |
| **数据指纹** | 对同一达人，不同账号返回微妙不同的非关键数据（如粉丝数 152,342 vs 152,346） | 可追溯数据泄露来源 |
| **请求水印** | 每次 API 返回包含不可见的 request_id 编码在数据结构中 | 审计追踪 |

> ZoomInfo 用"技术手段对合理使用施加限制"但不公开具体实现。Canary records 是数据行业公开的最佳实践（Acxiom 有相关专利）。

### 4.3 商业防御

| 机制 | 实现 | 竞品参考 |
|------|------|---------|
| **年付绑定** | Enterprise 客户年付优先，非年付不开放竞品级数据量 | Modash: 年付 only（$16.2K/yr），银行转账 |
| **已知竞品屏蔽** | 基于公司域名/邮箱屏蔽已知竞品注册 | 行业常规做法 |
| **KYB（Know Your Business）** | Growth+ 客户需提供公司信息，验证品牌身份 | ZoomInfo: 所有客户需资质审核 |
| **数据量与价格挂钩** | 搬取所需的 credit 量远超正常使用，成本是天然屏障 | 全行业共识 |
| **一次性免费不可重复** | 绑定邮箱域名 + 设备指纹，防止多账号白嫖 | Firecrawl 实践 |

### 4.4 法律防御

| 机制 | 条款要点 | 竞品参考 |
|------|---------|---------|
| **ToS 禁止条款** | 明确禁止：数据转售、批量存储、创建竞品数据库、分享 API Key | ZoomInfo: "prohibit unauthorized use or transfer of Contact Data" |
| **数据使用限制** | 数据仅限用户自身的达人营销活动使用 | ZoomInfo: "lawful sales, marketing, and recruiting" only |
| **审计权** | 保留审计数据使用的权利 | ZoomInfo: "technological means to place reasonable use limits" |
| **违约后果** | 违约即时终止服务 + 要求删除已获取数据 + 保留诉讼权 | ZoomInfo 曾起诉爬虫并胜诉 |

> **ZoomInfo 的法律威慑效果**：ZoomInfo 在数据爬取诉讼中"击败了 14 项指控中的 13 项"（Bloomberg Law）。明确的 ToS + 真实的诉讼记录是最强的法律威慑。

### 4.5 Agent 场景特殊风险 + 对策

| Agent 特殊风险 | 为什么比传统 API 更危险 | 对策 |
|--------------|----------------------|------|
| **自动化链式调用** | Agent 可在循环中调用 discover→analyze→export，无人工干预 | L3 模式检测 + 单小时 unique creator 上限 |
| **多 Agent 协作搬取** | 用多个 Agent 分别调用不同 Tool，汇总数据 | IP + API Key 关联分析，合并限额 |
| **Agent 身份不可知** | Agent 代表谁在调用？品牌还是竞品？ | 注册时绑定品牌身份，API Key 关联到具体公司 |
| **MCP 安全漏洞** | 2025 年扫描 ~2,000 MCP Server 全部缺乏认证；恶意 MCP Server 可注入指令 | NoxInfluencer 自有认证（API Key），不依赖 MCP 协议层认证 |
| **Credential 泄露** | Agent 可能将 API Key 输出到聊天记录 | API Key 只在 MCP 配置中存储，不在 Tool 返回中暴露 |

### 4.6 防搬策略分层总结

```
L1 经济防线 ←── Credit 消耗 = 搬取成本（最根本的防线）
    ↓
L2 技术限流 ←── Rate Limit + 并发限制（基础技术防线）
    ↓
L3 行为检测 ←── 使用模式异常检测（智能防线）
    ↓
L4 数据分级 ←── 敏感字段按 tier 门控（纵深防御）
    ↓
L5 蜜罐追踪 ←── Canary Records + 数据指纹（事后追溯）
    ↓
L6 商业屏障 ←── 年付绑定 + KYB + 竞品屏蔽（商业壁垒）
    ↓
L7 法律兜底 ←── ToS + 审计权 + 诉讼权（终极威慑）
```

**Day 1 必须实现**：L1（Credit 机制，已在 01-03 设计中）+ L2（Rate Limit）+ L4（数据分级返回）+ L7（ToS）

**上线后迭代**：L3（行为检测，需数据积累）+ L5（Canary Records）+ L6（KYB，Growth+ 客户才需要）
