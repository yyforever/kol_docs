## 🔍 查阅指南

### 按角色查阅

**CEO/BD** → 执行总结 (10分钟) → 数据对标 → 产品优先级

**产品经理** → 执行总结 → 深度分析 → 快速清单（全部浏览）

**运营负责人** → 快速参考清单 → 痛点分析 → 用户画像

**工程负责人** → 快速参考清单 (痛点维度) → 深度分析 (技术可行性部分)

**市场/销售** → 快速参考清单 (原话引用) → 对标数据 → 用户画像

### 按使用场景查阅

**内部决策会**
1. 执行总结：核心5大发现 + 优先级 (10分钟)
2. 快速清单：痛点矩阵 + 对标数据 (5分钟)
3. 决策：选择P0产品方向

**产品设计评审**
1. 深度分析：具体痛点 + 根本原因
2. 快速清单：工具评估表 + 现有缺口
3. 讨论：功能设计应该如何解决痛点

**融资演讲**
1. 执行总结：5大发现 + 市场规模
2. 快速清单：痛点矩阵 + 对标表格
3. 演讲：用数据讲故事

**用户访谈**
1. 痛点分析：了解既有发现
2. 快速清单：准备验证问题
3. 访谈：探索深度原因 + 付费意愿

**竞品分析**
1. 快速清单：工具评估表
2. 深度分析：工具缺口分析
3. 分析：竞品机会在哪
