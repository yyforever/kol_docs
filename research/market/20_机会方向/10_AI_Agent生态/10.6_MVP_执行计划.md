# 10.6 达人营销API市场 - MVP执行计划

> **更新时间**: 2026-02-06
> **项目代号**: 10.6C - Agent可调用的达人营销API市场
> **目标**: 4个月完成MVP，验证PMF

---

## 执行摘要

### 一句话定位

MCP-native的达人营销API市场，让品牌Agent和开发者通过标准化接口（MCP Server + REST API）发现、匹配、预定达人。

### 核心指标

| 维度 | 目标 |
|------|------|
| **开发时间** | 4个月（16周） |
| **团队规模** | 4人（Tech Lead + 全栈 + 数据 + DevOps） |
| **开发成本** | $65K-88K（美国）或 $30K-45K（亚洲） |
| **运营成本** | $350-1,200/月（初期） |
| **MVP验收** | ≥2个付费意向，NPS>40 |

### 技术可行性

✅ **4.2/5 高度可行**
- MCP SDK成熟（96小时可达生产级）
- 聚星1亿+数据优势
- 有成熟案例参考（CreatorDB MCP、Apify MCP）

### 最大风险

⚠️ **市场时机**：技术趋势驱动 vs 用户需求驱动
- 缓解：设定Kill Point（M4未达成→停止或Pivot）

## 一、MVP功能范围

### Phase 1核心功能（M1-M4）

| 功能 | 描述 | 业务价值 | 开发量 |
|------|------|---------|:------:|
| **1. 达人搜索API** | 按平台/品类/粉丝数/互动率筛选 | 发现达人 | 28天 |
| **2. 达人详情API** | 画像/受众/合作历史/假粉检测 | 深度了解 | 45天 |
| **3. 达人匹配推荐** | 基于品牌需求推荐达人 | 智能匹配 | 40天 |
| **4. MCP Server封装** | 让Claude/ChatGPT Agent可直接调用 | Agent集成 | 31天 |
| **5. 开发者门户** | 文档/Playground/密钥管理 | 自助服务 | 49天 |
| **6. 基础设施** | AWS架构/CI/CD/监控 | 生产就绪 | 41天 |

**总计**：234人日 ≈ 11.7人月 → **4人团队×4月**

### 不包含（Phase 2+）

- ❌ 交易撮合（报价/合同/支付）
- ❌ 品牌需求发布API
- ❌ A2A协议支持
- ❌ 达人侧Agent（10.3联动）
- ❌ Agent Skills发布（SkillsMP/Smithery）

## 二、技术架构

### 核心技术栈

| 组件 | 选型 | 理由 |
|------|------|------|
| **MCP Server** | TypeScript SDK | 官方支持，96h可达生产级 |
| **API Gateway** | AWS API Gateway | 主流选择，按量计费 |
| **数据库** | PostgreSQL + Redis | PG存数据，Redis缓存 |
| **搜索** | Elasticsearch | 达人搜索行业标配 |
| **部署** | Lambda + RDS | Serverless降低成本 |
| **认证** | OAuth 2.1 + API Key | MCP安全标准 |

### 部署架构（Phase 1）

```
Claude Desktop/Agent → MCP Client
                         ↓
                    API Gateway
                         ↓
                  Lambda Function
                    ↙        ↘
              PostgreSQL   Redis
                    ↘        ↙
                  Elasticsearch
```

成本：$150-450/月（低流量）

### 数据来源

| 数据 | 来源 | 可用性 |
|------|------|:------:|
| 达人基础信息（1亿+） | 聚星数据库 | 100% |
| 合作历史/履约数据 | 聚星交易记录 | 80% |
| TTS带货数据 | 聚星TikTok Shop | 60% |
| 补充数据（可选） | 第三方API | $1.5K/月 |

**核心优势**：聚星数据省$100K-200K/年数据获取成本

| 任务 | 负责人 | 可交付 |
|------|--------|--------|
| MCP Server框架搭建 | 全栈×2 | ✅ TypeScript SDK集成 |
| 达人详情API | 全栈×2 | ✅ 画像+受众分析 |

| MCP工具定义 | 全栈×2 | ✅ 4个核心工具 |
| Claude Desktop集成测试 | 全员 | ✅ 端到端流程 |

**验收标准**：
- [ ] Claude可通过MCP调用达人搜索
- [ ] 推荐算法准确率>60%
- [ ] 工具响应时间<500ms

**Go/No-Go决策点**：
- ✅ Go：MCP集成顺利，延迟<1s

### M3: 开发者门户月（Week 9-12）

## 五、关键风险与应对

### 高风险（需立即应对）🔴

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|:----:|---------|
| **1. 市场时机错误** | 致命 | 中 | 设定Kill Point（M4未达成→停止） |
| **2. 反爬政策收紧** | 严重 | 中 | 依赖聚星数据，多源策略 |
| **3. 数据合规风险** | 严重 | 低 | 法务review，仅公开数据 |

### 中风险（需监控）🟠

- MCP标准变化：关注Linux Foundation动态
- 性能瓶颈：压测验证
- 安全漏洞：第三方审计（Phase 2）
- LLM成本失控：Token优化

### Kill Point（杀死点）

**M4结束时，如果以下条件未满足，停止项目或Pivot**：
- ❌ Beta用户反馈"不需要这个产品"
- ❌ 无付费意向（<2个）
- ❌ NPS<40（用户不推荐）
- ❌ API可用性<99%（技术不稳定）

## 六、竞争优势

### 核心壁垒

| 壁垒 | 说明 | 竞品复制难度 | 持久性 |
|------|------|:------------:|:------:|
| **聚星1亿+数据** | 历史交易+TTS+中国平台 | ⭐⭐⭐⭐⭐ | 3-5年 |
| **MCP-Native设计** | 针对Agent优化 | ⭐⭐⭐ | 1-2年 |
| **真实交易数据** | 履约历史+信用评分 | ⭐⭐⭐⭐ | 2-3年 |
| **中国市场专精** | 抖音/小红书数据 | ⭐⭐⭐⭐ | 2-3年 |

### vs 主要竞品

| 竞品 | 优势 | 劣势 |
|------|------|------|
| **Modash** | 350M数据，品牌认知强 | $16K/年太贵，无中国数据，无MCP |
| **CreatorDB MCP** | 已有MCP Server | 仅3M数据，无真实交易数据 |
| **Agentio** | $340M估值验证市场 | 中心化平台，非开放API |

**差异化策略**：
- 定位"中国+东南亚市场专精"+"MCP-native"
- 强调"真实履约数据"（竞品无）
- 价格优势（$99-299/月 vs Modash $16K/年）

```
Week -2: 假门测试（2周，$0）
  → Landing Page（"达人营销API for AI Agents"）
  → 收集Waitlist
  → 目标：100+ signups

Week 1-16: MVP开发（4月，$65K-88K）
  → M1: 基础架构
  → M2: MCP集成
  → M3: 开发者门户
  → M4: Beta测试

Week 16: Go/No-Go决策
  ✅ Go条件：≥2个付费意向，NPS>40
  ❌ No-Go：Pivot或停止
```

### 路径B：稳健路径（保守）

```
Month 0-3: 内部孵化（3月，$0外部成本）
  → 先服务聚星内部品牌Agent需求
  → 验证MCP Server价值

Month 3: 外部化决策
  ✅ 内部验证成功 → Month 4-7外部MVP
  ❌ 验证失败 → 停止
```

**建议：路径A**
- 理由：市场窗口期短（MCP生态快速变化）
- 风险缓解：假门测试$0成本，4个月可快速pivot
