## 一、场景画像

### 1.1 行业背景

- 2025年全球网红营销市场规模 **$32.55B**，同比增长10%
- **60.2%** 品牌方已使用AI进行达人发现和campaign优化
- AnyMind Group 成为全球首批将 **MCP协议** 接入网红营销平台（AnyTag）的公司
- AMT 获 $3.5M 种子轮，其AI Agent "Lyra" 可在 **5分钟** 完成传统9小时的达人签约流程

### 1.2 典型使用场景

| # | 场景 | 触发条件 | 频率 | 规模 |
|---|------|---------|------|------|
| 1 | **新品/Campaign达人发现** | 新品推广、大促、进入新市场 | 月频1-4次 | 20-300个达人 |
| 2 | **竞品达人策略监控** | 竞品曝光增加、季度分析 | 周频-月频 | 5-50品牌×100+达人 |
| 3 | **单个达人快速尽调** | 达人主动联系、BD推荐 | **高频日常**（最高频场景） | 单个达人 |
| 4 | **达人自身竞品分析** | 制定策略、准备商务提案 | 月频-季频 | 10-50达人 |
| 5 | **投资/战略达人生态洞察** | VC尽调、CMO年度预算 | 季频-年频 | 100-1000+达人 |

#### Agent场景 vs 传统SaaS的本质差异

| 维度 | 传统SaaS | Agent + Skill |
|------|---------|---------------|
| **入口** | 独立平台，需登录切换 | 嵌入对话/工作流，零切换成本 |
| **交互** | GUI点击、筛选器 | 自然语言，支持模糊需求 |
| **上下文** | 无记忆，每次从零开始 | 记住品牌偏好、历史、排除名单 |
| **输出** | 数据表格/仪表盘 | 直接生成洞察、建议、可执行方案 |
| **组合性** | 单一功能 | 可串联：查数据→写Brief→生成邮件→发送外联 |
| **付费模式** | 年费$数千-数万 | 按需调用，低门槛 |

> **核心洞察**：达人数据Skill的最大价值不是"查数据"本身，而是成为**更大工作流的一个环节**。用户不是来"用达人平台"的，而是来"做营销"的，达人数据只是过程中被自动调用的一个能力。**从"达人数据库"变成"营销Agent的眼睛"。**

### 1.3 用户画像

| # | 画像 | 规模 | 付费意愿 | 采用速度 |
|---|------|------|---------|---------|
| 1 | **出海DTC品牌增长负责人** | 10-50人, $1-20M | 💰💰💰 高 | 🔥 最快 |
| 2 | **MCN机构运营/商务经理** | 20-200人, ¥500万-5亿 | 💰💰 中 | ⏳ 跟随 |
| 3 | **中小电商品牌营销经理** | 20-100人, ¥1000万-3亿 | 💰💰 中 | ⏳ 跟随 |
| 4 | **营销Agency策略师** | 10-50人精品Agency | 💰💰💰 中高 | 🔥 较快 |
| 5 | **个人创业者/独立营销人** | 1-3人 | 💰 低 | 🔥/⏳ 分化 |

**关键发现**：
- **SMB是AI Agent最大采用群体**（65%），不是大企业
- **出海DTC + Agency** 是最佳首批目标用户（付费意愿高 + 技术接受度高）
- 早期采用者 vs 主流用户的根本区别："**追求竞争优势**" vs "**害怕被甩下**"

### 1.4 购买行为

| 维度 | 发现 |
|------|------|
| **主流定价模式** | 混合模式（基础订阅+用量计费），61%的SaaS已采用 |
| **Agent专属定价** | 按执行/对话计费（Salesforce $2/对话）、按结果计费（$20/转化线索）|
| **营销类Agent预算** | 轻量级：$19-$299/月（vs 传统SaaS $2000-5000+/月）|
| **付费偏好** | 月付为主（AI迭代快→用户要灵活性），65%遭遇过AI意外账单 |
| **价格敏感度** | 高于传统SaaS（市场太新、开源替代多、"AI应该免费"心态）|
| **GPT Store变现** | 尚未成熟，无创作者直接付费系统 |
| **ClawHub变现** | 完全免费开放，无内置变现机制 |
| **MCP变现** | 刚起步，usage-based metering工具正在出现 |

**定价建议**：Freemium + 按调用计费 + 月度封顶，核心定价锚定 **$19-$99/月**

### 1.5 现有做法与空白

| 方式 | 代表 | 现状 |
|------|------|------|
| ① 直接问LLM | ChatGPT/Claude | 🔴 幻觉严重，编造达人名字，数据过时 |
| ② GPT Store Custom GPT | CreatorGPT by Afluencer | 🟡 42K达人库太小，仅限自家平台 |
| ③ MCP Server + Agent | Apify/Bright Data | 🔴 极早期（月活16人），技术门槛高 |
| ④ AI-Native SaaS | WHOTAG/Heylist/partnrUP | 🟡 体验好但必须切出Agent环境 |
| ⑤ 传统SaaS | Modash/Upfluence | 🟡 主流但贵（$300-800/月），且与Agent割裂 |

> **最大空白**：**Agent环境内的达人数据原生接入** — 目前零成熟方案。MCP协议已就绪，基础设施（Apify/BrightData）存在，但缺少"达人数据层"中间件。**谁先做好"用自然语言在Claude/ChatGPT里查达人，返回实时结构化数据"这件事，就占据了一个全新品类。**

## 二、竞品分析

> 详见：[达人数据Skill_竞品分析.md](达人数据Skill_竞品分析.md)

### 竞品格局一句话

> **"达人数据Skill"是全新品类，零直接竞品。** AnyTag MCP是最接近的（全球唯一），但定位企业级(¥10K+/月)。营销Agent赛道2025年融资$1.5亿+，但无人专做"数据层"。

### 核心竞品定位图

```
                执行深度
                  ↑
    AMT/Agentio   │   Aha Lab
    ($3.5M-$56M)  │   ($5.7M)
    全流程Agent    │   全流程Agent
                  │
    ─────────────────────────── → 数据广度
                  │
    AnyTag MCP    │   HypeAgent
    (¥10K+/月)    │   ($30/月)
    MCP+企业级     │   对话式Agent
                  │
                  │   🎯 聚星 Skill
                  │   (PLG $19-99/月)
                  │   数据+执行闭环
```

### 关键发现
1. **AnyTag已覆盖抖音+小红书**（小红书一级合作伙伴），但数据量仅100万（聚星100x）
2. **HypeAgent通过Composio MCP开放达人数据** — 验证了"达人数据Skill"品类的可行性
3. **Phyllo（创作者数据Plaid）CEO已转向新项目** — 纯API数据业务遇增长天花板
4. **Aha Lab vs 聚星 = Agent-first vs Tool-first的代际差异**

## 四、聚星优势

### SWOT矩阵

| | 正面 | 负面 |
|--|------|------|
| **内部** | **S**: 40万出海客户(需求垄断) + 独占交易数据 + 建联支付闭环 | **W**: 数据依赖爬虫(合规风险) + MCP落后6月 + 国际品牌弱 |
| **外部** | **O**: Agent爆发+MCP标准化 + 出海预算增长 + 竞品无执行层 | **T**: 平台封杀爬虫 + AnyTag先发 + 大平台自建匹配 |

### 绝对优势（竞品5年内难追）

| 维度 | 能力 | 为什么不可复制 |
|------|------|--------------|
| **C1 客户基数** | 40万+出海企业 | 需求侧垄断，竞品从零获客中国出海市场成本极高 |
| **C3 交易数据** | 真实报价/效果/合作历史 | 只有撮合平台才有，爬虫**永远拿不到** |
| **C5 执行闭环** | 邀约→合同→支付 | MCP可触发行动，从"信息工具"升级为"执行工具" |

### ⚠️ 聚星2025现实状况（需正视）

| 维度 | 实际状态 |
|------|---------|
| AI/Agent | ❌ 零动作，AI停留在传统ML层面 |
| MCP | ❌ 无 |
| API | ✅ 有（只读数据查询），需商务联系 |
| 数据库 | 2000万-7500万（非1亿+，多源数据不一致）|
| 市场评价 | G2仅2条，Capterra 0条 — **活跃用户基数存疑** |
| 定价 | $239-500/月，3个月起签 |
| 2025动态 | 仅TikTok Shop DM插件，无重大更新 |

> **关键警示**：聚星处于"维护模式"。在竞品纷纷AI Agent化的浪潮中，沉默本身就是风险。但反过来看——**这恰好说明做达人数据Skill的紧迫性**。

### 杀手级能力（如果做）
> `nox.getMarketRate(creator)` — 返回达人历史报价区间+效果中位数+适配品类
> **这个数据只有聚星有。**

### 推荐定位

> **❌** "通用达人数据API"（和Modash/Phyllo正面竞争，必输）
> **✅** "中国出海品牌的AI达人营销执行引擎" — **Action MCP，不只是Data MCP**

| 层级 | 价格 | 能力 |
|------|------|------|
| Explorer | 免费/$29/月 | 搜索+画像（抢HypeAgent市场）|
| Pro | $199/月 | + 交易智能 + 匹配推荐 |
| Enterprise | ¥5K-10K/月 | + 执行闭环 + 批量建联 + 支付 |

## 五、PLG可行性

### 总评分：2.5/5 — 纯PLG不可行，需混合模式

| 维度 | 评分 | 关键发现 |
|------|------|---------|
| 自然病毒传播 | ⭐⭐ | MCP是"后端组件"，无Slack式viral loop |
| **Time-to-Value** | **⭐⭐⭐⭐** | **最大优势** — 自然语言查达人秒级返回，vs传统SaaS 20分钟onboarding |
| Free→Paid转化 | ⭐⭐⭐ | "搜到完美达人但看不到联系方式"是强转化触发器 |
| 增长飞轮 | ⭐⭐⭐ | 数据网络效应存在但冷启动慢 |
| **PLG障碍** | **⭐⭐** | **最大风险** — MCP营销用户全球<5000人，60-70%目标用户无法自助用MCP |

### 推荐模式：PLG底座 + Sales上层

```
PLG自助（Free/$29/$199）→ 30%收入、70%客户数（获客漏斗）
Sales-Assisted（Enterprise）→ 70%收入、30%客户数（核心营收）
```

> 用免费层+Agent平台筑低成本获客漏斗，用Sales转化Enterprise贡献核心收入。**不要期待PLG自己跑通。**

MCP协议极简（一个Python装饰器就能暴露Tool），聚星真正挑战不在协议层。

| 模块 | 难度 | 说明 |
|------|------|------|

| 数据API封装 | ★★☆☆☆ | 聚星数据库直接复用 |
| Agent交互优化 | ★★★☆☆ | Tool描述质量需迭代 |

| **数据合规** | **★★★★☆** | 向第三方AI暴露达人数据的法律风险 |

### 时间线与成本

| 阶段 | 周期 | 团队 | 成本 |

| 优先级 | 阶段 | 功能 |
|--------|------|------|
| **P0** | 搜索+评估 | 自然语言达人搜索 + 结构化档案 + 按量付费 |

### Switch分析

促进力(Push+Pull)=10 > 阻碍力(Anxiety+Habit)=7 → **切换可能但需助推**

### Top 3 Underserved Outcomes

1. **Agent内即时获取数据**（19/20）— 目前0个成熟方案
2. **10倍价格降低**（18/20）— $0.05/次 vs $300+/月
3. **自然语言→结构化列表**（17/20）— 零学习曲线

### 关键洞察

> 真正竞争对手是"手动搜索+ChatGPT幻觉"，不是Modash。

## 八、第一性原理检验

### 核心假设检验

| 假设 | 成立概率 | 风险 |
|------|---------|------|
| Agent成为营销主界面 | 65% 中高 | 🟡 时间线风险（主流采纳可能要2027-2028）|
| MCP成为标准协议 | **80% 高** | 🟢 已事实领先 |
| 达人数据是必需输入 | **50% 中** | 🔴 TikTok Shop闭环+Affiliate自申请+AI生成内容 |
| 聚星交易数据不可替代 | **40% 中低** | 🔴 TikTok API开放+数据自然贬值 |
| PLG在MCP生态可行 | **35% 低中** | 🔴 MCP用户≠目标客户，中国品牌偏好关系销售 |
| 先发优势重要 | 50% 中 | 🟡 MCP server零切换成本，半衰期短 |

### ⚠️ 策略调整建议

> **不要做"达人数据MCP Server"，做"TikTok电商智能数据层"**
> MCP只是分发渠道之一，不是产品本身。

**三层架构：**
- **Layer 1 基础数据API**（REST+MCP）：达人profile/交易数据 → 低毛利高覆盖，用于获客
- **Layer 2 预测引擎**：达人-商品匹配/GMV预测 → **高毛利，真壁垒**
- **Layer 3 Action能力**：建联/佣金/合同 → 从数据产品升级为交易基础设施

**三条分发：**
1. MCP生态 → 全球开发者
2. 平台OEM → 嵌入SARAL/Grin等现有SaaS
3. 直销 → 中国头部出海品牌

### 需要挑战的假设
1. ❌ "MCP-first产品" → ✅ "数据产品，MCP是分发渠道之一"
2. ❌ "PLG模式" → ✅ "双轨：开发者PLG + 品牌直销"
3. ❌ "$29-199/月" → ✅ "开发者免费/低价 + 品牌$500+ + OEM按调用量"
4. ❌ "达人数据是核心" → ✅ "营销决策数据是核心，达人是维度之一"
