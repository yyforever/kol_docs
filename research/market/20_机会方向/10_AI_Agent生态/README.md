## 核心假设

**入口迁移假设**：新一代个人AI Agent（如OpenClaw）正在成为用户与数字世界的新交互入口。Skills是新的"应用"形态，流量分发逻辑将发生根本性改变。

---

## OpenClaw特征分析

### 与传统SaaS/App的差异

| 维度 | 传统SaaS | OpenClaw类Agent |
|------|---------|-----------------|
| 交互方式 | GUI，用户主动操作 | 自然语言，Agent执行 |
| 运行位置 | 云端 | 本地Gateway，隐私优先 |
| 能力扩展 | 产品内置功能 | Skills动态加载 |
| 跨平台 | 需要适配各平台 | 天然多渠道（WA/TG/Discord/iMessage） |
| 自动化 | 需要用户配置 | Agent自主判断+执行 |
| 数据 | 存在服务商云端 | 本地优先，用户控制 |

### OpenClaw核心能力

- **多渠道网关**：WhatsApp/Telegram/Discord/iMessage统一入口
- **Skills生态**：ClawHub分发，AgentSkills标准
- **工具调用**：browser、exec、cron、nodes等
- **Nodes连接**：iOS/Android设备联动
- **本地优先**：Gateway本地运行，隐私可控

## 机会方向

### 方向A：Skills提供者（承接新流量）

**定位**：为OpenClaw等个人Agent提供网红营销领域的数据和工具Skills

**具体Skills**：

| Skill | 功能 | 利用的聚星资源 |
|-------|------|---------------|
| 达人查询 | 按条件搜索达人、查看画像 | C2网红库、C4算法 |
| 达人匹配 | 输入产品/品牌，推荐合适达人 | C2、C4、C3品牌数据 |
| 竞品情报 | 查询竞品的达人合作情况 | C3品牌数据 |
| 达人建联 | 批量发送建联邮件 | C5建联技术 |
| 合作追踪 | 追踪达人合作状态 | C6 Agent技术 |

**商业模式**：
- Freemium：基础查询免费，高级功能订阅
- API调用计费
- 通过ClawHub分发，获取Agent用户

**目标用户**：
- 个人创业者/小团队
- 使用OpenClaw管理工作的品牌方
- 需要快速查询达人信息的用户

**核心价值**：
- 不需要登录额外SaaS，在Agent对话中直接使用
- 比传统工具更自然的交互方式
- 可以与其他Skills组合使用

---

### 方向B：营销Agent服务（包装Agent技术）

**定位**：提供端到端网红营销Agent服务，比传统SaaS更灵活、效果更好

**与传统SaaS的差异**：

| 维度 | 传统网红营销SaaS | 营销Agent |
|------|-----------------|----------|
| 使用方式 | 登录后台，手动操作 | 对话式，"帮我找10个美妆达人" |
| 执行 | 用户一步步操作 | Agent自主完成多步任务 |
| 灵活性 | 固定功能 | 可根据需求动态调整 |
| 集成 | API对接 | Agent间协作 |
| 学习 | 需要培训 | 即用即会 |

**产品形态**：
- 独立Agent服务（类似Manus的网红营销垂直版）
- 或作为OpenClaw的高级Skills组合
- 或白标给MCN/代理商

**核心能力**：
1. **自动达人发现**：根据产品自动筛选匹配达人
2. **自动建联**：批量发送个性化邀约
3. **自动跟进**：根据回复自动跟进，人工介入关键节点
4. **效果追踪**：自动汇总合作效果

**竞品参考**：
- Stormy（YC投资，AI Agent端到端网红营销）
- 其他正在出现的营销Agent创业公司

---

### 方向C：Agent基础设施（帮企业构建Agent）

**定位**：提供网红营销Agent的基础组件，帮助企业/MCN快速构建自己的Agent

**产品形态**：
- Agent SDK/框架
- 预训练的网红营销能力模块
- 可定制的Agent模板

**目标用户**：
- MCN想要自己的达人管理Agent
- 品牌想要自己的网红营销Agent
- SaaS公司想给产品加Agent能力

---

### 方向D：Agent数据层

**定位**：为各类Agent提供专业的网红营销数据API

**与传统数据API的差异**：
- 针对Agent调用优化的响应格式
- 支持Agent友好的查询方式
- 可嵌入Agent的上下文

**产品形态**：
- Agent-native的数据API
- MCP（Model Context Protocol）兼容的数据源
- 可被任何Agent调用的数据服务
